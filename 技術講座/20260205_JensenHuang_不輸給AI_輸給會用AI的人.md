### Section 2｜宏觀框架：60 年來第一次的全面重造（02:30 ~ 08:30）

**重點摘要：** Jensen 提出 AI 革命的 3 層觀點：運算範式從顯式→隱式、整個運算堆疊被重造、AI 必須「對人有用」才會起飛。

**內容：**
- **運算範式轉移**：從「顯式程式設計」（寫 C/C++/Fortran）→ 「隱式程式設計」（告訴電腦意圖，AI 自己推導）
- **整個運算堆疊被重造**：不只 processing（GPU），連 storage、network、security 全部要重來
- **AI 必須「對人有用」**：聊天機器人只解決「背過 + 泛化」的問題；真正的智慧要會解新題、拆解、會求助、會用工具、會做研究 → **agentic AI（代理式 AI）**
- 工具使用、RAG、memory、reasoning、planning 全部是 agentic AI 的基本功

### Section 3｜企業導入心法：先百花齊放，再修剪（08:30 ~ 18:00）

**重點摘要：** Jensen 明確反對先算 ROI，主張「先讓百花齊放，到了某個時間點再用判斷力修剪」，並以 NVIDIA 自己的 EDA 工具鏈為例。

**內容：**
- **不要先算 ROI**：新技術本來就很難塞進試算表
- **兩步驟工作法**：
  1. 百花齊放：鼓勵每個團隊都去試，先說「可以」，再問「為什麼」
  2. 集中資源：找出最佳平台，把箭射在同一點；但不能太早押注
- **NVIDIA 內部實況**：什麼 AI 工具都用——Anthropic、Codex、Gemini
- **核心工作辨識**：以 NVIDIA 為例，最重要 = 晶片設計、軟體工程、系統工程；策略是把 NVIDIA 自己的技術 100% 灌進 Synopsys / Cadence / Siemens 的 EDA 工具
- > 「創新本來就不可能永遠在控制之中。如果你非得掌控一切，第一你得去看諮商，第二那是幻覺。」
- > 「我對公司要的方法，跟我對孩子一樣——去探索人生。我不會先要他們證明『做這件事將來一定會帶來財務成功』。」

### Section 4｜AI 思維直覺：無限、零、光速（18:00 ~ 25:00）

**重點摘要：** 摩爾定律太慢，AI 進步 10 年 100 萬倍；Jensen 提出「豐裕」思維與 AI 直覺三件套（無限、零、光速），要企業把這套直覺套到公司最難的問題上。

**內容：**
- 摩爾定律 = 慢得跟蝸牛一樣（5 年 10 倍、10 年 100 倍）；**AI 進步 = 10 年 100 萬倍**
- **「豐裕（abundance）」定義**：看著巨大到離譜的問題說「算了，我全包了」——不是只治癌症，是把人類所有痛苦都解決掉
- **AI 直覺三件套**（套到公司最難的問題）：
  - **無限**：算力無限，不用再切小塊算
  - **零**：成本 / 重量 → 0
  - **光速**：速度不再重要，因為已經在光速
- **具體實例**：圖譜分析（graph analytics）——以前要把幾兆邊的圖切段，現在「全量處理」
- > 「如果你的競爭對手一開始就用這種直覺，一切都會被改寫。」

### Section 5｜五層蛋糕：AI 應用到底是什麼（25:00 ~ 36:00）

**重點摘要：** 智慧三件（感知、推理、規劃）→ AlexNet 突破 → 自我監督學習 → 模型規模爆炸 → 軟體從「預錄」走向「依脈絡生成」。

**內容：**
- **AI 智慧三大能力**：感知（perception）→ 推理（reasoning）→ 規劃（planning）
- **歷史脈絡**：
  - 2012 AlexNet（兩位工程師 + 幾張 GPU）打破 30 年電腦視覺研究
  - 突破原因：深度學習 + **自我監督學習**（不再受 label 限制）
  - 模型規模一路從幾百 → 幾億 → 幾十億 → 幾兆參數
- **軟體典範轉移**：
  - 過去（CD-ROM）：預錄 / 預先寫死 → 檢索式（retrieval-based）
  - 未來：依脈絡而生、每個 instance 都不同 → 生成式（generative）
- **對工程組織的衝擊**：軟體怎麼開發、PM 怎麼定義規格、QA 怎麼驗證、怎麼 patch、怎麼讓它永遠保持最新——Jensen 自問上千個問題
- > 「這場對話以前從沒發生過。概念早存在、先驗早存在，但每一個字都沒出現過。」

### Section 6｜實體 AI 與企業轉型：科技優先（36:00 ~ 結束）

**重點摘要：** 工具使用是 AI 最大突破；物理 AI 帶來新商機；TAM 暴增 100 倍；每家公司都該是科技公司；寫程式 = 打字的商品化 → 領域專業才是終極價值。

**內容：**
- **工具使用是 AI 最大近期突破**：因為工具本來就是「明確可用」的——F = ma、V = IR，你不會叫 AGI 重新發明螺絲起子
- **物理 AI（Physical AI）**：要懂因果關係；例子——小孩都懂「骨牌效應」，LLM 完全沒概念，所以要**教它** → 全新商機
- **TAM 論點**（人類史上第一次）：
  - IT 產業 ≈ 1 兆美元；全球經濟 ≈ 100 兆美元
  - **人類史上第一次，我們接觸得到全部**
- **每家公司都是科技公司**：

  | 傳統身分 | 真正想當的 |
  |----------|------------|
  | 迪士尼 | Netflix |
  | 賓士 | Tesla |
  | Walmart | Amazon |

- **核心論點**：處理電子（無限複製）vs 處理原子（被重量限制）→ 從 CD-ROM 變電子交付 → **價值暴增 1000 倍**
- > 「人類史上第一次，你可以用『隱式方式』去編程。你只要告訴它你要什麼，電腦就會替你把程式碼寫出來。」
- > 「我的問題比我的答案更有 IP 價值。」

### Section 7｜收尾：兩大實戰建議

**重點摘要：** 租 vs 自建 → 自己做一台電腦；顛覆「human in the loop」→ AI in the loop。

**內容：**
- **自己做一台電腦**：
  - 不要全租雲端，也不要全自建；至少掀引擎蓋、換機油——建立 **tactile understanding**
  - 主權 / 專有資料 / 機密對話 → 必須 on-prem
  - **最重要的 IP 不是答案，而是問題**（不想讓別人知道你在想什麼）
- **顛覆「human in the loop」→ AI in the loop**：
  - 每家公司都要「AI 在迴圈裡」
  - 員工 + 一群 AI = 公司未來的智慧財產
  - > 「未來每一位員工都會有很多個 AI 在迴圈裡，那些 AI 會變成公司的智慧財產——那就是未來的公司。」
- 收尾金句：**「你打給我，我去打給他」**（致 Chuck）—— 立刻行動

## 六、給後端工程師的 6 條行動筆記

1. **百花齊放**：團隊同時試 Anthropic / Codex / Gemini agent 框架，不先統一棧
2. **辨識核心工作**：對我們 = API 設計、DB schema、query 優化 → 灌 AI 進去做 EDA 等級的工具
3. **AI 直覺**：寫 code 時假設 compute / latency / cost → 0，重新思考 API 設計
4. **重寫工程組織**：PM 定義「意圖」、工程師變「意圖審稿人」、QA 驗證 agent 行為
5. **AI in the loop**：把 agent 變可累積經驗的 IP 資產（提示詞庫 + 微調模型 + 私有知識圖譜）
6. **資料主權**：on-prem AI 保護「問題 IP」，機密對話不外包公有雲
