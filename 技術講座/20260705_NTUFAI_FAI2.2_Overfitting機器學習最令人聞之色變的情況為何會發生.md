## 一、主題與背景

FAI 2.2「機器學習最令人聞之色變的情況為何會發生？」是林軒田教授 Machine Learning 章節進入 Overfitting 議題的第一講，約 16 分鐘。本講回答 FAI 2.1 末尾留下的關鍵問題：為什麼 Hypothesis Set 越複雜表示力越強，卻反而讓模型「變笨」？

Overfitting 的本質是「訓練資料做得好、測試資料做得差」。林軒田教授用一個直覺的類比：「像是在模擬考題上做 100 分，但只是把考古題死背下來，所以題目一變化就不行了。」Overfitting 正是 FAI 1.4 Generalization 議題的「反面案例」——$E_{in}$ 小不代表 $E_{out}$ 小，當 Hypothesis Set 太複雜時，模型把訓練資料的 noise 當作 signal 學進去，卻失去了對未見資料的預測能力。

本講的另一條主軸是 **Bias-Variance Trade-off**。林軒田教授用「打靶」做類比：靶心是理想 $f$，每次找到的 $g$ 像射出的箭；藍點與靶心的偏移 = bias（瞄準有沒有準）；藍點彼此之間的散開程度 = variance（手會不會抖）。最理想的狀態是 bias 小且 variance 小（瞄得準又穩），但實務上模型複雜度 ↑ 會讓 variance ↑ 而 bias ↓，形成 trade-off。

本講也引入 **Testing Error 的 Upper Bound** 概念：理論上 testing error 有一個 upper bound = training error + model complexity（VC Dimension）。這個 bound 告訴我們「$E_{out}$ 不可能比這個數字大太多」——但代價是 bound 本身也是一個估計，無法精確算出 $E_{out}$。當模型太複雜，bound 變寬鬆（即使 $E_{in}$ 下降，整個 bound 還是上升），這就是 Overfitting 的數學形式。

最後，林軒田教授用 **bias-variance 圖**總結：x 軸是 VC Dimension（模型複雜度），藍線是 $E_{in}$（隨複雜度下降），紅線是 model complexity（隨複雜度上升），testing error 的 upper bound = 兩者加總。理想點在中間：$E_{in}$ 已經下降但 model complexity 還沒上升太快。當模型太簡單（VC Dimension 小）叫 **Underfitting**——$E_{in}$ 大但 variance 小、bias 大；當模型太複雜（VC Dimension 大）叫 **Overfitting**——$E_{in}$ 小但 variance 大、bias 小。

**核心議題一句話**：Overfitting 是 ML 最令人聞之色變的情況——$E_{in}$ 小 $E_{out}$ 大，模型把 noise 當 signal；其成因是 Bias-Variance Trade-off：模型複雜度 ↑ → bias ↓ 但 variance ↑，存在一個最佳點讓 testing error 最小；Model Selection（選最適合複雜度的模型）是 ML 最困難的核心難題之一，需要 VC Dimension、Validation Set、Regularization 等多管齊下。

## 二、章節脈絡

### Section 1｜Overfitting 的定義：考古題 vs 實戰（00:00 ~ 01:30）

**重點摘要：** Overfitting = training 做得好但 testing 不好；像模擬考 100 分但只是死背考古題；題目變化就不行。

**內容：**
- Overfitting 的定義：training 時看起來做得好，但 testing 時做得不好
- 類比：在看到的考古題上做 100 分，但只是把考古題死背下來
- 題目一變化（沒見過的測試資料）就不行
- 這是 ML 中「最令人聞之色變」的情況
- Overfitting 是 FAI 1.4 Generalization 議題的「反面案例」

> "簡而言之他就是在training的時候看起來做得好，但是實際上呢，他在testing的時候做的其實不好，就是我在看到的東西，看到考古題上面做到100分，可是我可能就是把這些考古題死背下來，所以只要題目有變化，我就不行了。"

### Section 2｜打靶比喻：Bias 與 Variance（01:30 ~ 04:30）

**重點摘要：** 靶心 = 理想 $f$；藍點（找到的 $g$）跟靶心的偏移 = bias；藍點彼此的散開 = variance；理想狀態是 bias 小 + variance 小。

**內容：**
- 靶心：理想的 $f$（只有老天才知道）
- 每次 algorithm 跑出來的 $g$ 像射出的箭（藍點）
- 偏移量來自兩個東西：bias（瞄準有沒有準）+ variance（手會不會抖）
- 理想圖：瞄得準又穩（bias 小、variance 小）
- 次優情況：瞄準準但手抖（low bias、high variance）
- 另一個次優：瞄錯了但手穩（high bias、low variance）
- 最差：瞄錯且手抖（high bias、high variance）
- $g$ 跟 $f$ 的差異 = bias + variance 共同造成

> "這個偏移叫做 bias，就是你瞄準有沒有瞄準在靶心，然後每一次的data上面每一次它彼此之間的差異叫做 variance。"

### Section 3｜Model Complexity 對 Bias-Variance 的影響（04:30 ~ 09:00）

**重點摘要：** 簡單模型（Linear）→ low variance、high bias；複雜模型（Quadratic）→ high variance、low bias；存在最佳複雜度使 testing error 最小。

**內容：**
- 用 Linear Model：每次找出來的 $g$ 都很接近（線的可能性少）
- 簡單模型 → variance 小；但可能根本沒有 cover 到 $f$ → bias 大
- 用 Quadratic/Cubic Model：可以 cover 到 $f$（bias 小），但每次找出來的 $g$ 變異度大（variance 大）
- Trade-off：模型複雜度 ↑ → bias ↓ 但 variance ↑
- 太小也不好（包含不到 $f$）、太大也不好（variance 太大）
- 需要取得平衡：瞄準在中心 + 彈性不要太大
- 實務上非常困難

> "如果我的模型是比較簡單的，所以我的variance其實很低，可是它的representation power不夠力，它沒有辦法涵蓋到我理想上的想要的function的樣子。"

### Section 4｜Testing Error 的 Upper Bound（09:00 ~ 11:30）

**重點摘要：** $E_{out} \leq E_{in} + \Omega(H)$（model complexity）；bound = training error + 模型複雜度；理論證明但實務難算。

**內容：**
- 理論假設：training data 跟 testing data 同 distribution
- 如果不同 distribution，training 學到的東西就無法 generalize 到 testing
- Testing error 的 upper bound：$E_{out} \leq E_{in} + \Omega(H)$
- 第一項：training data 上的 error
- 第二項：模型的複雜度（參數量、VC Dimension 等）
- 兩項都希望越小越好
- 詳細證明需要修「系上彥寰老師」的理論課
- 概念理解即可：希望 testing error upper bound 越低越好

> "理論上我們的machine learning其實都會假設說我們的訓練資料跟測試資料它其實是同樣的distribution出來的，因為如果它不是同樣的distribution出來，那不就代表說你在訓練資料上面學其實它也沒有辦法通用到測試資料嗎。"

### Section 5｜Bias-Variance 圖：VC Dimension vs Error（11:30 ~ 14:00）

**重點摘要：** 藍線 $E_{in}$ 隨 VC Dimension 上升而下降；紅線 $\Omega(H)$ 隨 VC Dimension 上升而上升；兩者加總（upper bound）在中段有最低點。

**內容：**
- x 軸：VC Dimension（模型複雜度，越右越複雜）
- y 軸：error / complexity
- 藍線：in-sample error（$E_{in}$）隨 VC Dimension 上升而下降
- 紅線：model complexity（$\Omega(H)$）隨 VC Dimension 上升而上升
- Testing error upper bound = 藍線 + 紅線
- 最低點在中段：$E_{in}$ 下降得差不多了，但 $\Omega(H)$ 還沒上升太多
- 這個最低點就是「最佳模型複雜度」
- 右半邊：模型太複雜（overfitting 開始發生）
- 左半邊：模型太簡單（underfitting 發生）

> "理想上我們是要找一個最好的一個巧妙的sweet point，它 可以讓這兩個達成一個很好的平衡，就是最好的情況其實是在一個中間的位置，不要太高也不要太低。"

### Section 6｜Underfitting vs Overfitting（14:00 ~ 15:52）

**重點摘要：** Underfitting = 模型太簡單 → small variance + big bias；Overfitting = 模型太複雜 → big variance + small bias；Model Selection 非常困難。

**內容：**
- Underfitting：VC Dimension 小，模型簡單
  - $E_{in}$ 大（沒辦法 fit 好 training data）
  - variance 小、bias 大
  - 對應到打靶圖：瞄錯 + 穩定
- Overfitting：VC Dimension 大，模型複雜
  - $E_{in}$ 小（fit 得很好）
  - variance 大、bias 小
  - 對應到打靶圖：瞄準 + 手抖
- 兩者都不好，需要找到 sweet spot
- 怎麼找？非常困難（Model Selection 議題）
- 因為我們根本不知道理想的 $f$ 在哪裡

> "model selection 非常重要，可是非常的困難，非常不 naive，不知道怎麼找一個好的位置當成是最理想上的 model，因為我們根本不知道理想的這個 f 在哪裡。"

## 五、核心主旨

> Overfitting 是 Machine Learning 最令人聞之色變的情況——$E_{in}$ 小 $E_{out}$ 大，模型把 noise 當 signal 學進去。其根源是 Bias-Variance Trade-off：模型複雜度 ↑ → bias ↓ 但 variance ↑，存在一個最佳點使 testing error 最小。理論上 testing error 有 upper bound $E_{out} \leq E_{in} + \Omega(H)$（model complexity）；實務上需要 VC Dimension、Validation Set、Regularization 等方法做 Model Selection。Underfitting 是模型太簡單（small variance + big bias），Overfitting 是模型太複雜（big variance + small bias）——兩者都不好，需要在 Bias-Variance 圖中找到 sweet spot。Model Selection 之所以困難，是因為我們根本不知道理想的 $f$ 在哪裡。
