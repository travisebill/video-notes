## 一、主題與背景

FAI 5.2「K 平均分群演算法（K-Means Clustering）」是林軒田教授在 Clustering 章節的第一個具體演算法，把分群問題從概念形式化為一個可最佳化的數學問題。

節目一開始先把分群目標形式化。假設要把 N 筆資料分成 K 群，演算法需要同時決定兩件事：每一筆資料屬於哪一群（分群 S），以及每一群裡最具代表性的中心點 μ（Prototype）。教授用「隊長與小隊員」的譬喻說明這個雙重決策的本質——隊長要有足夠的代表性，能讓小隊員歸屬於他；小隊員的歸屬也要讓 E_in 越小越好。

接著把目標量化為 E_in 函數。E_in 是每個點與所屬群中心 μ 的距離平方的平均值，K-Means 的目標就是找到一種分法與對應的中心，使 E_in 越小越好。教授說明這個雙重最佳化問題很難一次解出，因此採用兩步交替最佳化的策略。

**核心議題一句話**：K-Means 是把分群問題形式化為 E_in 最小化的雙重最佳化問題，並用兩步交替（更新分群 S、更新中心 μ）迭代收斂的簡潔解法——簡單、快速，但 K 的選擇與初始中心的隨機性是兩大實務痛點。

## 二、章節脈絡

### Section 1｜分群目標的形式化（00:00 ~ 04:00）

**重點摘要：** 把分群目標量化為 E_in 函數——每個點與所屬群中心距離平方的平均值；同時決定分群 S 與中心 μ 是雙重最佳化問題，難以一次解出。

**內容：**
- 假設要把 N 筆資料分成 K 群
- 兩個決策變數：分群 S（每筆資料歸屬哪一群）、中心 μ（每群的代表性點）
- 隊長 vs 小隊員譬喻：隊長要有足夠代表性、小隊員要歸屬於最近的隊長
- E_in 定義：每個點與所屬群中心距離平方的平均值
- 目標：minimize E_in
- 雙重最佳化：S 與 μ 互相影響，很難一次同時最佳化

> "我們會因為有這個目標所以我們就會寫出一個這個演算法...就是找出來的這些partition跟這裡面具有代表性的這些每一個partition每一個Send裡面的這些mew它能夠滿足這件事情這是E-in了。"

### Section 2｜兩步交替最佳化（04:00 ~ 08:00）

**重點摘要：** K-Means 採用兩步交替最佳化——第一步固定 μ、更新 S（每筆資料歸屬最近中心）；第二步固定 S、更新 μ（用群內平均）；反覆迭代直到收斂。

**內容：**
- 兩步交替最佳化策略：
  - **步驟一（更新 S）**：固定中心 μ，把每筆資料歸屬到距離最近的中心 → 確保在 μ 固定下 E_in 最小
  - **步驟二（更新 μ）**：固定分群 S，把每群的平均值作為新中心 → 群內平方距離最小的點
- 兩步反覆迭代，直到分群不再改變（收斂）
- 收斂後 E_in 保證是區域最小值
- 計算效率高：每次迭代只需掃過所有 X 一次

> "它的做法呢其實是我先artimize partition，然後在artimize這個prootype，在artimize partition，在artimize prototype，它是這樣Ital-rated的來做。"

### Section 3｜互動 Demo 與收斂過程（08:00 ~ 11:00）

**重點摘要：** 透過互動 demo 展示 K-Means 的完整流程——隨機選 K 個中心、依距離分群、計算新中心、重新分群、直到分群不再改變；分群結果以顏色區分。

**內容：**
- 互動 demo：先在 2D 空間撒大量點（X 分布）
- 隨機選 K 個初始中心（可手動選或隨機）
- 第一輪：依距離分群（每個 X 歸屬最近的中心，用顏色區分）
- 第二輪：計算每群的中心點（Centroid）
- 第三輪：重新依距離分群
- 反覆迭代，直到分群不再改變
- 視覺化：每群用不同顏色表示，最終呈現「扛不下去」的狀態

> "它就會微調了，然後微調之後呢...re-design point...這就是很輕的Scentroid它就會在微調一下。"

### Section 4｜K-Means 的限制（11:00 ~ 16:17）

**重點摘要：** K-Means 三大限制——K 值必須事先指定（不同 K 結果差異極大）、初始中心影響最終結果、每筆資料只能歸屬一群（無法表達多重特質）；K 值選擇沒有標準方法，需搭配領域知識或交叉驗證。

**內容：**
- **限制一：K 值必須事先指定**
  - 真實資料往往不知道該分幾群
  - 同一份資料在 K=2、K=4、K=7 下的差異極大
  - 沒有標準方法決定 K，需搭配領域知識或實驗 tuning
- **限制二：初始中心敏感度**
  - 不同的初始中心可能收斂到不同的區域最佳解
  - 同一份分布可能有多種合理的分法（取決於初始選點）
- **限制三：硬分群**
  - 每筆資料只能歸屬一群
  - 對於同時具有多重特質的個案（例如既是學生也是高消費族）無法表達
  - 資料分布不均勻時，某些群容易出現規模過大的情況
- **優勢**：簡單、快速、可解釋、容易實作
- **改良方向**：用領域知識設定 K，或搭配其他演算法（如 RBF）做 prototype 選擇

> "你的K值其實會對它最後的分類分群的結果是非常Sacety的...也沒有一個Gonstander方法說你就照著這樣這這個Process你就可以找到最好的K。"

## 三、關鍵概念定義

| 概念 | 定義 | 重要性 |
|------|------|--------|
| 分群（Clustering） | 把相似的 X 歸類到同一群的無監督任務 | 非監督式學習的核心任務之一 |
| 中心（μ, Prototype） | 每群內最具代表性的點，通常取群內平均 | K-Means 的關鍵參數 |
| 分群（S, Partition） | 每筆資料歸屬的群組分配 | K-Means 的決策變數之一 |
| E_in | 群內平方距離的平均值，越小代表分群品質越好 | K-Means 的最佳化目標 |
| 兩步交替最佳化 | 固定一個變數、優化另一個，反覆迭代的策略 | K-Means 的核心演算法設計 |
| 區域最佳解（Local Optimum） | 演算法收斂到的局部最小 E_in | K-Means 只保證區域最佳 |
| 硬分群（Hard Clustering） | 每筆資料只能歸屬一群的分群方式 | K-Means 的特性（vs Soft Clustering） |
| K 值選擇 | 事先指定分群的數量 | K-Means 的關鍵超參數 |
| 初始中心敏感度 | 不同的初始中心可能收斂到不同的最終分群 | K-Means 的穩健性問題 |

## 五、核心主旨

> K 平均分群演算法（K-Means）是把分群問題形式化為「最小化 E_in（群內平方距離平均）」的雙重最佳化問題，並採用「更新分群 S、更新中心 μ」的兩步交替最佳化策略迭代求解。這種簡潔設計讓 K-Means 成為最容易實作、最快速的分群方法之一，互動 demo 清楚展示從隨機中心、依距離分群、計算新中心、重新分群、直到收斂的完整流程。然而 K-Means 也有三大限制：第一是 K 值必須事先指定且對結果影響極大；第二是初始中心的隨機性會導致不同的區域最佳解；第三是每筆資料只能歸屬一群的硬分群特性，無法表達多重特質的個案。實務上 K 值選擇沒有標準方法，需要搭配領域知識或交叉驗證，這也成為後續 RBF Network（用 K-Means 找 prototype）等進階方法的切入點——K-Means 是基礎，但對複雜分布往往需要更精緻的工具。
