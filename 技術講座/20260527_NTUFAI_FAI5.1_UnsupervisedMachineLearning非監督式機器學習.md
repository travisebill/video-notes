## 一、主題與背景

FAI 5.1「非監督式機器學習」是陳縕儂教授在 Unsupervised Learning 章節的開場影片，先為整個章節的範疇與四大任務定調，再透過具體應用案例幫助建立直覺。

節目一開始先把機器學習的三大類別整理清楚。監督式學習需要成對的（X, Y）資料，但實務上常有大量 X 卻缺乏 Y 標記的情況；增強式學習靠環境互動累積獎勵；非監督式學習則是在只有 X 的情況下，從資料分布本身挖掘結構。教授用硬幣分類做開場譬喻：若已知每枚硬幣的面額，就是監督式分類；若只知道尺寸與重量等特徵而不知面額，則只能依資料分布把它分成幾群，這就是分群（Clustering）的核心精神。

進入四大任務介紹。第一個是分群（Clustering）——屬於無標籤的多類別分類問題，教授舉出三個具體應用場景：電商平台依消費頻率與金額把顧客分群、賣場以車牌辨識搭配購物金額篩選高消費潛力客戶、半導體機台以感測器訊號偵測異常機台。第二個是異常偵測（Outlier Detection）——屬於資料極度不平衡的二元分類問題，Whoscall 在電話號碼尚未被大量標記前，透過撥打量與接觸模式偵測騷擾電話，就是典型案例。

第三個是密度估計（Density Estimation）——屬於無標籤的迴歸問題。教授以 Google Maps 為例：透過部分用戶的車速資料估計整條道路的真實車流密度，再以紅黃綠顏色呈現給所有駕駛；同樣的概念也能應用在疾病熱區預測與租金分布估計。第四個是表示學習（Representation Learning）——即便沒有 Y，仍可從 X 本身萃取出最具代表性的特徵。教授說明常見做法是把 X 的一部分變形當成自督式學習的目標 Y，這就是 Self-Supervised Learning 的精神。

節目接著展示 Self-Supervised Learning 的兩個經典案例。第一是 2013 年的 Word2Vec：透過上下文預測，把每個詞壓縮成向量，使語意相近的字在向量空間中彼此靠近。第二是當代的大型語言模型：把整段文字壓縮成向量，再用 next-token prediction 的方式學習。教授點出，這整個訓練機制其實都是 Self-Supervised Learning 的延伸，是表示學習的工程化典範。

**核心議題一句話**：非監督式學習是處理「只有 X、缺乏 Y」情境的完整方法論——分群處理無標籤分類、異常偵測處理不平衡二元分類、密度估計處理無標籤迴歸、表示學習從 X 中提取最具資訊的特徵；這四項任務構成後續 FAI 5.2~5.4 章節的骨架。

## 二、章節脈絡

### Section 1｜監督式 vs 非監督式 vs 增強式（00:00 ~ 10:00）

**重點摘要：** 三大機器學習類別的分野——監督式需要（X, Y）對、增強式靠環境互動累積獎勵、非監督式從 X 本身的分布挖掘結構；硬幣分類案例清楚展示監督式 vs 非監督式的差異。

**內容：**
- 監督式學習需要大量（X, Y）成對資料，但實務上常有大量 X、缺乏 Y 標記
- 增強式學習：靠環境互動累積獎勵（遊戲、對話、溝通）
- 非監督式學習：只有 X、沒有目標 Y，從資料分布挖掘結構
- 硬幣分類案例：有尺寸+重量+面額 = 監督式；只有尺寸+重量 = 必須依分布分群
- 分群（Clustering）：Unsupervised Multi-class Classification
- 應用：把硬幣分成三群（依尺寸+重量的分布），即使不知道面額也能分

> "什麼是UnsuperwiseLenning呢？就是我們剛剛把這些硬幣的這些X我們有很多它的尺寸拿跟它的重量，但是我們現在不知道這一個X對應的Y到底是什麼，所以我們沒有辦法把這BN標記上顏色。"

### Section 2｜分群的實務應用案例（10:00 ~ 22:00）

**重點摘要：** 分群在真實世界的三個具體應用——電商顧客分群（VIP vs 潛力客戶）、賣場車牌辨識（高消費潛力客戶篩選）、異常偵測（半導體機台感測器、Whoscall 騷擾電話）；資料極度不平衡是異常偵測的關鍵特徵。

**內容：**
- 電商顧客分群：依頻率與金額把顧客分成幾群
  - 高消費 + 低造訪 = VIP 客戶（重點保留）
  - 高造訪 + 低消費 = 潛力客戶（給折扣刺激轉換）
- 賣場車牌辨識 + 購物金額：篩選高級車但消費力未激發的客戶
- 異常偵測（Anomaly Detection）：
  - 半導體機台感測器訊號偏離正常 pattern → 需停機檢修
  - 大部分資料正常、僅少數為雜訊（資料極度不平衡）
  - Whoscall：透過撥打量（out-degree）模式判斷騷擾電話
    - 正常人 out-degree ≈ 5；騷擾電話 out-degree 極大
    - 即使號碼尚未被標記，也能透過 graph 模式偵測

> "Anonym基本上它就是大部分時候其實都正常的，只有少部分形狀是異常的，所以你可能會看到的是非常非常不平衡的這種Label。"

### Section 3｜密度估計（Density Estimation）（22:00 ~ 28:00）

**重點摘要：** 密度估計是無標籤迴歸問題——透過觀察到的 X 估計背後的真實分布；Google Maps 道路車流密度、疾病熱區預測、租金分布都是典型應用。

**內容：**
- 密度估計 = 無標籤的 Regression 問題（因為 density 也是一個數值）
- Google Maps：透過部分用戶的車速資料估計整條道路的真實車流密度
  - 紅色 = 壅塞、黃色 = 中等、綠色 = 暢通
  - 並非每台車都有裝，因此需要從部分觀察推估整體
- 疾病熱區預測：根據爆發點預估未觀察到的熱區，提前部署防疫
- 租金分布：估計一個城市不同區位的租金機率分布
- 與時間維度結合的挑戰：當下密度與未來密度不同（5 分鐘後的壅塞可能改變）

> "這是一個我記得是早上8點的時候學校附近的這些路的例子...這邊的這個Density Estimation它其實就是一個非常實用的一個運用在我們的交通的Trap上面去估計它的Density。"

### Section 4｜表示學習與自督式學習（28:00 ~ 39:44）

**重點摘要：** 表示學習從 X 中提取最具代表性的特徵；自督式學習（Self-Supervised Learning）把 X 的一部分變形當成目標 Y；Word2Vec 與大型語言模型都是自督式學習的經典案例。

**內容：**
- 表示學習的核心：在沒有 Y 的情況下，從 X 中學到「最關鍵特徵」的表示
- 自督式學習（Self-Supervised Learning）：
  - 把 X 的一部分變形當成目標 Y
  - 例如：把圖片切成拼圖讓模型拼回去、把圖片轉正讓模型糾正、彩色變灰階讓模型補色
  - 讓模型在「認識老虎之前」就學會「老虎的視覺特徵」
- Word2Vec（2013）：NLP 領域的經典自督式學習
  - 把每個詞壓縮成向量
  - 透過上下文預測（如「Eat Apple」中預測 Apple）
  - 語意相近的字在向量空間中彼此靠近（Apple vs Orange 都在水果區）
  - 每個字變成一個 Word Embedding
- 大型語言模型（LLM）：
  - 把整段文字壓縮成向量
  - 用 next-token prediction 學習
  - 概念與 Word2Vec 相同，只是粒度從「一個詞」升級到「整段文字」
- 自督式學習的精神：沒有外部 Y，但能從 X 內部結構中學到豐富表示

> "像這種self super把實際上你方法其實非常的常見...Wattoback它顧名思義就是把每一個Worn都變成是Bacter。"

## 三、關鍵概念定義

| 概念 | 定義 | 重要性 |
|------|------|--------|
| 監督式學習（Supervised Learning） | 需要（X, Y）對的機器學習 | 主流學習形式，但需要標記成本 |
| 增強式學習（Reinforcement Learning） | 靠環境互動累積獎勵的學習 | 遊戲、對話等互動場景的核心 |
| 非監督式學習（Unsupervised Learning） | 只有 X、缺乏 Y，從分布挖掘結構的學習 | 實務上最常見的場景（資料難標記） |
| 分群（Clustering） | 把相似的 X 歸類到同一群，無標籤的多類別分類 | 顧客分群、文件分群、圖像分群 |
| 異常偵測（Outlier Detection） | 大部分資料正常、僅少數為雜訊的不平衡二元分類 | 詐騙偵測、機台故障、垃圾郵件 |
| 密度估計（Density Estimation） | 從觀察到的 X 估計背後的真實分布 | 道路壅塞、疾病熱區、租金分布 |
| 表示學習（Representation Learning） | 從 X 中提取最具代表性特徵的學習 | 後續深度學習的基礎 |
| 自督式學習（Self-Supervised Learning） | 把 X 的一部分變形當成目標 Y 的學習 | Word2Vec、LLM 等現代模型的訓練機制 |
| Word Embedding | 把詞壓縮成稠密向量的表示 | NLP 領域的基礎設施 |
| 上下文預測（Context Prediction） | 用詞的上下文訓練表示的技術 | Word2Vec 的核心訓練機制 |

## 五、核心主旨

> 非監督式機器學習是處理「只有 X、缺乏 Y」情境的完整方法論，包含四大任務：分群（Clustering）處理無標籤的多類別分類、異常偵測（Outlier Detection）處理資料極度不平衡的二元分類、密度估計（Density Estimation）處理無標籤的迴歸問題、表示學習（Representation Learning）從 X 中提取最具資訊的特徵。實務應用上，分群可用於電商顧客經營、賣場高消費潛力客戶篩選；異常偵測可用於半導體機台故障預警、Whoscall 騷擾電話偵測；密度估計可用於 Google Maps 道路壅塞顯示、疾病熱區預測；表示學習則是 Word2Vec 與現代大型語言模型的共同訓練機制。這四項任務構成 FAI 5.2（K-Means）、FAI 5.3（Autoencoder 與表示學習）、FAI 5.4（RBF 與密度估計）三章節的骨架，理解這張地圖便能掌握整個 Unsupervised Learning 章節的全貌。
