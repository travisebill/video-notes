## 一、主題與背景

台大資訊工程學系於 2020 年正式將「人工智慧導論（FAI）」納入大三下必修（CourseSpydo 編號 FAI2026），由兩位老師共同授課——前半段由陳上則教授主講、後半段由陳縕儂教授接手。本講為整門課的開場：第一段說明課程改革的歷史脈絡（1999 → 2009 → 2018 → 2020 必修演變）、第二段帶學生走過評分 policy（4 次作業 + 期中考 + final project）、最後預告第三週就會 release 的 AI project（遊戲類）——這是本學期刻意把 project 提前的做法，要避免過去作業夾雜 project 導致學生期末趕死。

課程設計目標是當作「橋接課程」：讓同學具備 ML 與 AI 基礎知識後，能順利銜接高年級的進階課程與專題。教材前半段多為傳統 AI（Search、Knowledge Representation），後半段偏 Data-Driven（ML、Deep Learning 入門），並大量參考 MIT 6.0001 等國際課程的 topic mapping。

**核心議題一句話**：為什麼在 2026 年這個時間點，把人工智慧導論納入台大資訊系大三下必修是必要且迫切的——以及這門課如何用「前半 AI、後半 ML」的雙段設計，把學生從零帶到能開始做 AI project 的狀態。

## 二、章節脈絡

### Section 1｜課程改革的歷史脈絡（00:00 ~ 04:30）

**重點摘要：** AI 從選修變必修的決議路徑——1999 網路 → 2009 物件導向 → 2018 討論 → 2020 正式入法。

**內容：**
- 台大資訊系約每 10 年進行一次課程改革檢討
- 2010 年版必修 54 學分；2017 年因硬體實驗課減少變少；現行版本 51 學分，新增本門「人工智慧導論」
- 2018 年起課程改革小組進行調查：畢業校友回饋「AI 相關」與「軟體工程（Design Pattern）」兩項被強烈建議納入必修
- 2020 年經全校課程委員會通過，「人工智慧導論」正式成為大三下必修
- 過去系上 ML 課由林軒田老師（宣言老師）開在電機系、資訊系選修，超過一半本系大學部學生都修過——顯示 ML 是普遍需求的技能

> "這門課的設計其實是算是在近期的課程改革，2018 年的時候就有課程改革的小組進行一些討論…根據畢業的校友問卷，AI 相關的科目是有非常多校友反應這個是非常重要，應該把他納入必修。"

### Section 2｜為什麼放在大三下、為什麼列為必修（04:30 ~ 09:00）

**重點摘要：** 預修基礎（線代、機率）+ 課程負載分散 + 必修帶來的資源效應。

**內容：**
- 預修：學生需具備大二「線性代數」與「機率論」基礎——這也是這門課放在大三下的主因
- 大三上是相對繁忙的學期（多門專業必修），把 AI 導論安排在大三下反而讓學生有較多餘裕消化內容
- 學生若提早在大二下修完線代 + 機率，也可在大二下提前修；大三後可進入實驗室做專題
- 必修帶來的好處：穩定的修課人數 → 學校提供更多 TA、助教、教材資源支持
- 國際對照：MIT、UIUC 等校都有 Introduction to Machine Learning / AI 類的導論課程，是國際上 AI 必修化的共同趨勢
- 對應台大「AI 學位學程」的策略方向——本門課是學位學程的前哨基礎

> "很多同學是提早在大二下的時候修，那你要延到晚一點那時候其實也沒有關係，大三是一個還蠻好的時機，有這個概念之後大家就可以再去修更進階的課程或是進入各種實驗室做專題。"

### Section 3｜課程大綱與教材（09:00 ~ 13:00）

**重點摘要：** 前半段陳上則、後半段陳縕儂；前段偏 Search/KR、後段偏 ML/Data-Driven。

**內容：**
- 課程分為前半段（陳上則）與後半段（陳縕儂）兩大部分
- 前半段涵蓋：傳統 AI、Tree Search、Knowledge Representation、Planning
- 後半段涵蓋：基礎 ML、Regression、Classification、Deep Learning 入門、Generative AI 主題
- 教材：前半段多為 Russell & Norvig《Artificial Intelligence: A Modern Approach》；後半段為 Goodfellow《Deep Learning》部分章節 + 線上補充資源
- 與國際課程 topic mapping：MIT 6.0001、Stanford CS221 等都有對應章節
- 鼓勵跨域外系同學修課：日文系、心理系等都可選修，但本系優先

> "從一開始 AI 的崛起，有 Search-Driven、Knowledge-Driven，然後到近期比較偏向 Data-Driven 的這個方向，然後去連接比較 Advanced 的 Topic。"

### Section 4｜評分方式、Final Project 與資源（13:00 ~ 17:48）

**重點摘要：** 4 次作業（30%）+ 期中考（30%）+ Project（30%）+ Bonus；Project 第三週就 release，是這學期最大改動。

**內容：**
- 評分：4 次作業（合計約 30-40%）+ 期中考（30%）+ Final Project（30%）+ Bonus / 課堂參與（彈性）
- 作業類型混合：手寫題 + Programming 題（Python 為主）
- **Final Project 提前**：今年改為第三週就 release，TA 會先講解專題設計；學生從第三週開始可以先寫一個簡單 bot，再隨著後半段 ML 知識疊加上去
- Project 主題：遊戲類 AI（會在 NTU COOL 上公布詳細 spec）
- 修課方式：三類加選 → 第一週登記 → 抽籤 / 授權碼 → 第二週正式加選（授權碼通常都會發）
- 上課地點：原 102 教室 → 改為 103（空間較大，必要時 102 同步直播）
- 課程網頁：NTU COOL，所有錄影、公告、教材都會放上去；建議把學校信箱 forward 到常用 mail
- 與 AI / 同學討論都必須在作業中註記；會做相似度檢查，但有註記就算合理使用
- 鼓勵課堂提問、給老師即時 feedback（太慢、太快、不清楚都可以說）
- TA Hour 與 Office Hour 公告在 NTU COOL，鼓勵外系同學先去 TA Hour 起步

> "這次比較不一樣的地方是我們 Project…把 Project 在第三週就 release…大家就可以比較早開始試玩一個簡單的遊戲…前面學到東西就可以先做一些 agent 來玩玩看，後面學到東西可以在疊加上去。"

## 四、人物 / 角色分析

### 陳縕儂教授（後半段授課）
- 背景：台大資訊工程學系教授，研究領域包含 Speech / NLP / Generative AI
- 課程角色：FAI 後半段（Machine Learning 主題）主授；期末 Project 設計者
- 代表觀點：「把 Project 提早 release」是為了避免過去學生期末被作業 + Project 夾殺的慘況

### 陳上則教授（前半段授課）
- 背景：台大資訊工程學系教授，傳統 AI / Search / Knowledge Representation 領域
- 課程角色：FAI 前半段（AI 基礎 + Search）主授；前 2 次作業設計者
- 代表觀點：奠基在 Russell & Norvig 的傳統 AI 框架，強調 Knowledge Representation 與 Planning 的重要性

- 背景：台大資訊工程學系教授，知名 ML 教科書《Learning from Data》作者（與 Yaser S. Abu-Mostafa 合著）

- 統計：超過一半的本系大學部學生都曾修過林軒田的 ML 課

## 五、核心主旨

> 2020 年台大資訊系正式將「人工智慧導論」納入大三下必修，是 1999 → 2009 → 2020 二十年課程改革脈絡的最新一環——回應校友問卷中對 AI 與 Design Pattern 的強烈需求。這門課以「前半 Search/KR、後半 ML」的雙段設計，搭配第三週即 release 的 AI 專題，目標是讓所有資訊系學生在大三結束前具備銜接進階 AI 課程與專題的基礎能力。

## 六、金句摘錄

1. "1999 年的課程改革是把網路、OOP 這些納入必修，因為那個時候是網路剛起來；2009 年因為 OOP 可能就不是那麼的必要…就把它移出必修變成是選修；而 AI 我們新納入。"
2. "這門課的設計目標是因為大家也知道 AI 這個議題不只是熱門，應該是已經是到不可或缺的…所以我們希望能夠用這門基礎的課程去銜接其他相關的專業課程。"
3. "過去系上的機器學習課程是林軒田老師開設的…他統計一下有修過機器學習課程的學生數量，如果考慮本系的學生，那其實超過一半以上的本系大學部學生其實都有修過機器學習的課程。"
4. "這次比較不一樣的地方是我們 Project…把 Project 在第三週就 release…大家就可以比較早開始試玩一個遊戲，前面學到東西就可以先做一些 agent 來玩玩看。"
5. "我們會希望你最後寫上去的東西是吸收過，然後變成你自己的東西再講出來…就算你現在是在 AI 的幫助之下打對了或寫出這個程式，但是吸收之後再寫出來，其實也可以變成你的知識。"
