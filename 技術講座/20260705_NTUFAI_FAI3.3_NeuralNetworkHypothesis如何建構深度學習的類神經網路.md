## 一、主題與背景

FAI 3.3「如何建構深度學習的類神經網路」正式建立深度學習的數學假設集合，回答的核心問題是：「一個神經網路是如何從簡單的權重加總，一步步堆疊成具有強大表示力的深度學習模型？」

陳縕儂教授從單一神經元開始拆解。最簡單的神經元（Perceptron）執行的運算，是把輸入向量 x 與權重向量 w 內積，再加上偏置項 b，最後送入一個 activation function。這個 activation function 可以是 sigmoid、tanh、ReLU 等任意可微的非線性函數。這個結構本質上和 FAI 2.x 的 Linear Model 完全相同——一個神經元就是一個 hyperplane，當輸出大於某個閾值時，判定為正例；反之為反例，因此單神經元只能處理二元分類。

當任務需要多元分類時，單神經元便不足夠。陳縕儂教授以手寫數字辨識為例：0 到 9 共十個類別，需要十個神經元各自判斷「輸入是否對應某個數字」，最後比較十個分數哪個最高，即為預測結果。這樣的「一層神經網路」其實就是一個 Multi-Class Perceptron，每個神經元彼此獨立、各自學習一個分類邊界。

但單層結構有明顯的表示力限制。著名的 XOR 問題就是一例：當資料呈現「交叉」分布時，無論如何旋轉一條直線都無法正確分開。陳縕儂教授以「And、Or、Xor」三種邏輯閘做對比：And 與 Or 都可被單一超平面切開，Xor 則需要組合兩條以上的邊界才能達成。解決方案是堆疊更多層次——第二層神經元以第一層的輸出當作輸入，重新組合出更複雜的決策邊界。這就是 Multi-Layer Perception 的核心想法。

接下來，影片回答「為什麼必須是非線性的 activation？」如果每一層後面沒有非線性函數，那麼多層線性函數的組合仍然等價於單一線性函數，多層疊加便完全失去意義。影片列出常用的 activation function：sigmoid 將輸出壓到零到一之間；tanh 壓到負一至正一之間，是 sigmoid 的等比例縮放版本；ReLU 則只保留正值、負值歸零，計算簡單且能避免梯度在高層次中逐步消失的問題，這也讓它在深度學習後期成為主流。

最後，影片處理「為什麼要『深』而不只是『寬』？」這個關鍵的工程問題。從 Universal Approximation Theorem 來看，只要夠寬，理論上一層的網路就能逼近任意函數；那為何業界的實作選擇不斷加深而非加寬？原因是，在參數量相同的條件下，深層網路可以表示更高階的複雜度——例如寬度為 2d 的兩層網路，其表示力等價於維度為 d 平方的單層網路；因此，深度在參數效率上明顯勝過寬度。這也是現代深度學習普遍走向深層架構的根本原因。

**核心議題一句話**：深度學習的表示力並非來自神經元單體，而是來自多層非線性 activation 與權重共享所形成的高維函數逼近能力；同時，「為何深而不寬」背後有可量化的參數效率理論基礎，這也是深度學習在 2010 年代之後能主導影像、語音、文字任務的核心技術解釋。

## 二、章節脈絡

### Section 1｜單一神經元 (Perceptron)（00:00 ~ 05:00）

**重點摘要：** 單一神經元 = 加權內積 + activation；本質等同於 FAI 2.x 的 Linear Model；單一神經元只能處理二元分類。

**內容：**
- Neural Network Hypothesis Set = Perceptron 模型
- 最簡單的一層的神經網路就是 Perceptron
- 跟 Linear Model 是一模一樣的
- 單一個神經元做的事情：輸入 x 跟權重 w 內積，再加偏置項
- 放入 activation function（可以是 sigmoid 或 tanh）
- 得到輸出 y
- 這個權重 w 就是神經元的參數
- 每一個 w 對應到輸入的一個維度
- 以數字辨識為例：把數位化影像每個像素當作一個維度
- 中間做一個 decision making：y > 0.5 就判定為某個數字
- 這就是一個非常簡單的神經元可以做的事情
- 但單一神經元只能處理二元分類（binary classification）

> "單一個神經元它做的事情就是把前面這邊的這個x做一個waited sum並且放入這個activation function之後得到最後的score，所以我們把它寫成式子就是長這樣。"

### Section 2｜多分類：多個神經元（05:00 ~ 09:00）

**重點摘要：** 多分類任務需要多個神經元；10 個數字 → 10 個神經元；各自判斷，最後比哪個分數最高即為預測。

**內容：**
- 任務可能不只二元分類
- 例如辨識數字是什麼音，10 種音都要發出來
- 多分類問題：辨識輸入到底是哪一個類別
- 把單一神經元衍生變成多個神經元
- 10 個神經元各自判斷「輸入是否對應某個數字」
- 第一個神經元判斷是不是 1，第二個判斷是不是 2，以此類推
- 根據 10 個不同分數決定哪個最有可能是對的數字
- 分數最高的那個就是預測結果
- 準備的最後一個維度就是最後的 Label 數量
- 從一個神經元衍生到一層的神經元去處理多個輸出的狀況
- 像這種只有一層的就叫 Perceptron

> "我們就必須要把一個神經元可以衍生變成是多個神經元，所以就非常簡單把剛剛只有一個神經元做的事情，比如說現在總共有10個不同的digit，那我們就有10個。"

### Section 3｜單層 Perceptron 的表示力限制（09:00 ~ 14:00）

**重點摘要：** 單層 Perceptron 的表示力不足；XOR 問題證明單層無法處理「交叉」分布；And、Or 可被超平面切開，Xor 則需要組合兩個邊界。

**內容：**
- 單層 Perceptron 有明顯的局限
- 每個神經元獨立學習分類邊界
- 各神經元之間沒有任何關聯
- 對於 And 的情況：白色跟黑色可以被一條直線切開
- 對於 Or 的情況：同樣可以被一條直線切開
- 但對於 Xor（交叉）的情況：黑白交叉分布，無法用一條直線切開
- 這代表單層 Perceptron 的表示力不足以處理 XOR
- 但 XOR 並不複雜：可以用多個 And 和 Or 組合起來
- 用 And 跟 Or 來實現 XOR：把 XOR 拆解成兩個邊界的組合
- 「簡單運算用多次用多層可以模擬複雜運算」
- 延伸到 Multi-Layer Perception

> "Perceptron它其實是有一個大limit的，就是它的表示力其實是不足的，還記得之前我們有示意過這個例子嗎...如果是xor的狀況，黑的在這，白的在這，其實我就沒有辦法非常容易的直接用一個這樣子分開。"

### Section 4｜Multi-Layer Perception（14:00 ~ 22:00）

**重點摘要：** Multi-Layer Perception = 多層 Perceptron；第一層處理基本邊界，第二層組合；越深的網路可以表示越複雜的邊界。

**內容：**
- 解決單層表示力不足的方法：讓低層結果再次組合
- 第二層神經元以第一層的輸出當作輸入
- 重新組合出更複雜的決策邊界
- 這就是 Multi-Layer Perception 的核心
- 一層 Perceptron 的表示力是有限的
- 第一層只能模擬斜的邊界（hyperplane）
- 兩層 Perceptron 可以把兩個斜的邊界組合起來
- 模擬出更奇怪的形狀，例如 RIDGE 形式
- 三層 Perceptron 可以組合更多 ridge 形式
- 可以形成棒狀（rod）的形狀
- 三層就算滿 general：可以包住任何位置的點
- Multi-Layer 可以加強模型的複雜度，也就是上升它的表示力
- 就可以去模糊更複雜的目標函數

> "Multi-Layer Perception 的表示力就會比較強，你可以把它想成剛剛不是單獨一層的話，就是剛剛那個斜坡就是有點像是一個翹翹板的那種斜坡，但是如果是兩個兩層的話其實斜坡跟斜坡可以合在一起。"

### Section 5｜Activation Function 的角色（22:00 ~ 28:00）

**重點摘要：** 如果沒有非線性 activation，多層 Linear 等於單層 Linear；非線性激活讓多層疊加有意義；常用 sigmoid、tanh、ReLU。

**內容：**
- 為什麼需要非線性的 activation function？
- 假設每個神經元都是 Linear function
- 如果中間沒有非線性 activation function
- 多層線性函數的組合仍然等價於單一線性函數
- 加了非線性才能讓每一層可以做到複雜的組合
- Discretion 這種離散函數其實不太好 optimize
- 因為在切換位置無法微分
- Sigmoid：把輸出壓到 0-1 之間
- Tanh：把輸出壓到 -1 到 1 之間，是 sigmoid 的等比例縮放版本
- ReLU：只保留正值，負值歸零
- 為什麼 ReLU 在深度學習後期成為主流？
- 因為不像 sigmoid 或 tanh 都會把輸出壓到有限範圍
- 多層之後輸出會一直壓縮、累積資訊可能會消失
- ReLU 只保留正值、避免資訊壓縮問題
- 計算簡單：ReLU 微分要不是 1 就是 0
- 實務上 deep learning 廣泛使用 ReLU

> "如果我們沒有非線性的這個function的話，你就算疊了很多層其實就跟一層的linear transformation是一樣的。"

### Section 6｜為什麼「深」而不「寬」（28:00 ~ 35:00）

**重點摘要：** Universal Approximation Theorem：寬度夠就能逼近任意函數；但同參數量下，深層網路表示力更強；深層比寬層在參數效率上明顯勝出。

**內容：**
- Deep Learning 的層數多，參數量也多
- 常說這種可能是淺的（shallow）就一層這種
- Deep model 是多層的
- Universal Approximation Theorem：任何函數可以用一層 Hidden Layer 的網路來 realize
- 只要它夠大、夠寬就夠了
- 那為什麼業界不是長胖而是長深？
- 在參數量相同的情況下：
  - 寬度為 2d 的單層網路：表示力是 O(2d)
  - 寬度為 d 的兩層網路：表示力是 O(d²)
- 同樣 2d 個參數，深層可達 d² 的表示力
- 因此深層在參數效率上明顯勝過寬層
- 實務上 Deep model 通常用比較少的參數就能達到比較好的 Performance
- 這是深度學習選擇深而非寬的理論基礎

> "Universal Approximation Theorem它的這個理論是在說，其實呢任何的這個Network你可以用一層Hidden Layer的這種來realize，就是說其實一層就夠了只要它夠大夠胖...可是為什麼會是Deep Model而不是Fat Model呢？"

## 五、核心主旨

> 單一神經元的運算等同於 Linear Model，是加權內積加上 activation function 的結果。當任務升級為多分類時，需要多個神經元各自判斷，最後比較分數高低決定預測。單層神經網路在表示力上有明顯侷限——XOR 問題證明交叉分布的資料無法被單層線性區分，必須透過多層結構來組合多個邊界，這正是 Multi-Layer Perception 的核心想法。Activation Function 的存在至關重要：如果沒有非線性函數，多層 Linear 疊加仍等價於單層 Linear；ReLU、Tanh、Sigmoid 等非線性函數的選擇直接決定了多層網路能否學到有用的特徵表示。Universal Approximation Theorem 雖然告訴我們寬度夠就能逼近任意函數，但在「同參數量」的條件下，深層網路（d²）所能表示的複雜度明顯高於寬層網路（2d）。因此，現代深度學習普遍走向「深而非寬」的架構設計，這也是 2010 年代之後深度學習能主導影像、語音、文字任務的核心技術解釋。
