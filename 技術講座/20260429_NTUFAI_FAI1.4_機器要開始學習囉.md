## 一、主題與背景

FAI 1.4「機器要開始學習囉~」是林軒田教授 Machine Learning 章節的第四講，約 13.5 分鐘。本講承接 FAI 1.3 建立的 ML Recipe 框架，正式進入 Recipe 步驟 4「Learning」——也就是用什麼 algorithm 從 H 中找出最接近 $f$ 的 $g$。

林軒田教授用 PLA（Perceptron Learning Algorithm）作為切入點。PLA 是 1950-60 年代 Rosenblatt 提出的第一個 Learning Algorithm，靈感來自人類神經元的運作：在 2D 平面上隨機選一條直線開始，看哪些點分錯就調整直線方向——具體做法是「把法向量加上錯誤點的向量」讓直線往錯誤的方向轉。影片用 Animation Demo 展示 PLA 一步步收斂的過程：每次錯就微調，慢慢把所有點分對。

本講的另一條主軸是 **Generalization 的複雜性**。林軒田教授用「父母 vs 小孩」、「老師 vs 學生」兩個類比說明：小孩背考古題很厲害不代表 Generalization 強——因為看不到真正的 $f$，只能看到 noisy 的 training data。Noise 來自標籤錯誤、特徵不足等多重原因。我們只能期待 $g$ 接近 $f$，但無法確切驗證。這也是為什麼 Generalization 是一個「非常複雜」的問題——$E_{in}$ 小不代表 $E_{out}$ 小。最後影片以 1969 年 Minsky & Papert 的「Perceptrons」論文引爆的第一次 AI 寒冬作結，預示 Linear Model 的限制將引發典範轉移。

**核心議題一句話**：從無限多候選 hyperplane 中暴力搜尋不切實際——必須用聰明的 Learning Algorithm（如 PLA）逐步收斂；但 $E_{in}$ 小不代表 $E_{out}$ 小，Generalization 是 ML 最核心也最複雜的問題。

## 二、章節脈絡

### Section 1｜Learning 的本質：從無限多 Hyperplane 中搜尋（00:00 ~ 03:00）

**重點摘要：** 2D 平面直線無限多；不能暴力搜尋；需要聰明的 Learning Algorithm。

**內容：**
- Recipe 步驟 4：執行 Learning
- 目標：從 H 中找出 $g$ 接近 $f$，minimize $E_{in}$
- 問題：2D 平面的直線無限多，不可能暴力列舉所有直線找 $E_{in}$ 最小的
- 需要聰明的 Learning Algorithm
- 第一個候選：PLA（Perceptron Learning Algorithm）——模擬人類神經元

> "如果考慮所有可能的 $g$、所有可能的這些 Hypotheses，其實你是有無限多個的。在這個 2D 平面中所有的直線的數量是不是無限多？所以你好像不太可能暴力法把所有直線都列出來看看哪一個 Error 最小。"

### Section 2｜PLA Algorithm：法向量 + 錯誤點向量（03:00 ~ 08:00）

**重點摘要：** 從任一直線開始，看錯的點，把法向量加上錯誤點向量微調。

**內容：**
- PLA 流程：
  1. 從任一條直線（任意法向量）開始
  2. 看現在直線上有哪些點分錯
  3. 根據分錯的點調整直線位置
- 數學操作：法向量 + 錯誤點的向量 → 新的法向量
- 直觀：法向量決定直線方向，加上錯誤點向量讓直線往錯誤方向轉
- Animation Demo：藍點與紅點分佈 → 隨意選一條起始線 → 錯了就微調 → 慢慢收斂
- 一次可能轉太多而產生新錯誤，所以要反覆微調
- PLA 靈感來自神經元（Perceptron），簡化版的 Logistic Reasoning

> "這個方法呢是這樣子的，就根據這一個切線它有一個法向量，法向量就跟它垂直的…這個點呢到這個錯誤的這個點的這條向量，這兩個向量加在一起是在這邊，這邊呢就是它認為應該要更新的新的法向量。"

### Section 3｜1969 Perceptrons 論文：第一次 AI 寒冬（08:00 ~ 09:30）

**重點摘要：** Minsky & Papert 證明 Perceptron 表示力不足（XOR 不能分），引發第一次 AI 寒冬（1980 年代）。

**內容：**
- PLA 雖然是 1950-60 年代的突破，但 1969 年 Minsky & Papert 的「Perceptrons」論文證明它的表示力嚴重不足
- 簡單的 Perceptron 無法表示 XOR 等非線性函數
- 這個限制讓 AI 界對神經網路失去信心，1980 年代進入第一次 AI 寒冬
- 寒冬期間研究資金大幅縮減，但這也促使研究者尋找新的方法（最終促成 1980-90 年代的 backpropagation、1990 年代的 SVM 等突破）
- 為什麼這個歷史重要：提醒學生「模型限制」是 ML 永遠要考慮的問題

> "1969 年的這篇 paper 顯示它很多裡面的，之後呢後來就迎來了第一次的這個寒冬 1980 年代。這就是第一次寒冬，在這邊大家可能想說這個 Logistic Reasoning 的方法好像還不錯…就發現它其實沒有想像中那麼厲害。"

### Section 4｜Generalization 的複雜性：父母 vs 小孩、老師 vs 學生（09:30 ~ 13:34）

**重點摘要：** $E_{in}$ 小不代表 $E_{out}$ 小；小孩背考古題不代表理解概念；Generalization 是 ML 最複雜的問題。

**內容：**
- 回想：Training Data 是 noisy 的、有限的；看不到真正的 $f$
- 我們只能期待 $g$ 接近 $f$，但無法確切驗證
- 類比 1：父母 vs 小孩——父母（$f$）知道很多但只能示範部分；小孩（$g$）學到部分但不是全部
- 類比 2：老師 vs 學生——學生把所有考古題背完可能考試 100 分，但無法驗證是否真的理解
- 學生目標：在 seen data 拿滿分（$E_{in}$ 小）；老師目標：學生真正理解（$E_{out}$ 小）
- 兩者目標不一致，導致「背高分但不懂」的情況
- 簡化定義：$E_{in}$ 小通常代表有一定的 Generalization 能力
- 但 Hypothesis 太複雜會 overfit；太簡單會 underfit——bias-variance tradeoff
- **簡單模型通常 Generalization 較好**：如果 Linear Model 在 training 上不錯，就用 Linear，不要升級到複雜模型

> "Generalization 這件事情，其實是一個非常複雜的一個問題。那在 Machine Learning 的各種任務上面或者在真實情況上面，常常出現因為你 Memorized 不一定代表你 Generalized 好嘛。"

## 三、關鍵概念定義表

| 概念 | 定義 | 出處 / 應用 |
|------|------|-------------|
| PLA | Perceptron Learning Algorithm，1950-60 年代第一個 Learning Algorithm | Section 1, 2 |
| 法向量 | 與直線垂直的向量，決定直線方向 | Section 2 |
| Perceptron | 模擬神經元的單層 Linear Model | Section 2 |
| Multi-Layer Perceptron | 多層 Perception 疊起來，可學 XOR 等複雜函數 | FAI 1.3 已介紹 |
| XOR | 互斥或運算，單層 Perceptron 無法表示 | Section 3 |
| AI 寒冬 | AI 研究因期望落空而資金縮減的時期（1980s、1990s） | Section 3 |
| Minsky & Papert | 1969《Perceptrons》論文作者，證明 Perceptron 限制 | Section 3 |
| Generalization | 模型在 unseen data 上的表現 | Section 4 |
| Overfitting | $E_{in}$ 小但 $E_{out}$ 大（過度擬合） | Section 4 |
| Underfitting | $E_{in}$ 本身就大（模型太簡單） | Section 4 |
| Bias-Variance Tradeoff | 簡單模型 high bias、複雜模型 high variance 的取捨 | Section 4 |
| Memorization | 背下訓練資料（與 Generalization 對比） | Section 4 |
| E_in | In-Sample Error，training data 上的 error | Section 4 |
| E_out | Out-of-Sample Error，unseen test data 上的 error | Section 4 |

## 五、核心主旨

> 從無限多候選 hyperplane 中暴力搜尋不切實際——必須用聰明的 Learning Algorithm 如 PLA：從任一條直線開始，看哪些點分錯就把法向量加上錯誤點向量微調，逐步收斂。但 $E_{in}$ 小不代表 $E_{out}$ 小：小孩背考古題很厲害不代表理解概念——這是 Generalization 的複雜性。1969 年 Minsky & Papert 證明 Perceptron 無法表示 XOR，引發第一次 AI 寒冬，提醒我們「模型限制」是 ML 永遠要考慮的問題。簡單模型通常 Generalization 較好——bias-variance tradeoff 是 ML 的核心張力。

## 六、金句摘錄

1. "在這個 2D 平面中所有的直線的數量是不是無限多，所以不太可能暴力法把所有直線都列出來看看哪一個 Error 最小，那我就選擇這一個直線好像不太 Make Sense。所以有沒有一些比較聰明的演算法呢？"
2. "1969 年的這篇 paper 顯示它很多裡面的，之後呢後來就迎來了第一次的這個寒冬 1980 年代。這就是第一次寒冬，因為就發現它其實沒有想像中那麼厲害。"
3. "Generalization 這件事情，其實是一個非常複雜的一個問題。那在 Machine Learning 的各種任務上面或者在真實情況上面，常常出現因為你 Memorized 不一定代表你 Generalized 好。"
4. "學生對於他覺得他 Perfect，對於老師來講不一定是好的，因為老師可能希望你真的學會，但是學生他可能只是想說在這一個考試上面拿滿分。"
5. "如果你的 Hypothesis 就是你的 Model 選擇相當之下如果是比較簡單的話，比如說像我們今天講到的 Linear Model 就是我們是線這樣子，在 2D 平面的話是線，Generalization 這件事情是比較可能的。"
