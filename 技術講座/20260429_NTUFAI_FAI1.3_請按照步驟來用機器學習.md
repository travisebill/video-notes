## 一、主題與背景

FAI 1.3「請按照步驟來用機器學習」是林軒田教授 Machine Learning 章節的第三講，約 25 分鐘。本講的核心任務是把前兩講（ML 的本質定義、應用情境）整理成一套可操作的標準化 Recipe。林軒田教授用「申請工作」這個生活化例子貫穿全講——履歷（features）→ 公司好壞（label）→ 找出 pattern → 用 ML 學習。

影片最後整理出 **ML Recipe 五大步驟**：(0) 判斷 ML 是否適合、(1) 收集資料、(2) 選擇 Hypotheses Set、(3) 設計 Error Measure、(4) 執行 Learning、(5) 評估 Generalization。這套 Recipe 是後續 FAI 1.4-1.6 所有具體模型（Linear、Logistic）的標準操作框架。

本講的另一條主軸是 **Hypotheses Set 的選擇 = Model 選擇**。林軒田教授明確指出：Linear Model 的 Hypotheses Set 就是所有 hyperplane 的集合 $\mathbf{w}^T \mathbf{x}$；選擇 Linear Model 就是把候選函數限制在「直線」這個集合中。如果任務比 Linear 複雜（例如 XOR），單一 Perception 就不夠，需要 Multi-Layer Perception——這也是後續 Deep Learning 的理論基礎。

**核心議題一句話**：ML 是一套有 Recipe 的工程實踐——給定任務，先判斷 ML 適不適合，再依序走過 Data → Hypotheses → Error Measure → Learning → Generalization 五個步驟，每一步都必須有明確的設計決策。

## 二、章節脈絡

### Section 1｜申請工作範例：履歷 → 適合 / 不適合（00:00 ~ 04:00）

**重點摘要：** 用「申請工作」具象化 ML 的 Input / Output / Feature / Label 概念。

**內容：**
- 任務：給定申請者的履歷，判斷他是否適合公司
- Features：年紀、性別、使用的 OS、實習次數、project、主題、GPA 等
- 6 維空間中的一個點 = 一個申請者
- Input: $\mathbf{x}$ = 履歷 feature vector；Output: $y$ = 適合 / 不適合 (binary)
- 目標：找到一個 function $f$ 把 $\mathbf{x}$ 對應到 $y$（假設存在這樣的 pattern）
- 從過去 hiring 紀錄收集 training data：申請者履歷 + 進公司後表現好 / 壞
- Hypotheses Set $H$ = 所有可能的 $f$ 候選（無限多）

> "我們的目標呢就是希望能夠有一個機器呢，它可以告訴我說基本這一個申請者的這些 feature 的資料，我能不能夠判斷說這個申請者是不是適合現在這間公司。"

### Section 2｜ML Recipe 步驟 0：判斷 ML 是否適合（04:00 ~ 08:00）

**重點摘要：** 三個前提：(1) 有 Pattern、(2) Pattern 難以用 Rule 寫、(3) 有 Data。

**內容：**
- Recipe 步驟 0（最重要）：判斷這個任務適不適合用 ML
- 三個前提必須都滿足：
  1. 有 Pattern 可學（如果 X 跟 Y 完全 random，就不該用 ML）
  2. Pattern 無法（或很難）用 Rule 寫（例如餐廳評價分類）
  3. 有 Data（沒有 Data ML 就無用武之地）
- 反例：努力 vs 中樂透——沒有 pattern，不該用 ML
- 反例：if Internship >= 2 就 accept——可以直接用 Rule，不需要 ML

> "我們要進入我們訓練你之前，你都一定要先想過說這個任務到底適不適合用機器學習。"

### Section 3｜ML Recipe 步驟 1：收集 Data（08:00 ~ 10:00）

**重點摘要：** Data 是 (x, y) 的配對；資料量決定模型上限。

**內容：**
- 收集過去 hiring 的歷史資料：申請者履歷（x）+ 進公司後表現好 / 壞（y）
- 這些 (x, y) 配對就是 Supervised Learning 的 Training Data
- 重要：Data 的 quality 與 quantity 決定 ML 的上限
- 真實範例：2010 KDD Cup——學生特徵 + 考試題目特徵 → 預測學生能不能答對題目
- 當年台大團隊拿世界冠軍
- 訓練完的 model 會在 unseen test data 上評估 generalization

> "第二步呢就是要決定 Hypothesis Set。"

### Section 4｜ML Recipe 步驟 2：選擇 Hypotheses Set（10:00 ~ 16:00）

**重點摘要：** Hypotheses Set = Model 選擇；Linear Model 的 H 是所有 hyperplane。

**內容：**
- Hypothesis Set $H$ = 所有可能的候選函數集合
- 範例 H1：只要 GPA > 4.0 就 accept（單一 feature 切分）
- 範例 H2：只要有 NLP 相關 project 就 accept
- 範例 H3：H1 AND H2 / H1 OR H2 / …
- 現實中 H 非常大，幾乎無限多可能
- 數學表示：H = {所有 $\mathbf{w}^T \mathbf{x} > \text{threshold}$ 的 hyperplane 集合}（Linear Model）
- Linear Model 的限制：只能用直線 / hyperplane 切——簡單但有侷限性（XOR 不能分）
- 多層 Perception (Multi-Layer Perceptron) 可以突破：每一層都是 Linear，多層疊起來可以表示 XOR
- 這也是 Deep Learning 的理論基礎：很多層簡單的 Linear 疊起來可以學到非常複雜的 function

> "一個 Linear Model 就是只要是任何一個 $\mathbf{w}$ 的值，其實它都會是屬於這個 Linear 的這個 Hypothesis Set…所以這個就是根據你要考慮多少種 Hypotheses 所定義出來的，跟你選的那個 Model 是有關的。"

### Section 5｜ML Recipe 步驟 3：選擇 Error Measure（16:00 ~ 20:00）

**重點摘要：** Error Measure 量化「$g$ 距離 $f$ 多遠」；常見有 0/1 Error 與 Squared Error。

**內容：**
- 目標：找 $g$ 接近真實的 $f$
- Error Measure 量化 $g$ 跟 $f$ 的差距
- Pointwise Error：每個點算一個 error，加起來平均
- 兩種常見：
  - In-Sample Error $E_{in}(g)$：在 training data 上的 error
  - Out-of-Sample Error $E_{out}(g)$：在 unseen data 上的 error
- 我們能 minimize 的是 $E_{in}$（因為看不到 $E_{out}$）
- 類比：用前 2 年考古題讀書，用第 3 年考古題測試 generalization
- 分類任務：0/1 Error（錯 1 個算 1 個）
- 回歸任務：Squared Error（差距的平方）
- Logistic：Cross-Entropy

> "在 Training Data 裡面，我們會叫它 In-Sample…Aro 就是預測出來的值跟我們真實收集到的這個 Y 之間的差異。"

### Section 6｜ML Recipe 步驟 4-5：執行 Learning + 評估 Generalization（20:00 ~ 24:52）

**重點摘要：** Learning = 用 algorithm 從 H 中找 $g$ minimize $E_{in}$；Generalization 才是最終目標。

**內容：**
- Training：給定 $D$、$H$、Error Measure → algorithm → $g$
- Testing：給 unseen $D'$ → 用 $g$ 預測 → 算 $E_{out}$
- 目標：$g$ 在 unseen data 上也表現好 = Generalization
- Training 的目標是 minimize $E_{in}$，但最終希望 $E_{in}$ 跟 $E_{out}$ 都小
- Recap：ML Recipe 5 步驟——(0) 判斷 → (1) Data → (2) Hypotheses → (3) Error → (4) Learning → (5) Test
- Linear Model 的 Hypothesis Set：所有 hyperplane
- Linear Classification：sign function 切兩邊
- 2D 平面是一條線；高維是 hyperplane
- Linear Model 在高維空間的限制：某些 distribution 切不開（XOR）

> "Training 的過程呢，就是要 Given 已經觀察到的一塔，去找最好的方式；Testing 呢，就是用這個已經訓練好的方式，然後來預測我們現在新的這些 Label。"

## 五、核心主旨

> Machine Learning 是一套有 Recipe 的工程實踐——不是寫個 model.fit() 就結束的魔術，而是 (0) 判斷 ML 是否適合、(1) 收集 Data、(2) 選擇 Hypotheses Set、(3) 設計 Error Measure、(4) 執行 Learning、(5) 評估 Generalization 的完整流程。Hypotheses Set 的選擇 = Model 的選擇，決定你能學到什麼樣的 function；Error Measure 的選擇決定你「學得好不好」的量化標準；最終目標不是 $E_{in}$ 最小化，而是 $E_{out}$ 最小化——也就是 Generalization。
