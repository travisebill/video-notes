## 一、主題與背景

本課程為李宏毅教授 2026 年「機器學習導論」中關於 AI 自我修正（Self-Correction）能力的系統性整理。從三個技術維度——**Decoding（解碼過程）**、**Harness（工作流程）**、**Reasoning（推理訓練）**——探討語言模型是否能在沒有人類介入的情況下，自己發現錯誤並修正行為。課程也涵蓋了 RL（強化學習）在推理能力訓練中的爭論：到底是「激發既有能力」還是「學到新能力」。

## 二、章節脈絡

### Section 1｜開場：AI 能自我修正嗎？（00:00 ~ 05:00）

**重點摘要：** 語言模型可以在人類給回饋後修正錯誤，但真正的 Self-Correction 是在沒有人類介入的情況下自己發現並修正錯誤。這堂課從三個方向探討這個問題。

**內容：**
- 已知：LLM 可以根據人類回饋修正錯誤（「你做錯了，錯在哪裡」→ 模型更正）
- 問題：能不能**沒人介入**，模型輸出後自己發現錯、自己修正？
- 三個技術方向：
  1. 修改 Inference / Decoding 過程
  2. 修改 Harness / 工作流程
  3. 從模型參數著手（Reasoning / RL 訓練）

> 「批判比生成容易。我不會寫小說，但我還是可以知道一部小說好不好看。」

### Section 2｜Decoding 層面的自我修正（05:00 ~ 20:00）

**重點摘要：** 在生成過程中直接偵測和修正錯誤——從 token 機率分布和內部表徵中預測模型何時可能犯錯。

**內容：**

**方法一：從機率分布預測錯誤**
- 觀察生成時每個 token 的機率分布
- 若模型對某 token 信心很低（entropy 高、機率分散）→ 可能是錯誤點
- 可在這些位置觸發修正機制（重新 sample、beam search 擴展等）

**方法二：從內部表徵預測錯誤**
- Transformer 的 hidden states 中可能編碼了「模型是否在胡說」的訊號
- 可訓練一個分類器，輸入 hidden states → 預測該 token 是否可能錯誤
- 這是 Inference-Time Intervention 的核心思路

**限制：**
- 機率信心低 ≠ 一定錯；信心高 ≠ 一定對（overconfidence 問題）
- Hidden state 分類器本身需要訓練 → 需要標註「錯在哪」的資料

### Section 3｜Harness 層面的自我修正：Self-Refine（20:00 ~ 38:00）

**重點摘要：** 透過修改模型的工作流程來實現自我修正——最簡單的方式就是在輸出後插入「再檢查一下」的 prompt。

**內容：**

**Self-Refine 基本架構：**
1. 模型收到任務 → 產生第一版答案
2. 自動插入反思指令（如「再檢查一下」、「請反思你的答案」）
3. 模型檢查自己的答案 → 發現問題 → 修正
4. 重複步驟 2-3 直到滿意

**為什麼反思可能有用？**
- **批判比生成容易**：模型不一定能生成正確答案，但可能能辨識錯誤
- **生成無法回頭**：Auto-Regressive 生成第一步錯就一路錯到底。反思給了「回頭」的機會
- **文字接龍的本質**：「再檢查一下」在訓練資料中後面常接著修正內容

**關鍵發現：**
- 反思指令是通用泛化的（不需要針對特定任務），因此可由程式自動插入，不算人類介入
- 模型不一定能在一次反思內修正，可能需要多輪
- 反思的效果高度依賴模型基礎能力——太弱的模型連「檢查」都做不好

**實驗發現：**
- 自我反思確實能提升某些任務的準確率
- 但也有反效果的時候：模型可能「反思後反而改錯」——把對的答案改成錯的
- 這說明了 Self-Correction 不等於「總是變好」

### Section 4｜Reasoning 層面的自我修正（38:00 ~ 55:00）

**重點摘要：** Reasoning（推理）本身就是一種自我修正的形式——Chain-of-Thought 中每一步都在檢查和調整前面的推論。RL 訓練如何改變模型的推理能力是當前最大的爭論點。

**內容：**

**Reasoning 與 Self-Correction 的關係：**
- Chain-of-Thought（CoT）：模型邊思考邊修正自己的思路
- 「等一下，我剛才的說法可能有問題，讓我重新想一遍」→ 這就是在 CoT 中發生的自我修正
- **Reasoning 本質上就是結構化的 Self-Correction**

**RL 訓練與 Reasoning 的爭論：**

| 立場 | 論點 |
|------|------|
| **RL 只是激發既有能力** | Base model 在大量 sample 下也能答對；RL 只是提高正確路徑的機率 |
| **RL 教了模型新能力** | Base model 的「正確」往往是猜對的，RL 讓模型學會真正正確的推理過程 |

**關鍵實驗（COTPath）：**
- 不能只看答案對不對，還要檢查「計算過程」是否正確
- 加上計算過程檢查後：沒做 RL 的模型大幅落後於有做 RL 的模型
- 但檢查計算過程的「評審」又是另一個語言模型 → 能算數嗎？

> 「有人覺得 LLM 在 RL 之前就有 reasoning 能力，有人覺得 RL 之後才學會——也有人覺得兩者都有。」

**Paper 結論（Debate on RL vs Reasoning Capability Boundary）：**
- **訓練初期**：RL 主要利用既有能力，改變機率分布
- **訓練後期**：RL 訓練夠久，模型確實可能展現新能力
- 核心問題轉向：什麼樣的 RL 演算法 / reward signal 最能激發新能力？

### Section 5｜Self-Correction 的限制與未來方向（55:00 ~ 75:00）

**重點摘要：** 自我修正不是萬靈丹——Context Window 耗盡、模型自我欺騙、Reward Hacking 等都是尚未解決的問題。

**內容：**

**主要限制：**
- **Context Window 瓶頸**：每輪反思都佔上下文 → 88 分鐘反覆修正很快就滿 → 模型開始遺忘早期內容
- **不知道自己不知道**：模型對某些領域完全沒有正確知識時，反思也無法產生正確答案
- **反覆修正後的退化**：有時答案被反覆修改後偏離正確方向（「過度修正」）
- **模型可能「作弊」式的自我修正**：把明顯錯誤改成不明顯錯誤 → 評分者（另一個 LLM）看不出來

**對比三層方法的取捨：**

| 層面 | 優點 | 缺點 |
|------|------|------|
| Decoding | 不需要額外 prompt，即時 | 偵測不可靠，過度/不足信心問題 |
| Harness | 最簡單實作，通用性高 | 佔 Context，可能改錯 |
| Reasoning + RL | 最深層的改變，效果最佳 | 訓練成本高，爭論還沒定論 |

### Section 6｜最終問題：Self-Correction 夠好了嗎？（75:00 ~ 結束）

**重點摘要：** 2026 年的 Self-Correction 技術已取得相當進展，但距離「完全不需要人類檢查」還有一段距離。

**內容：**
- 自我修正在結構化任務（數學、程式碼）上效果最好——有客觀對錯標準
- 開放式任務（寫作、創意）上自我修正的邊界模糊——什麼叫「更好」難以定義
- 多數前沿模型（GPT-5、Claude 4.5）已內建多層 Self-Correction 機制
- 未來方向：減少對 Context Window 的依賴、外部驗證機制的整合、更智慧的「何時該停止修正」的判斷

> 「今天模型自我修正的能力已經很不錯了——但還不夠好到我們可以完全不看它的輸出。」

## 三、關鍵概念定義

| 概念 | 定義 |
|------|------|
| **Self-Correction** | 語言模型在無人類介入下，自行發現並修正錯誤的能力 |
| **Inference-Time Intervention** | 在解碼過程中透過內部表徵預測錯誤並干預生成 |
| **Self-Refine** | 透過插入反思 prompt 讓模型重複檢查和修正自己的輸出 |
| **CoT（Chain-of-Thought）** | 讓模型逐步展示推理過程，每一步都可能修正前一步 |
| **RL for Reasoning** | 用強化學習訓練模型更精確的推理能力（vs 單純提高正確機率） |
| **COTPath** | 不只檢查答案對錯，也檢查推理計算過程是否正確的評估方法 |
| **Pass@k** | Sample k 次中至少一次答對的機率（衡量「運氣成分」） |
| **Overcorrection** | 過度修正導致把正確答案改成錯誤答案的現象 |

## 五、核心主旨

> AI 的自我修正能力可以從三個層面來加強——解碼過程的即時偵測、工作流程的反思循環、以及 RL 訓練下的深度推理。2026 年這三條路線都有顯著進展，但自我修正仍非萬靈丹：Context Window 限制、過度修正、以及「模型不知道自己不知道」等根本問題，意味著我們還不能完全不看 AI 的輸出。
