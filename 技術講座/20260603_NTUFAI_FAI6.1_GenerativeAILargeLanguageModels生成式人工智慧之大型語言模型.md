## 一、主題與背景

FAI 6.1「生成式人工智慧與大型語言模型」是林軒田教授在 Generative AI 章節的開場影片，從 generative vs discriminative 的本質差異談起，串接 specialist 走向 generalist 的典範轉移，並以 GPT-3 與 InstructGPT 為案例，說明大型語言模型如何從「讀書」走到「聽懂人話」的完整技術路徑。

節目一開始先把 generative AI 與過去判別式模型的本質差異講清楚。判別式模型回答 0 還是 1、屬於哪一類，generative AI 則從零產出文字、圖片、整段文章——這種「主動生成」相較於「被動判斷」，難度有數量級的差異。訓練方式也截然不同：過去每一個任務都得收集對應的輸入與標註資料、訓練專屬模型，一個模型只能做一件事，稱為 specialist；現在的大型語言模型則打破了這種限制。

核心突破是從 specialist 走向 generalist——同一個模型可以摘要、翻譯、解釋、寫程式、回答問題，什麼任務都能處理。林軒田用 GPT-3 在 2020 年的實驗展示這項轉折：橫軸為模型尺寸，縱軸為人類判斷「這段文字是人類還是機器寫的」的準確率。模型越大，人類判斷率從 70-80% 一路掉到 50%——也就是跟亂猜差不多，代表模型生成文字的能力已逼近真人水準。

GPT-3 的關鍵機制是 in-context learning。使用者只需提供 prompt 或幾個範例，模型就能根據上下文推斷當前任務，不必重新訓練參數。這讓同一個模型可以處理各式各樣的任務。然而讀完書的 GPT-3 不見得聽懂人類指令——給定輸入後，模型可能只是照樣照句繼續寫，並未真正理解使用者的意圖。

2022 年 InstructGPT 論文提出三階段對齊解法。第一階段是 SFT（Supervised Fine-Tuning），用人類標註的「指令—理想回答」配對訓練模型聽懂指令。第二階段是訓練 reward model——讓人類對同一個問題的多個模型輸出排序評分，AI 老師學會判斷好壞。第三階段使用強化學習 PPO，讓模型根據 reward model 的評分調整策略，朝高評分方向優化。

這三階段流程同時解決 hallucination 與安全性問題——模型會學會什麼內容該講、什麼內容不該講，也會對歧視性、危險性指令提高警覺。這套 SFT 加 RLHF 的對齊機制，正是後續 ChatGPT 商業化成功的技術基礎。

**核心議題一句話**：生成式大型語言模型的本質突破在於從 specialist 走向 generalist——同一個模型可處理各式任務，配合 in-context learning 與 InstructGPT 的 SFT + RLHF 三階段對齊，奠定了現代 ChatGPT 商業化的完整技術基礎。

## 二、章節脈絡

### Section 1｜從判別式到生成式（00:00 ~ 05:30）

**重點摘要：** generative AI 與過去 discriminative 模型的本質差異在於輸出形式——前者從零產出文字、圖片、文章，後者回答分類問題；訓練上過去每個任務都需要 specialist 模型，現在 generalist 模型可同時處理多種任務。

**內容：**
- 判別式模型：回答 0/1、屬於哪一類
- 生成式模型：直接從零產出文字、圖片、文章
- 過去訓練模式：先準備資料、決定任務、收集對應的 (X, Y) 對、訓練 specialist 模型
- specialist 限制：只能在單一任務做到 master 等級，換任務就失效
- generalist 模型（GPT-3 起）：同一個模型可同時做摘要、翻譯、解釋

> "生成是人工智慧最大的我覺得最大的不同其實是我們漸漸從每一個任務需要有一個專材開始，然後可能會目的到這種通彩，就像 lest 就是它可能對於很多任務都非常擅長。"

### Section 2｜GPT-3 文字生成實驗與 in-context learning（05:30 ~ 13:00）

**重點摘要：** GPT-3（2020 年）展示 generalist 模型的可行性——人類判斷機器寫作 vs 真人寫作的準確率隨模型尺寸增大而逼近 50%（亂猜水準）；in-context learning 是核心機制，使用者提供 prompt 或 few-shot examples 即可讓模型推斷任務。

**內容：**
- GPT-3 於 2020 年推出，是 OpenAI 第三代 GPT
- 文字生成實驗：模型尺寸越大，人類判斷率從 70-80% 降到 50%
  - 50% 代表亂猜水準，模型產出文字已逼近真人
- in-context learning（提示學習）：
  - 不重新訓練模型參數
  - 純粹根據 prompt 或 few-shot examples 推斷當前任務
  - 類似考試時看題型說明 + 範例 → 解新題目
- few-shot 不一定比 one-shot 好：有時一個例子就夠，給太多反而干擾

> "它測試說這樣尺寸的模型它要深圳出來的資料跟一個可能是真人寫的資料，然後隨機的給一個真人看，然後這個真人你覺得這是機器寫的還是真人寫的。"

### Section 3｜InstructGPT 三階段對齊（SFT + Reward Model + PPO）（13:00 ~ 25:02）

**重點摘要：** 2022 年 InstructGPT 解決 GPT-3 聽不懂指令的問題——三階段流程為 SFT（指令—理想回答配對）→ Reward Model（人類排序評分訓練 AI 老師）→ PPO 強化學習（學生根據老師評分調整策略）；同時解決 hallucination 與安全性。

**內容：**
- GPT-3 問題：能續寫但不聽指令（照樣照句而非真正理解意圖）
- 第一階段 SFT（Supervised Fine-Tuning）：
  - 收集人類指令 + 人類理想回答的配對資料
  - 用這些資料 fine-tune 原本讀完書的 GPT-3
  - 資料多樣性很重要：避免變成只會做單一任務的 specialist
- 第二階段 Reward Model：
  - 人類對同一問題的多個模型輸出排序（哪個比較好）
  - 訓練 AI 老師（評分模型）24 小時提供分數
- 第三階段 PPO（Proximal Policy Optimization）：
  - 學生模型（GPT）不斷回答問題
  - AI 老師持續評分
  - 學生根據評分調整生成策略
  - 目標：讓 reward 跟人類真實偏好一致
- 額外效益：減少 hallucination 與有害言論
  - 進入「社會化」階段如同員工出社會後學習企業規範
  - 模型學會什麼該講、什麼不該講（歧視、危險內容）

> "最重要的事情其實就是要讓它整個社會聽懂人類的指令，所以它就收集了非常多人類它提出來的指令以及人類它覺得它出了這個指令之後它期待 AI 怎麼回這樣子的這個例子。"

## 三、關鍵概念定義

| 概念 | 定義 | 對應段落 |
|------|------|----------|
| Discriminative Model（判別式模型） | 學習 P(Y\|X)，回答分類問題 | Section 1 |
| Generative Model（生成式模型） | 學習 P(X\|Y) 或 P(X)，從零產出資料 | Section 1 |
| Specialist Model | 單一任務專精模型，一個模型只能做一件事 | Section 1 |
| Generalist Model | 同一模型可處理多種任務（GPT-3、Claude 等 LLM） | Section 1 |
| In-Context Learning | 模型根據 prompt 或 few-shot examples 推斷任務，不更新參數 | Section 2 |
| Prompt Engineering | 設計 prompt 與 examples 引導模型行為 | Section 2 |
| SFT（Supervised Fine-Tuning） | 用「指令—理想回答」配對資料 fine-tune 預訓練模型 | Section 3 |
| Reward Model | 學習人類偏好的評分模型，作為強化學習的獎勵函數 | Section 3 |
| PPO（Proximal Policy Optimization） | 強化學習算法，讓模型根據 reward model 評分調整策略 | Section 3 |
| RLHF（Reinforcement Learning from Human Feedback） | 以人類回饋為獎勵的強化學習對齊方法 | Section 3 |
| Hallucination | 模型生成看似合理但實際錯誤的內容 | Section 3 |
| Alignment | 讓模型行為對齊人類價值觀與意圖 | Section 3 |

## 六、核心主旨總結

生成式大型語言模型的核心突破，是從 specialist 走向 generalist 的典範轉移。GPT-3 透過自督式學習讀完海量文字後，in-context learning 機制讓同一個模型可處理各式任務。然而「讀完書」與「聽懂指令」之間仍存在巨大鴻溝——InstructGPT 透過 SFT（學會指令格式）+ Reward Model（學會人類偏好）+ PPO（學會優化方向）三階段對齊流程，讓模型不僅能產出文字，更能產出符合人類意圖、安全與價值觀的內容。這套 RLHF 對齊機制，正是現代 ChatGPT 商業化成功的關鍵技術基礎。
