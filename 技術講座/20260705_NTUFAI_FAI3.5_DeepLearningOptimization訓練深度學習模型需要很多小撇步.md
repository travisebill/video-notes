## 一、主題與背景

FAI 3.5「訓練深度學習模型需要很多小撇步」是 FAI 第三章的尾聲，處理深度學習中的最佳化問題——給定 Loss Function 與模型結構後，要如何實際把參數 θ 推向最小值。陳縕儂教授把焦點放在三個關鍵的工程變量：起步位置、步伐大小、與每一步看到的樣本數。

第一個問題是「滾球會卡在局部低谷」。影片把 Gradient Descent 比喻為沿著 Loss 地形圖滾下的小球，從某個起點開始，計算當下位置的切線斜率，若斜率為負就往 θ 增加的方向滾一步；若斜率為正就往 θ 減少的方向滾。最終小球會停在斜率為零的位置，也就是局部低谷。然而深度學習的 Loss 地形遠比線性模型來得崎嶇——層數一多，局部低谷的數量呈指數級增加；從不同起點出發，最終停下的位置也會截然不同。這就是為什麼深度學習對「初始化點」極為敏感。實務上的解法是在多個隨機起點上各跑一次，最終選 Loss 最低的那一個 θ 作為候選解。

第二個問題是「步伐大小」。學習率（Learning Rate）若設得太大，參數可能直接跨過低谷，反而讓 Loss 上升；若太小，雖然方向正確，卻會在低谷附近卡住，動彈不得。理想的策略是「遠離低谷時大步走，接近低谷時小步走」——也就是自適應（adaptive）的學習率調整。現代最佳化器如 Adam、RMSProp 內建了這類機制，自動根據梯度歷史調整步伐；若使用固定學習率，則需要手動監控訓練曲線、逐步下調。

第三個問題是「每一步要看到多少資料」。標準的 Full-Batch Gradient Descent 每次看完所有訓練樣本才更新一次，方向最精準但速度最慢，尤其當資料量達到上萬筆以上時，每一次更新都要運算整體平均，效率極差；SGD（Stochastic Gradient Descent）則每看完一筆就更新一次，更新次數多，但單筆的梯度估計雜訊大；實務上的最佳折衷是 Mini-Batch SGD——一次取 B 筆（例如 32、64、128）平均後再更新，兼顧速度與穩定性，並且能充分利用矩陣與矩陣相乘的硬體加速效益。

影片最後提醒，深度學習的最佳化並不是「找到一個確定最佳的 θ」，而是「盡量少卡在糟糕的局部低谷」。Batch Size 通常建議固定，避免每次重新調整 cache 機制；每個 epoch 內對訓練資料做隨機洗牌，確保模型學不到資料順序的偽相關；Learning Rate 則建議參考已發表論文的設定，並結合 gradient clipping 等技巧處理梯度爆炸問題。

**核心議題一句話**：深度學習的最佳化既包含梯度下降的數學原理，也疊加了 Learning Rate、Batch Size、隨機化與 adaptive optimizer 等多項工程技巧；這些「小撇步」看似瑣碎，卻是把理論模型推向實際可用的關鍵所在。

## 二、章節脈絡

### Section 1｜最佳化目標：minimize Loss（00:00 ~ 02:30）

**重點摘要：** 給定 Loss Function 與模型參數 θ；暴力法不可行；最佳化的數學基礎是偏微分。

**內容：**
- 已經有了模型，也有了 Loss Function 的定義
- 接下來就要開始學這個模型
- 目標：找一組參數 w 能夠 minimize E_in
- 暴力法：把所有可能的 θ 都帶進去看哪個最低
- 但參數量大的時候組合數無限多種，暴力法不可行
- 聰明的辦法是利用偏微分
- minimize Loss 時，微分為零時就是 Loss 為極值的地方
- 因此可以利用偏微分的概念尋找比較好的 w
- 但深度學習的 Loss 沒有 Linear Model 那麼漂亮
- 層數多時 Loss 的 surface 會非常複雜
- 要在上面找到 Loss 比較低的地方，是個難題

> "一個聰明的方法呢就是利用偏微分...能夠minimize Loss的時候，一定就是偏微分為零的時候。"

### Section 2｜Gradient Descent：滾球比喻（02:30 ~ 08:00）

**重點摘要：** Gradient Descent = 沿 Loss 地形圖滾球；從初始點出發計算切線斜率決定移動方向；最終會停在局部低谷。

**內容：**
- Gradient Descent 的概念：滾球
- 隨便 Random 一個初始位置 θ_0
- 在那個位置看對 θ 的偏微分，也就是切線
- 切線斜率為負 → 往 θ 增加方向走（Loss 下降）
- 切線斜率為正 → 往 θ 減少方向走（Loss 下降）
- 一直走到斜率為零的地方就停下
- 這個位置就是 local minima
- 這只是 local minima 不是 global minima
- Loss 表面可能有更深的谷沒被走到
- 所以 Deep Learning 對初始化點很敏感
- 從不同地方丟球，可能會滾到不同位置

> "Gradient Descent的做法其實就是一個滾球的做法...最後的位置就會是一個local minima。"

### Section 3｜處理 Local Minima 的實務技巧（08:00 ~ 13:00）

**重點摘要：** Local Minima 無法完全避免；實務做法是 Random Initialize 多個起點；選 Loss 最低的那個。

**內容：**
- 用 Gradient Descent 的確沒辦法保證找到 global minima
- 因為 Loss 表面不是漂亮的單一谷底
- 引發出一些更簡單的做法
- 例如 Random Initialize 多次
- 每次都跑一次 Gradient Descent
- 看看哪一個可以滾到相較之下最低的點
- 那個可能就是比較好的 θ 的選擇
- 實務上：如果你發現模型訓練不起來
- 可以看一下是不是每次固定同樣的起點
- 同樣的起點開始可能都會跑到同樣的地方
- 那就要試試看 Random 不同起點
- 這是非常簡單的做法

> "我們可以Initialize多次，然後每一次都跑一下，然後看看哪一個可以滾到相較之下是最低的這個點。"

### Section 4｜Learning Rate：步伐大小的策略（13:00 ~ 19:00）

**重點摘要：** Learning Rate 太大會跨過低谷；太小會卡住；理想是自適應調整；現代 optimizer 如 Adam 內建 adaptive 機制。

**內容：**
- Gradient Descent 式子：θ 對 θ 做偏微分，更新 θ
- 學習率 η 決定每一步跨大步還是小步
- Learning Rate 太大的問題：可能跨過低谷
- 設得太大時，網友一步步跨過，Loss 反而上升
- Learning Rate 太小的問題：可能卡在低谷附近
- 雖然方向對，但動彈不得
- 理想情況：Loss 慢慢慢慢下降
- 比較理想的應該是：離低谷遠時大步走，接近低谷時小步走
- Learning Rate 不應該固定一個值，要能 adapt 調整
- 現代大家比較方便：使用已經設計好的 optimizer
- 有些會自己去 adapt 調整 Learning Rate

> "如果你選用的這個learning rate設得太大的話...就會完全就是一跳就越過了那一整個低低谷的地方，所以設定到太大你反而cost會上升。"

### Section 5｜Batch Size 取捨（19:00 ~ 25:00）

**重點摘要：** Full-Batch 看所有樣本才更新一次 → 慢；SGD 看一筆就更新 → 雜訊大；Mini-Batch 看 B 筆平衡速度與穩定性。

**內容：**
- 一個 epoch = 看完所有 training data 一次
- Full-Batch：每次看完所有 training data 才更新一次 θ
- SGD：每看完一個 example 就更新一次
- Mini-Batch SGD：一次看 B 個 example 後更新一次
- SGD 雖然更新次數多，但每次只用一筆資料
- 可以更新很多次，但單筆的梯度估計雜訊大
- Mini-Batch 利用硬體的矩陣運算加速
- 矩陣跟矩陣相乘，比向量跟矩陣相乘更有效率
- 所以 Mini-Batch SGD 既快速又穩定
- 實務上 Batch Size 32 或 64 或 128 是常見設定

> "因為你就是不能夠每一次都做vector跟matrix的相乘...Mini-Batch它的好處就是它利用了在硬體上面矩陣跟矩陣相乘它已經要去plane它的好處。"

### Section 6｜Training 隨機化與其他小撇步（25:00 ~ 29:59）

**重點摘要：** Batch Size 要固定；資料每 epoch 隨機洗牌避免學到順序；Learning Rate 與 Batch Size 比例有實驗結果；最終回顧 Deep Learning 訓練挑戰。

**內容：**
- Mini-Batch Training 的小撇步
- 每個 epoch 內的 training data 應該要被洗牌
- 如果每次都一樣的 batch，可能會學到資料順序的 pattern
- Training data 要 Random 洗牌才對
- Batch Size 要固定，不要每次設不同
- 固定後每次 SWAP 進去的 X 就可以繼續使用
- 不要重新 batch 一次
- Learning Rate 跟 Batch Size 比例有 Pretty Good 實驗
- 探討怎麼樣的比例設定比較理想
- 結尾 review：Deep Learning 訓練需要考量四大面向
  - Efficient Gradient Computation（Backpropagation）
  - Careful Initialization（避免梯度消失 / 爆炸）
  - Modify Learning Rate（adaptive optimizer）
  - Domain Knowledge（選擇適合的模型架構）
  - Many Tricks：Regularization、Dropout、Early Stopping
- 要小心地 Returnation 跟 Regularization，這是 Deep Learning 訓練中最重要的事

> "Learning Rate其實會跟Batch Size是有關的，那會有一些pretty good的實驗就是有看說這個batch size的這個跟learning rate他之間的這個比例的設定怎麼樣是更加理想的。"

## 五、核心主旨

> 給定 Loss Function 與模型參數 θ，深度學習的最佳化是利用偏微分找到 Loss 的極小值。Gradient Descent 沿著 Loss 地形圖滾動找最低點，但實際上只能抵達局部低谷，因此深度學習對初始化極為敏感——常見的實務做法是在多個隨機起點上各自跑一次，最終選 Loss 最低的那一組 θ。Learning Rate 若太大會跨過低谷、若太小會卡住，理想的策略是自適應調整步伐——現代的 Adam 與 RMSProp optimizer 已內建這項機制。Batch Size 方面，Full-Batch 更新方向準確但速度慢，SGD 更新次數多但雜訊大，Mini-Batch SGD 取 B 筆平均後更新，兼具速度與穩定性，並能充分利用矩陣相乘的硬體加速。訓練細節上，固定 Batch Size、每 epoch 隨機洗牌、調整 Learning Rate 與 Batch Size 比例，都是實際可操作的「小撇步」。深度學習的最佳化既包含梯度下降的數學原理，也疊加了十幾項工程技巧；前者給出理論方向，後者把理論模型推向實際可用的層次。
