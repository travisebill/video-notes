## 一、主題與背景

- **講者：** Boris，Claude Code 的創作者（被稱為「Claude Code 之父」），任職於 Anthropic。先前創辦過多間新創公司，經歷過 Y Combinator，編寫過程式設計和 TypeScript 教科書。自稱「過去一年幾乎沒親手寫過一行程式碼」
- **場合：** Square 開發者大會現場訪談（主持人事前調查：全場絕大多數人都在用 Claude Code，許多人自稱有「Claude Code 精神病」）
- **影片類型：** 技術論壇訪談 + 現場問答
- **核心議題：** AI 正將軟體開發的成本降低 10-100 倍——這對程式設計師、軟體產品、新創公司和整個產業意味著什麼？Boris 用 15 世紀印刷機的類比，預測一個「寫程式像傳簡訊一樣普及」的未來

## 二、章節脈絡

### Section 1｜開場：誰在用 Claude Code？（00:00 ~ 02:00）

**重點摘要：** 主持人暖場調查，全場舉手表示用過 Claude Code；許多人承認有「Claude Code 精神病」。Boris 被介紹為「改變了整個軟體開發模式」的人。

**內容：**
- 主持人的開場調查：
  - 誰用過 Claude Code？（幾乎全場舉手）
  - 誰有「Claude Code 精神病」？（大量舉手）
- > 「我們知道整個軟體開發的未來都壓在你的肩膀上，感謝你抽出時間。」

### Section 2｜工程師角色的轉變：從寫程式到設計系統（02:00 ~ 06:00）

**重點摘要：** Boris 認為 AI 不會消滅工程師，但會徹底改變他們的工作內容——從實作細節轉向系統設計、產品判斷，以及跨領域的通才能力。

**內容：**
- 軟體開發中最困難的部分不是編碼，而是理解問題、定義架構、整合系統
- 「我們不再需要花時間處理實作細節，AI 可以處理那些」
- 工程師的角色轉變：從「寫程式的人」→「設計系統的人」
- Anthropic 團隊的現狀：**每個人都在寫程式**——工程經理、產品經理、設計師、資料科學家、財務人員、使用者研究員——全部
- > 「我們的設計師會寫程式，我們的產品經理會寫程式。跨領域的通才——工程師同時擅長產品、設計、資料科學——這是我們在團隊中開始看到的現象。」

### Section 3｜SaaS 末日？——七種競爭優勢的重新洗牌（06:00 ~ 10:00）

**重點摘要：** 全場最精彩的段落。Boris 引用 Hamilton Helmer《七種力量》的商業護城河理論，分析哪些護城河在 AI 時代變弱、哪些不受影響，以及為什麼現在是「創業最好的時代」。

**內容：**
- 引用《Acquired》播客和 Hamilton Helmer《七種力量》（7 Powers）的商業護城河框架
- **AI 時代變弱的護城河：**
  - ❌ **轉換成本（Switching Costs）**：「你可以用模型直接從一個平台移植到另一個。」
  - ❌ **流程優勢（Process Power）**：「Claude 非常擅長理解流程。尤其是 4.7——你給它一個目標，告訴它迭代到完成為止，它就會做到。這是第一個能做到這一點的模型。」
- **不受 AI 影響的護城河：**
  - ✅ 網路效應（Network Effects）
  - ✅ 規模經濟（Scale Economies）
  - ✅ 壟斷資源（Cornered Resources）
- 對新創的預測：
  - 未來 10 年新創數量將增加 10 倍
  - > 「現在一個微型新創可以打造出跟大公司一樣有價值的產品，而且可以正面競爭。因為大公司必須改變營運流程、重新訓練員工使用新技術——他們會面臨巨大的內部阻力。但從零開始的你沒有這個問題。」

### Section 4｜模型 vs 產品：Claude Code 成功的關鍵比例（10:00 ~ 14:00）

**重點摘要：** 現場問答精華。Boris 被問到 Claude Code 的成功有多少歸功於模型、多少歸功於產品——他給出「一年前 50-50，未來產品比重會下降」的坦誠回答，並強調「做出人們真正喜愛的東西」這個 YC 核心教條從未過時。

**內容：**
- 問：Claude Code 的成功歸因——模型 vs 產品？
- 答：一年前約 50-50；六個月前還是 50-50
- 兩年後呢？Boris 坦言不知道——Anthropic 只計劃「一週後的事」
- > 「在 YC 一再強調的一件事——做出人們喜愛的東西。無論模型、技術如何，最終你還是必須做出人們真心喜歡使用的東西。」
- 「我們花非常多注意力在那些小細節上——這樣你整天使用它時，體驗才會真正好。」
- 未來預測：一年後模型本身會更對齊，現有的所有安全機制（提示注入防護、靜態驗證、權限模式）都將不再那麼重要——因為模型會「直接做正確的事」

### Section 5｜程式設計的民主化：從印刷機到 AI（14:00 ~ 18:00）

**重點摘要：** Boris 用 15 世紀歐洲印刷機革命的類比，預測軟體開發將像「識字率」一樣從極少數精英技能變為全民普及——而且變革速度會快得多。

**內容：**
- 被問到：寫程式會不會變成像使用 Microsoft Office 一樣的基本技能？
- Boris 的歷史類比——15 世紀歐洲印刷機：
  - 印刷機之前：歐洲僅 10% 人口識字，識字是國王和貴族僱用的專業技能
  - 印刷機之後 50 年：出版的文獻總量超過之前 1000 年的總和
  - 書籍成本下降 100 倍
  - 之後幾百年：全球識字率從 10% → 70%+
- > 「寫程式這項技能將會變成——我不知道——就像我會傳簡訊一樣。你不需要一個閱讀寫作學位才學會讀寫。雖然仍然有專業作家存在。」
- > 「最好寫會計軟體的人，可能今天已經不是工程師了——而是一個真正優秀的會計師。因為他們對領域的理解很深，而編碼是簡單的部分。知道領域才是困難的部分。」

### Section 6｜Anthropic 內部：活在未來的工作方式（18:00 ~ 21:00）

**重點摘要：** Boris 揭露 Anthropic 內部的極致 AI 化工作方式——全部程式碼由模型產生、Claude 們互相在 Slack 上溝通協作、組織流程才是真正的差距所在。

**內容：**
- 問：Anthropic 內部與外界的差距有多大？
- 模型上沒有差距——Anthropic 內部使用跟外界相同的模型
- 產品端差距較大——但那是因為組織流程的改變
- Anthropic 內部的工作方式：
  - > 「我們用 Claude 做所有事情——literally everything。」
  - > 「我的 Claude 們整天都在交談——當我的 Claude 在迴圈中寫程式時，它們會透過 Slack 跟其他人的 Claude 溝通，那些 Claude 也在迴圈中運行，互相解決未知問題。」
  - > 「公司裡再也沒有任何一行手寫的程式碼。所有 SQL 都是模型寫的，所有東西都是模型建的。」
- 真正的差距不在技術——在組織架構和流程

### Section 7｜多代理、電腦操作、本地 vs 雲端——快問快答（21:00 ~ 24:35）

**重點摘要：** 最後幾輪現場快問，涵蓋多代理並行化、MCP（Model Context Protocol）、電腦操作（Computer Use）、本地模型 vs 雲端模型等熱門話題。

**內容：**
- **多代理並行化：** 產品端只是提示工程的問題——每週調整提示讓模型更好地並行工作。但 Opus 4.7 已經開始「自己決定」要開迴圈和啟動代理
  - > 「我告訴 Opus 4.7 去拉這個資料查詢，它會說『嘿，我注意到資料在隨時間變化，我開一個迴圈每 30 分鐘給你報告』——然後用 Slack MCP 發給我。」
- **Co-Work 工具的存取：** 答案就是 MCP——同一個 MCP 連接器，Claude Code 可以用、Co-Work 可以用。對於沒有 MCP 的系統——電腦操作（Computer Use）是萬能方案
  - > 「電腦操作是一切的兜底方案。Anthropic 在這方面領先很多。它能使用你電腦上幾乎任何軟體——雖然很慢，但做得很好。」
- **本地 vs 雲端模型：** Boris 的終極答案——「這不重要」
  - > 「幾年後，模型會自己決定用本地還是雲端模型。我們作為工程師不再需要做這些決定了。」

## 三、關鍵概念定義表

| 概念 | 定義 | 出處／應用 |
|------|------|-----------|
| 七種力量（7 Powers） | Hamilton Helmer 提出的七種商業護城河：網路效應、規模經濟、壟斷資源、流程優勢、轉換成本、品牌、訂位 | 分析 AI 如何重塑 SaaS 競爭格局 |
| 產品懸垂（Product Overhang） | 產品已準備好，只等模型能力跟上來就會爆發的現象 | Claude Code 的開發策略：提前 6 個月打造產品 |
| 狗糧化（Dogfooding） | 公司內部使用自家產品作為開發流程的一部分 | Anthropic 全員使用 Claude Code 和 Co-Work |
| MCP（Model Context Protocol） | Anthropic 提出的模型連接外部工具和服務的標準協定 | Claude Code、Co-Work 統一使用 MCP 串接 SalesForce、Google Docs 等 |
| 電腦操作（Computer Use） | 讓 AI 直接操控電腦圖形介面——不需 API 就能使用任何軟體 | MCP 不存在時的萬能兜底方案 |
| 通才工程師（Generalist Across Disciplines） | 同時精通產品、設計、資料科學等多領域的工程師 | Anthropic 團隊的招聘和組織趨勢 |

## 四、人物分析

### Boris（Claude Code 創作者）
- **背景：** 經歷過多間新創公司和 Y Combinator，編寫過程式設計和 TypeScript 教科書。自稱過去一年幾乎沒寫過程式碼
- **思維風格：** 極度坦誠——公開承認「只計劃一週後的事」、承認自己無法預測兩年後的模型-產品比例
- **核心世界觀：** 用歷史類比來理解 AI 革命的規模——印刷機讓識字率從 10%→70%，AI 將讓程式設計從極少數專業技能→全民普及，而且速度更快
- **產品哲學：** YC 核心教條從未過時——「做出人們喜愛的東西」。無論技術多強，產品的細節和體驗仍舊決定一切

---

## 五、核心主旨

> 軟體開發正經歷類似 15 世紀印刷機對識字率的衝擊——從極少數精英的專業技能，變成全民普及的基本能力。這不是工程師的末日，而是軟體產品的黃金時代。轉換成本和流程優勢這兩大傳統護城河正在消失，但網路效應和規模經濟仍然堅不可摧。最快適應的不是擁有最好模型的公司，而是最快改變組織流程的團隊。
