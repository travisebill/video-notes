## 一、主題與背景

FAI 5.4「RBF 網路與密度估計（Density Estimation with RBF Networks）」是陳縕儂教授在 Unsupervised Learning 章節的最後一講，把密度估計問題從直方圖的啟發式做法推向嚴格的模型化方法，並展示 RBF Network 如何整合先前章節的多個工具。

節目一開始複習機率密度函數（PDF）的概念。對一個連續隨機變數，PDF 給出落在某個位置附近的機率密度，例如 IQ 分布大致呈常態分配。實務上常以直方圖（Histogram）估計，但直方圖的問題是切 bin 寬度會嚴重影響形狀——鄰近 bin 的差異可能被過度放大或抹平（159 與 150 落在同一 bin，159 與 161 只差 2 卻落在不同 bin）。教授點出，更好的做法是把密度視為若干「高斯形的山丘」加總而成，這正是 RBF Network 的物理圖像。

接著介紹 RBF Network 的數學結構。RBF 是「Radial Basis Function」的縮寫，包含兩個關鍵字：Basis Function 是構成整體形狀的基本元件；Radial 描述的是「以某個中心 μ 為基準，距離越遠強度越弱」的形狀，最典型的就是高斯函數。把 K 個高斯基底函數各自乘上一個權重 β，再加上一個偏置項，線性組合起來就是 RBF Network 的輸出公式。當有監督資料時，β 與 μ 可以直接學；本節重點放在無監督情境下的做法。

**核心議題一句話**：密度估計是把「資料分布的形狀」用若干高斯山丘加總來逼近的問題——KDE 是最直觀的特例（每筆 X 都當一個獨立高斯），K-Means + RBF 是更具代表性的改良方案（先用 K-Means 找 prototype 再組 RBF）。

## 二、章節脈絡

### Section 1｜PDF 與直方圖的限制（00:00 ~ 05:00）

**重點摘要：** 機率密度函數（PDF）描述隨機變數在某位置的機率密度；直方圖是常見估計方法但 bin 寬度敏感（159 vs 150 同一 bin、159 vs 161 不同 bin）；更好的做法是用高斯山丘加總逼近真實分布。

**內容：**
- PDF（Probability Density Function）= 在某位置的機率密度
- 直方圖估計的問題：
  - Bin 寬度嚴重影響形狀
  - 鄰近 bin 的差異可能被過度放大或抹平
  - 例如：159 與 150 落在同一 bin，但 159 與 161（只差 2）落在不同 bin
- 改進方向：用若干高斯山丘加總逼近真實分布
- 高斯山丘的物理意義：例如不同城市的收入分布（雙峰）、租金分布等都是高斯混合
- 目標：從有限的觀察樣本估計整體分布

> "像剛剛的這一個藍色的這一一個Curve它其實很像是兩個高峰...它是兩個高選而跌加出來的這一個分布。"

### Section 2｜RBF Network 的數學結構（05:00 ~ 13:00）

**重點摘要：** RBF = Radial Basis Function，Basis Function 是基本元件、Radial 是「距離中心越遠強度越弱」的形狀；K 個高斯基底函數線性組合就是 RBF Network；有監督時 β 與 μ 可直接學。

**內容：**
- RBF = Radial Basis Function（徑向基底函數）
- Basis Function：構成整體形狀的基本元件
- Radial：以某個中心 μ 為基準，距離越遠強度越弱
- 高斯函數是最典型的 RBF
- RBF Network 結構：把 K 個高斯基底函數各自乘上權重 β，再加上一個偏置項，線性組合得到輸出
- 數學公式：y(x) = Σ βₖ × RBF(x, μₖ) + bias
- 與傳統神經網路的差異：
  - 傳統神經網路中間層是 sigmoid/tanh（內積型函數）
  - RBF Network 中間層是高斯函數（距離型函數）
  - 兩者的輸出層都是線性組合
- 有監督情境下：β 與 μ 都可以直接從資料學
- 本節重點：無監督情境下如何用 RBF 做密度估計

> "BasisFunction其實就是像剛剛說的其實你可以把它想成就是拿幾個基本的Function而組成的...Radio是什麼呢？它是輔射的意思，所以輔射是越接近中心的地方它的輔射是於輔射強度越強。"

### Section 3｜KDE 與 Naive Density Estimation（13:00 ~ 18:00）

**重點摘要：** KDE（核密度估計）是最簡單的密度估計做法——假設每筆 X 都來自一個獨立的、以該 X 為中心的高斯，每個高斯權重相等（1/N），把所有高斯加起來得到整體密度估計。

**內容：**
- KDE（Kernel Density Estimation）的核心想法：
  - 每一筆觀測到的 X 都來自一個獨立的、以該 X 為中心的高斯
  - 每個高斯的權重相等（都是 1/N）
  - 把所有高斯加起來，得到整體密度的估計值
- 物理意義：每個 X 都是某個分布的「樣本」，背後都有一個高斯
- 數學公式：density(x) = (1/N) × Σ Gaussian(x; Xᵢ, σ²)
- 優點：直觀、簡單、無需選擇 K
- 缺點：
  - 把每一筆 X 都當成獨立高斯，過於精細
  - 容易過擬合（overfit）
  - 計算量隨樣本數線性增長
- 與 K-Means 的對比：K-Means 用 K 個中心代表整體分布，KDE 用 N 個中心

> "它的物理意義其實就是每一個你Samport到的X其實都是從你後面的這個高選出來的...這個這樣子的這個Network其實就叫做Colonel Density Estimation就是KDE。"

### Section 4｜K-Means + RBF 的改良方案（18:00 ~ 23:00）

**重點摘要：** KDE 過於精細的問題可用 K-Means + RBF 改良——先用 K-Means 把資料分成 K 群，再以每群中心作為 RBF 的中心 μ，每群權重依群內比例設定；既保留 RBF 幾何直覺又避免 KDE 過擬合。

**內容：**
- KDE 的問題：把每一筆 X 都當獨立高斯，過於精細
- 改良方向：用少數代表性的高斯中心（prototype）取代所有 X
- 具體做法（K-Means + RBF）：
  - 第一步：用 K-Means 把資料分成 K 群
  - 第二步：以每群中心 μₖ 作為 RBF 的中心
  - 第三步：每群權重 βₖ 依群內資料比例設定
- 這個做法把 FAI 5.2 的 K-Means 與本講的 RBF 結合
- 優點：
  - 避免 KDE 的過擬合
  - 計算量大幅降低（K 個高斯 vs N 個高斯）
  - 仍保留 RBF 的幾何直覺
- 與一般神經網路的最終對比：
  - 一般 NN：sigmoid/tanh 中間層 → 線性輸出
  - RBF：高斯中間層（距離型） → 線性輸出
  - 兩者本質上都是「中間層特徵變換 + 線性組合」的設計

> "Rbf network我們希望它能夠找到稍微比較少一點比較具有代表性的這個ProType來當成它的Basis...K-Mix我們其實是可以根據Data先做一個Cross3就是先把它分成K群。"

### Section 5｜RBF Network 在異常偵測的應用（23:00 ~ 28:30）

**重點摘要：** RBF Network 估計出的密度可直接用於異常偵測——落在低密度區域的樣本即為異常點；K-Means 與密度估計兩種觀點可能給出不同的異常判定，實務上可互補使用。

**內容：**
- 異常偵測的兩種做法：
  - K-Means 觀點：距離所有 cluster 都很遠 → 可能是異常
  - 密度估計觀點：落在低密度區域 → 可能是異常
- RBF 估計出的密度函數可直接用於異常偵測：
  - 高密度區域（如資料集中區）→ 正常
  - 低密度區域（如分布邊緣）→ 異常
- 範例：半導體機台製程
  - 依據 LightDensity 與 Throughput 的分布
  - 大部分資料集中在某個 cluster
  - 偏離該 cluster 的點可能是異常機台，需 Morning 處理
- K-Means 與密度估計的差異：
  - K-Means 可能把「離群但周圍還有資料」的點誤判為異常
  - 密度估計能更細緻地反映實際機率
- 實務建議：兩種方法可互補使用，或依場景選擇
- 章節總結：Unsupervised Learning 四大工具（Clustering、Outlier Detection、Density Estimation、Representation Learning）的整合

> "Density Estimation也是其中一個預估在這個Data背後它實際上的這些密度它的Deutrebution...Kaming可能是也可以用的，那當然我們的DensityDemotion也是可以用的。"

## 三、關鍵概念定義

| 概念 | 定義 | 重要性 |
|------|------|--------|
| 機率密度函數（PDF） | 描述隨機變數在某位置附近的機率密度 | 密度估計的理論基礎 |
| 直方圖（Histogram） | 用 bin 統計樣本分布的估計方法 | 簡單但 bin 寬度敏感 |
| RBF（徑向基底函數） | 以中心 μ 為基準、距離越遠強度越弱的函數 | RBF Network 的核心元件 |

| 核密度估計（KDE） | 每筆 X 都當獨立高斯、加總估計密度的方法 | 最簡單的密度估計做法 |
| 樸素密度估計（Naive Density Estimation） | KDE 的另一個稱呼 | 強調其簡單直觀 |
| Prototype（代表性樣本） | 用少數代表性樣本代表整體分布的概念 | K-Means + RBF 的核心 |
| 異常偵測（Anomaly Detection） | 識別偏離正常分布的樣本 | RBF 的典型應用 |

## 五、核心主旨

> 密度估計（Density Estimation）是把「資料分布的形狀」用若干高斯山丘加總來逼近的問題，而徑向基底函數網路（RBF Network）是其中最具模型觀點的做法。RBF 由「Basis Function（基本元件）」與「Radial（距離中心越遠強度越弱）」兩部分構成，最典型的 RBF 是高斯函數；把 K 個高斯基底函數各自乘上權重 β，再加上一個偏置項，線性組合起來就是 RBF Network 的輸出公式。最簡單的無監督做法是核密度估計（KDE）：假設每一筆 X 都來自一個獨立的、以該 X 為中心的高斯，每個高斯的權重相等（1/N），把所有高斯加起來即得整體密度估計。但 KDE 把每一筆 X 都當成獨立高斯，過於精細且容易過擬合，更務實的做法是用 K-Means 先把資料分成 K 群，再以每群的中心作為 RBF 的中心 μ，每群的權重 β 則依群內資料比例設定——這種 hybrid 做法既保留了 RBF 的幾何直覺，又避免了 KDE 的過擬合問題。RBF Network 估計出的密度函數可直接用於異常偵測：落在低密度區域的樣本即為異常點；K-Means 與密度估計兩種觀點可能給出不同的異常判定，實務上可互補使用。本講也展示了 FAI 5.2 的 K-Means 如何被整合進 RBF Network，揭示非監督式學習四大工具（分群、異常偵測、密度估計、表示學習）的內在統一性。
