## 一、主題與背景

FAI 2.1「為什麼需要非線性的模型呢？」是陳縕儂教授 Machine Learning 章節進入「Nonlinear Models」的開篇，約 40 分鐘。本講回答 FAI 1.x 系列一直迴避的關鍵問題：當資料在原本的 $X$ 空間無法被 hyperplane 切開時，Linear Model 就失敗了——怎麼辦？

線性模型的限制來自於「直線切不開」的幾何事實。Linear Model 在 2D 平面上就是一條直線，在更高維度就是一個 hyperplane；如果資料的分佈是圓形、橢圓、S 形、雙曲線等，hyperplane 就無法把它分好。例如陳縕儂教授在影片中展示的「圈圈包圍叉叉」資料：用任何直線切都會有大量錯誤，$E_{in}$（in-sample error）居高不下。

「$E_{in}$ 大」本身就已經是大問題——回想 FAI 1.4 的 Generalization 討論：$E_{in}$ 是 $E_{out}$ 的 proxy，$E_{in}$ 大通常代表 $E_{out}$ 也大，這種情況下不論模型多簡單都救不回來。我們必須從「換模型」下手。

本講的核心觀念是 **Feature Transform（特徵轉換）**：把資料 $X$ 經由非線性函數 $\Phi$ 映射到新的 $Z$ 空間，使得原本不可分的資料在 $Z$ 空間中變得線性可分。這個概念是 1990 年代 Kernel Methods（如 SVM with RBF kernel）、2000 年代 Deep Learning（Convolutional Layers）的共同祖先。Linear Model 在 $Z$ 空間仍然是 Linear，但「對應回 $X$ 空間」就成為非線性曲線（圓、橢圓、拋物線等）。這個看似簡單的「換個空間做線性」想法，卻是整個非線性機器學習的核心 trick。

陳縕儂教授用「圓的多項式」當第一個範例：$h(x) = (x_1^2 + x_2^2 - r)$ 對 $x$ 不是線性，但如果定義 $z_0 = 1$、$z_1 = x_1^2$、$z_2 = x_2^2$，則 $h(x) = \tilde{w}_0 z_0 + \tilde{w}_1 z_1 + \tilde{w}_2 z_2 = \tilde{w}^T z$ 對 $z$ 就是線性。這個 Quadratic Hypothesis 的代表力比 Linear 強很多——它可以包含圓、橢圓、雙曲線、拋物線等所有二次曲線，而 Linear Model 是它的特例（二次項係數為 0）。

本講的另一條主軸是 **Model Complexity 與 Overfitting 的 trade-off**。當我們把 Polynomial Order $Q$ 拉高（比如 $Q=4$、$Q=10$），Hypothesis Set 變得非常大（$1+d+d^2+\dots+d^Q = O(d^Q)$ 維度），$E_{in}$ 可以壓到 0（Fully-grown），但同時 $E_{out}$ 也容易暴漲——因為模型把訓練資料的「noise」也學進去了。陳縕儂教授用一個關鍵直覺：「就像背考古題一樣，背得滾瓜爛熟不代表真的會」——模型太複雜就是「死背」，遇到新題目就不行。這個 insight 為 FAI 2.2（Overfitting）埋下伏筆。

**核心議題一句話**：Linear Model 在 $X$ 空間切不開資料時，需要 Feature Transform $\Phi$ 將 $X$ 映射到 $Z$ 空間，讓資料在 $Z$ 空間線性可分——這個「換個空間做線性」的 trick 是整個非線性 ML 的起點；但 Hypothesis Set 越複雜（Order $Q$ 越高）表示力越強，也越容易 Overfitting，$E_{in}$ 變小不代表 $E_{out}$ 變小——這是 Model Selection 的核心難題。

### Section 2｜Linear Model 的限制：直線切不開（05:00 ~ 11:00）

**重點摘要：** Linear Model 在 2D 是一條直線；如果資料是「圈圈包圍叉叉」直線切不開，$E_{in}$ 永遠大；Generalization 再強也沒用。

**內容：**
- Linear Model $h(x) = w^T x$ 是 2D 平面上的一條直線
- 數學式：input $x$ 乘上對應的 weight 加起來
- Linear Model 的好處：簡單、快速、Generalization 強（FAI 1.4 的結論）
- 為什麼 Generalization 強？因為簡單的模型只記得 high-level 大概念，不會死背資料
- 但 Linear Model 的限制：直線切不開某些資料（例如「圈圈包叉叉」）
- $E_{in}$ 居高不下 → 不論 Generalization 多好，$E_{out}$ 也不會好
- 結論：模型必須升級，但升級會犧牲簡單性、增加 Overfitting 風險

> "如果你設定的越複雜你其實就需要更多的資料來把它學的好，因為它非常的容易overfitting。"

### Section 3｜Quadratic Hypothesis：圓的多項式（11:00 ~ 18:00）

**重點摘要：** 用「圈圈包圍叉叉」資料，示範 Hypothesis Set 升級為二次多項式；圓 $h(x) = (x_1^2 + x_2^2 - r)$ 對 $x$ 非線性，但對 $z = (1, x_1^2, x_2^2)$ 是線性。

**內容：**
- 「圈圈包叉叉」資料示意：直線無法切開，但用一個圓可以
- 圓的方程式：$(x_1 - a)^2 + (x_2 - b)^2 = r^2$，展開為 $h(x) = x_1^2 + x_2^2 - r$（取決於 sign）
- 圓的多項式：$a x_1^2 + a x_2^2 + b$（係數可能為 0）
- General 寫成 Quadratic Hypothesis：$a_1 x_1^2 + a_2 x_2^2 + a_3 x_1 x_2 + a_4 x_1 + a_5 x_2 + a_6$
- 圓只是 Quadratic 的一個 special case（其他係數為 0）
- 包含 Linear（一次項）、Quadratic（二次項）、所有二次曲線（圓、橢圓、拋物線、雙曲線、旋轉後的橢圓）

> "h(x) 這個 hypothesis 對 x 不是線性的關係，那我現在做一個小小的轉換...h(x) 是不是就會變成 w 跟 z 之間就是 vector 相乘的關係。"

### Section 4｜Feature Transform：把 $X$ 映射到 $Z$ 空間（18:00 ~ 24:00）

**重點摘要：** 定義 $\Phi(x) = (1, x_1^2, x_2^2) = z$，在 $Z$ 空間用 Linear Model 學 $w^T z$；$Z$ 空間的 hyperplane 對應回 $X$ 空間是二次曲線。

**內容：**
- 定義 $\Phi: X \rightarrow Z$，其中 $z_0 = 1$、$z_1 = x_1^2$、$z_2 = x_2^2$
- 在 $Z$ 空間，Linear Model $\tilde{w}^T z$ 學出來一條 hyperplane，把資料切開
- 在 $X$ 空間，這個 hyperplane 對應回來就是「圈圈」（二次曲線）
- 換言之：在 $X$ 不可分 ≠ 在 $Z$ 不可分；如果能找到 $\Phi$ 使 $Z$ 線性可分，就能用 Linear Model 解
- 這就是 **Nonlinear Transform = Feature Engineering** 的核心 idea
- General 化為 Quadratic Hypothesis Set $\mathcal{H}_\Phi$：包含所有二次曲線

> "於是我們轉到 z 空間中我們就只需要用我們原本會用會學 linear hypothesis 的方式在這邊學這條線，我們是不是就可以把圈圈跟叉叉分開了。"

### Section 5｜Recap：Reduction 思路（24:00 ~ 28:00）

**重點摘要：** Circular Learning reduce 到 Linear Learning；演算法課學過的 reduction 概念用在 ML；關鍵是找到好的 $\Phi$。

**內容：**
- 演算法課學過的 reduction：把問題 A reduce 到問題 B，用解 B 的方式解 A
- 這裡的 reduction：Circular Learning → Linear Learning
- 找到合適的 $\Phi$ 後，剩下的工作就跟 FAI 1.x 一模一樣
- 但 $\Phi$ 的選擇不是 trivial——「看到資料就知道要用什麼轉換」需要經驗
- 這也開啟了 ML 中「特徵工程」（Feature Engineering）的整個領域

> "大家應該在演算法的時候有學過reduction吧，就是我們可以把一個問題reduce成另外一個問題，然後我們就用解另外一個問題的方式來解我們遇到原本的這個問題。"

### Section 6｜應用：Spam Email Classification（28:00 ~ 33:00）

**重點摘要：** 1990 年代 Spam Classification 論文：Linear Model 7% error rate，Quadratic Model 1.68% error rate；Linear 不夠好時，升級 Hypothesis 有效。

**內容：**
- 應用：垃圾郵件分類（Spam Detection）
- Input：email 的特徵（哪些字出現、頻率、連續字、出現次數）
- Linear Model：testing error rate 約 7%（93% 正確率）
- 升級到 Quadratic Model（nonlinear）：testing error rate 降到 1.68%
- 原因：$f(x)$ 本身可能就是二次的，Linear Model 跟 $f$ 的差距大；Quadratic Model 接近 $f$
- 但我們永遠不知道 $f$ 是什麼——這是 ML 的根本難題
- 啟示：「模型越接近 $f$，performance 越好」是原則，但實務上需要經驗嘗試

> "如果我們理想的這個 f(x) 它其實是二次項的，那我們可能就應該要找一個二次項的這個 g 才可以學得不錯。"

### Section 7｜應用：Digit Recognition（33:00 ~ 38:00）

**重點摘要：** 手寫數字辨識（Yann LeCun 1989），從 raw pixel 到 engineered feature；郵遞區號辨識；從 Manual Feature Engineering 到 Deep Learning 自動 Feature Learning。

**內容：**
- Yann LeCun 1989 年的 Digit Recognition（CNN 起源之一）
- 應用：郵遞區號自動辨識（美國郵政）
- 從 raw pixel 設計 feature：對稱性、筆劃密度、孔洞數量等
- 設計 feature → 丟入 Linear/Quadratic Model → 預測
- 這就是 Manual Feature Engineering 的經典做法
- Deep Learning 的差異：模型自己學 feature（Convolutional Layers）
- Deep Learning 對影像、文字、語音效果特別好（資料量大、難以 manual feature）

> "deep learning 就可以把它想成中間最重要的這個跟其他非deep learning的方法很大的一個不同就是，它也幫你做了feature transform的這件事情，它直接在裡面學。"

### Section 8｜Higher Order Polynomial 與 Computational Cost（38:00 ~ 40:11）

**重點摘要：** Q-th order polynomial：維度 $1+d+d^2+\dots+d^Q = O(d^Q)$；維度越高計算成本越高；模型複雜度與計算 trade-off。

**內容：**
- General Polynomial Order $Q$：包含所有 $Q$ 次以下的項
- 維度：$1 + d + d^2 + \dots + d^Q$，隨 $Q$ 指數增長
- 計算成本：維度太高時，即使 Linear Model 也要花很多時間
- 還要存儲 $w$、存儲 transform $\Phi$、學出來的曲線可能很複雜
- trade-off：表示力 ↑ vs 計算成本 ↑ vs Overfitting 風險 ↑
- 怎麼選 $Q$？需要 Model Selection（FAI 2.2 主題）

> "越高維度的模型它的表示力是越好的，但是它的缺點就是它非常的難去計算，而且你還要存嘛。"

## 五、核心主旨

> Linear Model 在 $X$ 空間無法切開某些資料（圈圈包叉叉、橢圓等）時，必須升級 Hypothesis Set。Feature Transform $\Phi: X \rightarrow Z$ 把資料映射到新的 $Z$ 空間，使資料在 $Z$ 空間線性可分——在 $Z$ 空間用 Linear Model 學 $\tilde{w}^T z$，對應回 $X$ 空間就是非線性曲線（圓、橢圓、拋物線等）。這個「Reduction: Circular Learning → Linear Learning」的技巧是整個非線性 ML 的起點，是 1990 年代 Kernel Methods 與 2000 年代 Deep Learning 的共同祖先。但 Hypothesis Set 越複雜（Order $Q$ 越高）表示力越強、計算成本越高、Overfitting 風險越大——Model Selection 是 ML 最困難的問題之一。Linear Model 是 Quadratic 的特例（$Q=1$），Deep Learning 是自動 Feature Transform 的延伸。實務上，模型越接近理想 $f$ 表現越好，但我們永遠不知道 $f$ 是什麼——只能靠經驗和驗證集做選擇。
