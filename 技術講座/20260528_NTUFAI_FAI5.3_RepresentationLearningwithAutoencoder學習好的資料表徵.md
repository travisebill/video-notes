## 一、主題與背景

FAI 5.3「表示學習與自編碼器（Representation Learning with Autoencoder）」是林軒田教授在 Representation Learning 章節的核心影片，把先前 FAI 4.3 提過的 Autoencoder 概念做嚴格的線性代數推導，並展示它與 PCA、PageRank 的數學對應關係。

節目一開始複習自編碼器的核心架構。編碼器（Encoder）把輸入 X 壓縮到低維度的隱藏層表示 h，再由解碼器（Decoder）還原回與 X 接近的輸出。訓練目標是讓還原誤差最小化，因此中間的 h 必須保留 X 最關鍵的資訊，這就是表示學習的精神。教授也提到，加入雜訊後再要求還原到乾淨版本（去噪自編碼器，Denoising Autoencoder）能進一步強化表示的穩健性，達到類似正則化的效果。

接著進入線性自編碼器的推導，這是本講最關鍵的數學段落。當隱藏層移除非線性激活函式時，編碼與解碼都只是矩陣乘法，整個網路可以化簡為 X 乘上某個合成矩陣。要讓還原誤差最小，這個矩陣的封閉解會收斂到資料共變異數矩陣的特徵向量（Eigenvector）——也就是在資料分布中方差最大的方向。

這個結論直接連接到主成分分析（PCA）。PCA 的標準做法是先扣除資料平均值，再對共變異數矩陣做特徵值分解，取前幾大的特徵向量當作新座標軸。投影到這幾個軸上不僅能保留最大方差、實現降維，還能把高維資料視覺化到二維或三維空間中。教授用一題學測題目對比：最小化投影方差的方向與最大化投影方差的方向彼此正交，這正好對應「還原誤差最小」與「投影方差最大」兩種目標的數學對偶性。

節目最後展示 PageRank 範例，把表示學習的數學結構延伸到圖譜分析。網頁重要性可用連結關係矩陣的最大特徵向量估計，與線性自編碼器的數學本質完全相同。

**核心議題一句話**：自編碼器與 PCA 在線性設定下數學等價，背後都是用特徵向量找出資料最大方差方向；理解這個共通結構，便能把表示學習、線性降維、圖譜分析等看似不同的工具收斂到同一套幾何直覺上。

## 二、章節脈絡

### Section 1｜Autoencoder 架構複習（00:00 ~ 03:00）

**重點摘要：** Autoencoder 由 Encoder + Decoder 組成，中間隱藏層 h 是壓縮表示；訓練目標是還原誤差最小化；去噪自編碼器透過加入雜訊強化表示的穩健性。

**內容：**
- Autoencoder 架構：Encoder 把 X 壓縮到 h（低維表示）、Decoder 從 h 還原回接近 X 的輸出
- 訓練目標：minimize 還原誤差（X 與 Decoder 輸出的距離）
- h 必須保留 X 最關鍵的資訊 → 表示學習的核心
- 去噪自編碼器（Denoising Autoencoder）：
  - 在輸入 X 加雜訊
  - 要求還原到乾淨版本
  - 強化模型對重要資訊的識別能力，達到類似正則化的效果

> "中間的這個heat and layer他其實就是對於這個資料很好的一個recentation，就是可以足夠充分的表示這一個data他的重要feature。"

### Section 2｜線性自編碼器的封閉解（03:00 ~ 07:00）

**重點摘要：** 移除非線性激活函式後，自編碼器化簡為線性變換；最小化還原誤差的封閉解是資料共變異數矩陣的特徵向量——也就是資料分布中方差最大的方向。

**內容：**
- 線性自編碼器設定：移除隱藏層的非線性激活函式
- 編碼 = X × W1，解碼 = h × W2，整個網路 = X × W1 × W2
- 因為沒有非線性，W1 × W2 可以合成為單一矩陣 W
- 還原誤差 = ||X - X × W||²，最小化此誤差
- 封閉解：W 收斂到資料共變異數矩陣 X^T X 的特徵向量（Eigenvector）
- 特徵向量的物理意義：在資料分布中方差最大的方向
- 第一大特徵向量 = 方差最大的方向
- 第二大特徵向量 = 與第一個正交、方差次大的方向
- 這就是「找新座標軸」的數學本質

> "如果大家還記得的話，最後你的這個solution其實會找到的是IconVector。IconVector其實就是在一個DataDitribution上面在這個X的Data上面如果找這個這些X的IconVector。"

### Section 3｜PCA 與線性自編碼器的等價（07:00 ~ 10:00）

**重點摘要：** PCA 與線性自編碼器在數學上等價——都是對共變異數矩陣做特徵值分解；PCA 多一個「先扣平均值」的步驟；投影到前幾大特徵向量能保留最大方差、實現降維與視覺化。

**內容：**
- PCA（Principal Component Analysis）的標準做法：
  - 第一步：先扣資料平均值（中心化）
  - 第二步：對中心化後資料的共變異數矩陣做特徵值分解
  - 第三步：取前幾大特徵向量當作新座標軸
- PCA 與線性自編碼器的差異：PCA 多一個「扣平均值」的步驟
- 投影到二維或三維空間能直接視覺化高維資料
- 學測題目對比：最小化投影方差 vs 最大化投影方差，兩者方向彼此正交
- 還原公式：X̂ = (X - mean) × V × V^T + mean（V 是特徵向量矩陣）
- 線性降維的實用價值：降低維度 → 減少過擬合風險、加速計算、便於視覺化

> "LinearOwnCode其實就是在希望能夠讓它投印過後它能夠Recon刷出原本的這個X出來...PC的概念其實也是一樣的那只是它是比較多從自己上面那邊來的。"

### Section 4｜PageRank 與線性代數的統一觀點（10:00 ~ 14:36）

**重點摘要：** PageRank 用連結關係矩陣的最大特徵向量估計網頁重要性，與線性自編碼器、PCA 的數學本質完全相同；展示從表示學習、線性降維、圖譜分析的統一數學觀點。

**內容：**
- PageRank 問題：估計每個網頁的重要性
- 連結關係矩陣：每個網頁指向誰、被誰指向
- 重要性估計的核心想法：被重要網頁連結到的網頁也重要
- 形式化：V = P × V（V 是重要性向量，P 是正規化後的連結矩陣）
- 收斂解：V 是 P 的最大特徵向量
- 與自編碼器的數學等價：都是「找特徵向量」問題
- 延伸應用：摘要任務中估計句子的重要性（與其他句子在 embedding 空間的相似度）
- 統一觀點：表示學習、降維、圖譜分析都是同一套線性代數的多種面貌

> "它其實就直接根據Prom找它的IconMactor找出來的這個指其實就會是這邊Converge之後這五個點它的Importance。"

## 三、關鍵概念定義

| 概念 | 定義 | 重要性 |
|------|------|--------|
| 自編碼器（Autoencoder） | 由 Encoder + Decoder 組成的網路，學習資料的壓縮表示 | 表示學習的經典架構 |
| 編碼器（Encoder） | 把 X 壓縮到低維表示 h 的網路部分 | 自編碼器的關鍵組件 |
| 解碼器（Decoder） | 從 h 還原回接近 X 的輸出的網路部分 | 自編碼器的關鍵組件 |
| 去噪自編碼器 | 在輸入加雜訊、要求還原到乾淨版本的訓練策略 | 強化表示的穩健性 |
| 線性自編碼器 | 移除非線性激活函式的自編碼器 | 有封閉解，可嚴格數學分析 |
| 特徵向量（Eigenvector） | 矩陣變換後方向不變的向量 | 線性降維的核心數學工具 |
| 主成分分析（PCA） | 對共變異數矩陣做特徵值分解的降維方法 | 最常用的線性降維技術 |
| 共變異數矩陣 | 描述資料各維度間線性相關性的矩陣 | PCA 的運算對象 |
| PageRank | 用連結矩陣的最大特徵向量估計網頁重要性的演算法 | 圖譜分析的經典應用 |
| 線性降維 | 用線性變換把高維資料降到低維空間 | 表示學習的核心操作 |

## 五、核心主旨

> 表示學習（Representation Learning）的核心是從資料中自動萃取出最具代表性的特徵，而自編碼器（Autoencoder）是其中最經典的架構——Encoder 把輸入 X 壓縮到低維表示 h，Decoder 再從 h 還原回接近 X 的輸出，訓練目標是讓還原誤差最小化。當把隱藏層的非線性激活函式移除時，整個網路化簡為線性變換，最小化還原誤差的封閉解會收斂到資料共變異數矩陣的特徵向量（Eigenvector）。這個結論有兩個重要延伸：第一，線性自編碼器與主成分分析（PCA）在數學上等價——PCA 只是多一個「先扣平均值」的步驟，兩者都是用前幾大特徵向量當作新座標軸，投影後保留最大方差；第二，PageRank 也是同一套數學——用連結關係矩陣的最大特徵向量估計網頁重要性。這條從 Autoencoder、特徵向量、PCA 一路串到 PageRank 的脈絡，揭示了表示學習、線性降維、圖譜分析看似不同的工具，背後其實共享同一套幾何直覺——找出資料分布中方差最大的方向，便能從中提取最關鍵的結構資訊。
