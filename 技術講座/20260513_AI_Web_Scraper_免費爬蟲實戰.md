**內容：**
- 場景：一位剛搬入新城市的婚禮攝影師，需要更多客戶
- 需求：從婚禮場地網站抓取所有場地資訊
- 爬蟲任務：

  - 儲存為 CSV，最終可上傳到 Google Sheets 交給客戶
- > 「AI 會為每個場地生成一句話描述——這樣攝影師打電話時就能進行有智慧的交談。」

### Section 4｜Crawl4AI 基礎架構解析（10:00 ~ 14:00）

**重點摘要：** 講者先展示 Crawl4AI 的基本用法，再擴展為完整爬蟲。介紹兩個核心設定：Browser Config（瀏覽器設定）和 Crawler Run Config（爬蟲執行設定）。

**內容：**
- **Browser Config**：
  - 選擇瀏覽器（Chrome）
  - 設定視窗大小
  - `headless=False`——可以看到瀏覽器實際操作（便於除錯）；`headless=True` 則在背景運行
- **Crawler Run Config**：
  - 指定 LLM 提取策略（例如使用 DeepSeek 模型提取資訊）
  - 可注入 JavaScript：等待特定元素載入後才開始爬取
  - 可截圖
  - 豐富的參數選項
- 基本爬蟲流程：
  1. 設定瀏覽器 → 2. 設定爬蟲配置 → 3. 執行 `crawler.run(url, config)`
  4. 結果以 Markdown 格式回傳（LLM 原生支援的格式）

### Section 5｜完整爬蟲程式碼逐行解析（14:00 ~ 20:00）

**重點摘要：** 全片核心段落——講者逐行解釋完整爬蟲程式碼，展示如何將 Crawl4AI、DeepSeek、Grok 三者串聯。

**內容：**
- **核心函式：`crawl_all_venues()`**
- **資料模型（Venue Model）**：定義每個場地要提取的欄位
  - 名稱、位置、價格、容量、描述
- **LLM Extraction Strategy**：
  - 告訴 LLM：「從原始爬取資料中，提取所有婚禮場地，並填入對應的欄位資訊」
  - LLM 也會自動生成一句話描述
- **迴圈爬取邏輯**：
  - 從第 1 頁開始 → 爬取 → 翻到下一頁 → 重複
  - 直到找不到下一頁按鈕為止（沒有更多結果）
- **CSS 選擇器**：指定 Info Container 區域，只爬取有用內容而非整個網頁
- **結果處理**：
  - 解析 LLM 回傳的 JSON 字串
  - 轉換為場地模型列表
  - 儲存為 CSV 檔案

## 三、關鍵概念定義表

| 概念 | 定義 | 出處／應用 |
|------|------|-----------|
| Crawl4AI | 開源 Python 爬蟲框架，能將爬取結果標記並傳遞給 LLM | 本專案的核心爬蟲引擎 |
| DeepSeek R1 | 中國開發的推理型 LLM，與 OpenAI O1 同等級，成本低 20 倍 | 用於從原始資料中智慧提取結構化資訊 |
| Grok | 擁有專用 AI 晶片的雲端服務，提供 DeepSeek 等模型的免費快速運行 | 免費運行 DeepSeek 的平台 |
| LLM Extraction Strategy | 定義 LLM 如何從爬取的原始資料中提取結構化資訊的策略 | Crawl4AI 的核心功能——爬取 + AI 提取 |
| Browser Config | 設定爬蟲使用的瀏覽器類型、視窗大小、是否顯示 | Crawl4AI 的基礎設定 |
| Headless Mode | 瀏覽器在背景運行（True）或顯示視窗（False） | 除錯時建議設為 False |
| CSS Selector 鎖定 | 用 CSS 選擇器指定只爬取網頁中的特定區塊 | 避免爬取整個網頁，只抓有用內容 |
| Info Container | 網頁中每個資料單元（如一個場地卡片）的 HTML 容器 | 爬蟲遍歷的基本單位 |
