## 核心理念：讓智慧觸手可及

我們的主要追求之一是**構建更好的開源模型**。開源模型的核心價值在於民主化智慧：

- **部署靈活性**：可以部署在任何地方——本地伺服器、雲端
- **完全透明**：你可以訪問模型的所有權重，而不是只面對一個黑箱
- **快速追趕**：開源模型正在快速縮小與專有模型的差距，正在接近前沿

> 「隨著更好的開源模型越來越出色，我們相信可以讓世界上每個角落的任何人更容易獲得智慧。」

## 三大擴展維度

影片將模型的發展歸納為三個擴展維度：

| 維度 | 描述 | 對 Agent 的意義 |
|------|------|----------------|
| **Token 效率** | 用同樣的訓練 tokens 達到更低的 loss | 更強的先驗，Agent RL 搜索更高效 |
| **長上下文** | 增加上下文長度，支援更長任務 | Agent 運行時間可達數天甚至數週 |
| **Agent Swarm** | 多個 Agent 並行協作完成複雜任務 | 增添全新維度的擴展能力 |

**最終願景**：每個 Agent 都具備超長上下文和極強先驗，共同構成一個龐大的 Swarm 系統。

### 維度一：Token 效率

#### 為什麼 Token 效率不只是效率？

Token 效率的提升**同時也提升了智慧的上界**。原因：

- 假設有 50 兆高品質 tokens，使用新的 Mewen 優化器可獲得 2 倍 token 效率
- 這相當於魔術般地獲得了 100 兆 tokens
- 當我們正面臨「資料牆」（data wall），高品質資料有限時，提升效率就等於提升智慧

#### Mewen 優化器

Mewen 是一個**二階優化器**，核心思想：每次梯度更新時，每個參數都與其他參數正交。

- 傳統 Adam 優化器：梯度更新可能相互衝突
- Mewen：確保每次更新相互正交，最大化資訊利用

**論文貢獻**：Moonshot 是**第一個**展示 Mewen 優化器可擴展至 LLM 訓練的團隊。

#### 兩項關鍵技術

**1. Way Decay（路徑衰減）**

對扩展到大模型至關重要。

**2. 一致的 RMS 更新**

引入一個可調整係數，讓更新後的 RMS 與 Adam 相當。

#### 規模化挑戰：訓練不穩定性

當扩展 Mewen 到**1 兆參數模型**時，遇到了新的訓練不穩定問題：

- Max logits 迅速爆炸，超過 1000（正常值約 50-100）
- 訓練 loss 先下降，最後爆炸，無法收斂

**解決方案：QK Clip**

對於每個注意力頭，在前向傳播中：
1. 計算 max logits
2. 計算一個除數因子，應用於 key projection 和 query projection
3. 將 max logits 限制在一個給定範圍內

**效果**：
- 左圖：加入 Clip 前後，訓練 loss 曲線完全重疊——不影響收斂
- 右圖：max logits 被有效約束——先爆炸，但在值為 100 時被 clip 成常數，之後自然下降

**成果**：成功將 Mewen 規模化至 1 兆參數——這是機器學習歷史上**首個大規模 Mewen 訓練案例**。

### 維度二：長上下文

#### 為什麼 Transformers 比 LSTM 更適合長上下文？

右圖顯示：隨著 token 索引增加，Transformers 的 loss 持續下降，而 LSTM 在某個點之後就饱和了。

這就是為什麼 10 年前用 LSTM 做機器翻譯，卻無法用 LSTM 理解整個程式碼庫、或運行超長的 Agent 軌跡從頭寫 Linux 核心。

#### Kimmelinia 架構

引入一種新架構，包含 **Kimmel Delta Attention**（改進自 GDR），透過**細粒度衰減因子**解決線性注意力的表達能力問題。

**核心問題（線性注意力的原有缺陷）**：
- 記憶是全域的，只有單一衰減因子
- 兩種極端：要么忘記一切，要么保留一切——沒有中間地帶

**Kimmel Delta Attention 的創新**：
- 衰減因子從 scalar 變成**對角矩陣**
- 每個 channel 有獨立的衰減率
- 某些 channel 衰減極慢（保留長程上下文），某些快速衰減（吸收新資訊）

#### 實測結果

| 任務類型 | Kimmelinia 表現 |
|----------|-----------------|
| 短上下文（MMU） | 優於 MLA 和 GDN |
| 長上下文（Ruler） | 勝過其他變體，且更高效 |
| 擴展到 100 萬 tokens | 顯著優於所有 baselines |

**重要意義**：這是**首個在短上下文、長輸入任務和長輸出任務上全面超越全注意力的架構**。

### 維度三：Agent Swarm

#### 與單一 Agent 的對比

Agent Swarm  paradigm 就像人類社會的分工協作：

- **Orchestrator（協調者）**：類似 CEO，負責分解任務、分配給不同角色
- **Sub-agents**：AI 研究員、網頁開發者、物理研究員等，各自並行研究不同主題
- **匯集階段**：收集結果，啟動事實核查員、網頁開發者、文件下載者等子代理，將結果組裝成單一報告

```
複雜度（Y軸） vs 執行時間（X軸）
Agent Swarm 大幅降低執行時間，提升效率
```

#### 可擴展的方向

| 維度 | 說明 |
|------|------|
| **輸入擴展** | 並行下載閱讀數百甚至數千個來源 |
| **輸出擴展** | 並行寫作 100 頁文獻綜述 |
| **行動擴展** | 並行對 10 個不同任務執行數據分析 |
| **編排擴展** | 學習如何設計子任務、聚合結果 |

#### 三項強化學習目標函數

傳統 Single Agent RL 只有 outcome reward，Agent Swarm 新增了兩個目標：

**1. 實例化獎勵（Instantiation Reward）**
- 激勵子代理的實例化，防止「序列化崩潰」（serial collapse）
- 避免默認退化成單一代理執行
- 訓練初期權重高，後期逐漸衰減

**2. 完成獎勵（Finished Reward）**
- 觀察到有些子任務被創建但從未完成
- 激勵每個子任務都有較高的完成率
- 避免「偽造」第一個獎勵——只 spawn 一堆子代理但任務太複雜或根本沒意義

**3. 標準 Outcome Reward**
- 衡量整個任務是否完成

## K2.5 模型亮點

K2.5 是整合三個維度技術的成果，於一個多月前發布。

### 史詩級訓練曲線

> 「這是我一生中見過最美的曲線之一。」

K2.5 的訓練過程：
- 預訓練階段：15 兆 tokens
- K2.5 額外訓練：另外 15 兆 tokens
- **極度穩定**：完全沒有 loss spike，尤其在使用 Mewen 優化器後
- 平穩的訓練過程帶來了強大且穩定的基础模型

### 原生視覺-文字融合

**創新之處**：K2.5 是**首個原生 jointly vision-text 能力的開源模型**。

| 對比 | 過去的方法 | K2.5 的方法 |
|------|-----------|-------------|
| **名稱** | Late Fusion（後融合） | Early Fusion（早融合） |
| **做法** | 先訓練 20 兆 tokens 文字模型，再加 2 兆 tokens 視覺 post-training | 從第 0 天起就合併視覺和文字 tokens |
| **效果** | 視覺能力附加於文字之上 | 兩種 modality 深度融合 |

### 跨模態增強效應

**驚人發現**：視覺和文字兩種模態可以**相互增強**。

> 「很長時間以來，大家認為把視覺能力加到文字模型會損害文字效能。但我們發現，如果你訓練得當，這兩種模態其實可以相互增強。」

**視覺 RL 提升文字**：只用視覺任務（無任何文字任務）做 RL，文字任務效能也提升了。

**文字提升視覺（Zero Vision SFT）**：
- 完全沒有視覺 SFT 資料
- 只有文字 SFT 資料
- 透過 Joint RL，最終在視覺任務上達到幾乎 SOTA 的效能

## 下一代架構預覽：Attention Residue

這是昨天在技術報告中發布的新架構，算是對下一代模型技術的預覽。

#### 動機：將時間維度的技術應用到深度維度

ResNet 的故事（2016 ICML）：
- 在 ResNet 之前，沒人能訓練深層網路——梯度消失、訓練不穩定
- ResNet 引入後，你可以堆疊任意層數而不必擔心穩定性
- Ilya 的觀點：Residue connection 是 LSTM 的變體，只是旋轉了 90 度

**LSTM（時間維度）**：
- 從上一步獲取 hidden states
- 透過 gating mechanism 產生當前 states

**Residue Connection（深度維度）**：
- 從上一層獲取 outputs
- 應用函數產生當前層 outputs
- 只是用固定的加法替代了 LSTM 的 gating

#### 創新：Attention Residue

**核心問題**：能否將 LSTM 替換成 Attention？

**解決方案**：
- 不只取最後一個 hidden state
- 考慮所有之前的 hidden states
- 用 Attention 機制聚合這些狀態來計算當前層輸出

這就是 **Attention rotated by 90 degrees**——是 Residue Connections 在 LSTM 类比下的自然推廣。

#### Block Attention Residue（效率優化版）

為降低基礎設施開銷（通訊和記憶體）：
- 將網路分為多個 block（如每 block 16 層）
- 只在每個 block 的輸出上應用 Attention Residue
- block 內部仍使用標准 Residue

#### 驚人成果

| 指標 | 成果 |
|------|------|
| Token 效率提升 | **+24%**（50 兆 tokens 魔法般變成 60 兆） |
| Validation Loss | 一致低於原始曲線 |
| 特定任務提升 | GPQA、Math、HumanEval 等 coding、math、reasoning 任務最佳 |

## 總結

Moonshot AI 的核心策略：

```
Adam (2014) → Mewen (開源替代方案) → +24% token 效率
Attention (8年前) → Kimmelinia (線性注意力) → 全方位超越全注意力
Residue → Attention Residue → 開源實現
```

**研究心態的轉變**：10 年前主要是發表新想法，實驗嚴謹度不足很難產生可靠的結果。但現在有規模化的階梯、有完整的基準測試，更容易得出有信心且堅實的結論。這也是我們在古老技術上持續看到新進展的原因。

**展望未來**：將繼續在三個維度上擴展模型。Agent Swarm 不是終點，我們將持續看到新的擴展維度。很高興能與整個開源社區一起邁進，共同實現越來越出色的智慧。
