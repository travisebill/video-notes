## 一、主題與背景

FAI 2.3「自動學習模擬人類決策過程的模型」是陳縕儂教授 Machine Learning 章節介紹 Decision Tree 的第一講，約 36 分鐘。本講正式介紹第一個具體的非線性模型——Decision Tree——從人類「做決策」的直覺出發，模擬專家系統的 if-and-else 邏輯。

Decision Tree 的歷史定位與 1980 年代「第二次 AI 寒冬」密切相關。1960-70 年代 Expert System 曾是 AI 的頂峰——領域專家把 if-and-else 規則寫進程式，讓機器自動判斷。但 Expert System 不夠通用，因為人類很難把所有情況都寫進 SOP；寫不完的 case 還是需要真人處理，無法完全自動化，於是 1980 年代進入第二次 AI 寒冬。Decision Tree 從這個寒冬崛起：它「從資料裡學出 if-and-else 規則」——既像專家系統（學出來可以寫成 if-else），又從資料學（比人手寫更 general、更全面），於是逐漸取代了傳統 Expert System。

陳縕儂教授用「同學決定要不要寫作業」的決策樹做開場：deadline 還有多久 → 題目數量 → 有沒有朋友寫完 → 寫/不寫。這棵樹的每個 internal node 都是「條件」，每個 leaf 是「決定」。如果收集到「這些學生寫作業 / 不寫作業」的歷史資料，就能從資料裡學出這棵樹。

本講的另一條主軸是 **Recursive Formulation**。Decision Tree 可以寫成兩種形式：(1) Path-based：每條 path 對應一個 hypothesis $g_t(x)$ 加上一個 indicator $q_t(x)$（這條 path 是否 match），整個樹 = 所有 path 的加總。(2) Recursive：$G(x)$ 從 root 開始，看 branching criteria 決定走哪個 subtree，然後 recursive 展開。Recursion 形式讓演算法變得非常直覺——base case（什麼時候停）、branching criteria（從哪裡切）、subtree（剩下的部分）三個要素。

本講重點介紹 **CART（Classification and Regression Tree）** 演算法的設計選擇。CART 的關鍵決定：(1) 每次切兩半（binary tree）；(2) base hypothesis $g_t(x)$ 用 majority（分類）或 average（回歸）；(3) branching criteria：選一個 dimension + 一個 threshold，把資料切開；(4) terminate criteria：全部分到每個 leaf 只有一個 data（fully-grown）或達到某個 performance threshold。

Decision Tree 有幾個重要優點：(1) 可解釋性極高——切完之後可以看到「哪些 dimension 重要、什麼 threshold 適合」；(2) 對 categorical feature 友善——不需 encoding 直接處理；(3) 對 missing feature 友善——某個 dimension 沒值就跳過這個分支；(4) 訓練與測試都很快——訓練是 divide-and-conquer，測試就是 if-else。Decision Tree 的缺點：(1) 容易 Overfitting——切太細就把 noise 學進去；(2) 不容易 model feature 之間的 interaction——每個 dimension 獨立切；(3) 對類別多的 categorical feature 有偏誤（傾向用該 feature 切）。

**核心議題一句話**：Decision Tree 是第一個具體的非線性模型——模擬人類決策的 if-and-else 邏輯，從資料自動學出來；用 CART 演算法的 recursive formulation（branching criteria + base case + subtree），具有可解釋性強、可處理 categorical/missing feature 等優點，但容易 Overfitting——這為 FAI 2.4（Random Forest）埋下伏筆。

## 二、章節脈絡

### Section 1｜AI 第二次寒冬與 Expert System 限制（00:00 ~ 03:00）

**重點摘要：** Expert System 在 1980 年代達高峰但因不夠通用進入第二次 AI 寒冬；Decision Tree 從這個寒冬崛起，從資料學出 if-and-else 規則。

**內容：**
- AI 歷史高峰：Expert System（1980 年代）
- Expert System 做法：領域專家寫 if-and-else 規則
- 限制：很難把所有情況都寫進 SOP；寫不完的 case 還是需要真人
- 結論：無法完全自動化，於是進入第二次 AI 寒冬
- Decision Tree 崛起：用 ML 從資料裡學模擬專家系統
- 學出來的結果像專家系統（可以寫成 if-else），又從資料學（更 general）
- 從第二次寒冬開始，Exponential 成長到現在

> "雖然直接找這一個專家就直接讓專家去寫這個 if and else 好像不夠通用，好像有點困難，但有沒有辦法用一個 machine learning 的方法讓他去學，我們專家通常是怎麼在裡面做判斷的。"

### Section 2｜寫作業 Decision Tree 範例（03:00 ~ 07:00）

**重點摘要：** 用「要不要寫作業」決策樹做開場：deadline → 題目數量 → 朋友進度；Decision Tree 可從學生的歷史資料學出來。

**內容：**
- 情境：今天出作業三，三週 deadline
- 同學腦中的決策樹：
  - deadline 大於兩週？→ 不寫（no）
  - 一到兩週？→ 看題目數量
    - 大於 10 題？→ 開始寫（yes）
    - 小於 10 題？→ 不寫
  - 小於一週？→ 看朋友進度
    - 有朋友寫完？→ 開始寫（yes）
    - 沒朋友寫完？→ 不寫
- 從學生「要不要寫作業」的歷史 data，可以學出這棵樹
- 學出來就像專家系統，直接跑 if-else 就好
- 關鍵：人說不出來自己怎麼決策的，但 data 學得出來

> "根據你要不要寫作業這些 data，說不定我們是可以學出這一個 decision tree 的，而且學出來這個 decision tree 之後，它就像是一個專家系統。"

### Section 3｜Decision Tree 數學式：Path-based 形式（07:00 ~ 12:00）

**重點摘要：** 每個 leaf 是 base hypothesis $g_t(x)$；path condition $q_t(x)$ = 0/1 indicator；整個樹 = 所有 path 的加總；T = 總 path 數。

**內容：**
- Base hypothesis $g_t(x)$：每個 leaf 的輸出值（決定 / 分類）
- Path condition $q_t(x)$：這個 $x$ 是否滿足這條 path 的 condition（1 或 0）
- 整個 tree：$G(x) = \sum_{t=1}^{T} q_t(x) \cdot g_t(x)$
- 當 $x$ 滿足某條 path 時，用對應的 $g_t(x)$；不滿足時 $q_t(x)=0$
- 每個 leaf 都是「滿足某一組條件」的 $x$ 集合
- 這就是 Decision Tree 的 Path-based 寫法

> "其實這個就是一個每一條 path 的 transform，所以我們可以簡單的把它把這一個模型這個 decision tree 寫成這個樣子。"

### Section 4｜Decision Tree Recursive 形式（12:00 ~ 17:00）

**重點摘要：** $G(x) = b(x) + \sum_{c} b(x, c) \cdot G_c(x)$；$b(x)$ 是 branching criteria；$G_c(x)$ 是第 c 個 subtree；這是 divide-and-conquer 的 recursive 寫法。

**內容：**
- 換個角度想：$G(x)$ 從 root 開始
- $b(x)$：branching criteria（哪個條件要分開）
- 如果 $b(x, c)$ 滿足（branching criteria = c）→ 展開成 subtree $G_c(x)$
- 每個 subtree 也是一棵 tree，recursive call
- 整棵 tree = $b(x)$ + 多個 subtree 的加總
- 這是 recursive 的 divide-and-conquer 演算法
- 對應到程式：recursive function，base case 要寫出來（不然會 segmentation fault）

> "這個 recursive tree 然後就是從 root 的開始，然後變成它的 subtree 一個一個這樣長開。"

### Section 5｜CART 演算法的四個關鍵決定（17:00 ~ 24:00）

**重點摘要：** 切幾個 branch（CART 用 binary, C=2）；base hypothesis（majority 分類 / average 回歸）；branching criteria（dimension + threshold）；terminate criteria（fully-grown 或 threshold）。

**內容：**
- 整個 Decision Tree algorithm 包含四個 choice：
  1. C：每次切幾個 branch（CART 用 2, binary）
  2. $g_t(x)$：base hypothesis（分類用 majority，回歸用 average）
  3. branching criteria：要從哪個 feature 切（dimension + threshold）
  4. terminate criteria：什麼時候停止（fully-grown 或 performance threshold）
- 不同的 choice 對應不同的演算法：CART、C4.5、ID3 等
- 這就是 Decision Tree 不是單一演算法，而是一個「category」的原因
- CART 的具體設定：binary tree + simple $g_t$ + dimension/threshold branching

> "這就是 decision tree 這整個 category，decision tree 不是某一個演算法，而是有很多演算法，只要符合這樣子的概念，它都是叫做 decision tree。"

### Section 6｜CART 在 2D 空間的示意（24:00 ~ 29:00）

**重點摘要：** 2D 二元分類：CART 只能用橫線或直線切；先找一個 dimension + threshold 切一刀，分開後對剩下的 recursive 再切。

**內容：**
- 2D 二元分類：只有兩個 dimension
- CART 只能找橫線（horizontal）或直線（vertical）切——因為是選 dimension + threshold
- 不能斜切
- 過程：
  1. 找一個 dimension + threshold 切一刀
  2. 對分開後的 subset，recursive 再切
- 例如：先切直的，左邊都是 O，右邊還有 mix → 對右邊再切
- 對右邊再切橫的或直的 → 完全分開
- 總共四次 branching，四個 condition
- 這就是 divide-and-conquer 的視覺化

> "在這個圖上面我只能找橫的線或者是直的線，不能斜的，一定要橫的或直的，因為我就是找一個 dimension。"

### Section 7｜Decision Tree 的優點（29:00 ~ 33:00）

**重點摘要：** 可解釋性高；對 categorical feature 友善（直接處理不需 encoding）；對 missing feature 友善（跳過該 dimension 不影響）；訓練測試都快。

**內容：**
- 可解釋性高：學出來可以看哪些 dimension 重要、threshold 多少
  - 例：年收入大於多少 → 大部分繳款正常
- 對 categorical feature 友善：例如「作業是計算題 / 證明題 / 程式題」
  - 其他演算法（Linear、SVM）需要 encoding（dummy variable）
  - Decision Tree 直接處理——這個 feature 屬於哪個 category 就這樣切
- 對 missing feature 友善：某個 dimension 沒值，跳過就好
  - 其他演算法需要 imputation（補 0、補平均等）
  - Decision Tree 完全不用處理
- 訓練與測試都快：訓練 divide-and-conquer，測試直接 if-else

> "如果你是用 decision tree，你完全不用管它這邊 missing，沒差因為你每一個 dimension 是各自獨立判斷的。"

### Section 8｜Decision Tree 的缺點（33:00 ~ 36:08）

**重點摘要：** 容易 Overfitting（切太細）；對類別多的 categorical feature 有偏誤；不能 model feature 間的 interaction（每個 dimension 獨立切）；其他演算法需要不同策略處理。

**內容：**
- 缺點 1：容易 Overfitting
  - 切切切切 → 長出很大的樹
  - 太深的樹 focus 在 noise 上
  - 需要 regularization 或 pruning（FAI 2.4 主題）
- 缺點 2：對類別多的 categorical feature 有偏誤
  - 該 feature 類別很多時，演算法傾向一直用該 feature 切
  - 不一定 reasonable
- 缺點 3：不能 model feature 間的 interaction
  - 每個 dimension 獨立切
  - 例：feature A 跟 feature B 一起才有資訊，但 Decision Tree 切不開
- San Francisco vs New York 房價分類實例：training accuracy 100% 但 testing 89.7%（Overfit 約 10%）
- 啟示：需要方法減緩 Overfitting（下一講 Random Forest）

> "這個 feature 因為它是每一個 dimension 是不是各自切開，所以每一個 dimension 它是獨立判斷的，所以它其實沒有辦法 model 說 feature 跟 feature 之間的資訊。"

## 五、核心主旨

> Decision Tree 是第一個具體的非線性模型——從「同學寫作業」的日常決策切入，模擬人類 if-and-else 思考邏輯。數學上有兩種寫法：Path-based（每條 path 加總）與 Recursive（root + 多個 subtree）。CART 演算法的核心選擇是 binary tree + simple base hypothesis + dimension/threshold branching + terminate criteria。Decision Tree 有可解釋性強、可處理 categorical/missing feature、訓練測試都快等優點，但容易 Overfitting、不能 model feature 間 interaction、對類別多的 categorical feature 有偏誤。Decision Tree 不是單一演算法而是一個 category——CART、C4.5、ID3 等都是。Decision Tree 的 Overfitting 問題將在 FAI 2.4 用 Random Forest（ensemble）解決。
