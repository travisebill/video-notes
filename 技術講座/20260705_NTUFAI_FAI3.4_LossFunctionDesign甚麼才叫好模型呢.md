## 一、主題與背景

FAI 3.4「甚麼才叫『好』模型呢？Loss Function Design」是 FAI 第三章中一段短而關鍵的銜接講，處理深度學習中的「Loss Function」，回答的核心問題是：「當模型 g(x) 對真實函數 f(x) 做預測時，要用什麼樣的指標來量化『預測有多差』，以便為後續最佳化提供方向？」

林軒田教授首先區分 Loss Function 與 Error Measure。Error Measure 是評估模型好壞的最終指標；Loss Function 則是實際餵給最佳化演算法的代價函數，兩者在多數情境下會被設計為近似等價。Loss Function 的本質，是測量模型預測輸出與真實標籤之間的差距，並把多筆訓練樣本的平均差距彙整成一個數值，作為當前參數組合下模型表現的總評分。

為了讓 Loss Function 能被最佳化處理，所有「模型的參數」會被整併為一個向量 θ，從輸入層、隱藏層、到輸出層的每一組權重 w 與偏置 b 都被收納其中。因此，選擇一個模型假設 g，就等於從這個高維參數空間中選一組 θ。最優的 θ 就是那組讓 In-Sample Error 最小化的參數組合。

接著，影片用一個視覺化的方式呈現最佳化目標：將 Loss 視為 θ 空間中的一張高低起伏的地形圖，最低谷就是「最好的模型」。最佳化的任務，就是在地形圖上找到這個最低點；而 Loss Function 的設計是否合理，將直接決定這張地形圖的形狀——設計得太過平坦，最低點附近的梯度訊號微弱，最佳化容易卡住；設計得過於崎嶇，則容易把最佳化引入局部低谷，難以觸及真正的全域最佳解。

整段影片並未深入介紹實際的 Loss Function（如 MSE、Cross Entropy、Hinge Loss 等），而是先建立 Loss Function 在最佳化流程中的角色定位——它是 Loss 地形的「設計者」，決定後續 Gradient Descent 在什麼樣的地形上工作。

**核心議題一句話**：Loss Function 不只是一個評分工具，更是整個深度學習最佳化流程的「地形設計者」；它的好壞，將決定模型能不能被有效訓練，也決定最終模型的泛化表現。

## 二、章節脈絡

### Section 1｜Loss Function 與 Error Measure 的關係（00:00 ~ 01:30）

**重點摘要：** Error Measure 是評估指標；Loss Function 是餵給最佳化演算法的代價函數；兩者常被設計為近似等價。

**內容：**
- 定義這個 Error Measure = Loss Function
- Loss Function 其實跟 Error Measure 是一樣的東西
- 它就是在定義模型的 g(x) 跟真實 f(x) 有多接近
- 也就是模型預測的 y 值跟真實收集到的 y 值差異是多少
- Loss 越低越好
- Loss、Cost、Error 都可以視為同樣一件事情
- 希望這個東西可以 minimize

> "Loss function其實跟Error Measure是一樣的東西，所以我們常常會講的叫做loss function。"

### Section 2｜模型參數整併為 θ（01:30 ~ 03:00）

**重點摘要：** 模型參數包含所有層的 w 跟 b；整併為向量 θ；選模型 g 就是從高維空間中選一組 θ。

**內容：**
- 模型可變動的就是模型的參數
- 參數就是 w 跟 b
- 每選一組 w 跟 b 的值就得到一個假設 g
- 多組可能的參數 assignment 對應多個不同的假設
- 為了方便，把這些參數整併寫成向量 θ
- θ 內含每一層的 w 跟 b
- 選一個模型假設 g 等於選一組 θ

> "我們可以變動的這些東西，這是模型的參數...選一個假設g其實就是在選一組w跟b的這些值，OK就是這個θ的值。"

### Section 3｜Loss 視為 θ 空間的地形圖（03:00 ~ 04:17）

**重點摘要：** Loss Function 給一個 θ 可以算出 Loss 值；整個 Loss 形成地形圖；最佳化目標是找到地形圖的最低谷。

**內容：**
- 一個好模型的定義：g 模型產生的輸出與 target y 足夠接近
- 計算每筆樣本的 y 跟 g 的差距，平均後就是 E_in
- 將 Loss Function 跟參數 θ 結合：得到 θ 帶進 Loss 後的數值
- E_in 就是參數帶進去之後得到的 g 的預測值
- 最佳化目標：minimize 這個 E_in
- 把 Loss 視為 θ 空間中的高低起伏地形圖
- 最低谷就是最好的模型
- Loss Function 設計是否合理，決定地形圖的好壞
- 直接影響最佳化是否容易成功

> "一個好的模型的定義是不是就是我們g這個模型產生出來的這個output的值會跟我們target的這個收集到的這個y值是足夠接近的...最好的模型就是能夠minimize這邊的這個E_in就是能夠把這個in-sample的error能夠壓到最低的那一個參數就是一個最好的模型。"

## 五、核心主旨

> Error Measure 與 Loss Function 在多數情境下被設計為近似等價，但前者是評估模型好壞的最終指標，後者是實際餵給最佳化演算法的代價函數。所有模型的參數——輸入層、隱藏層、輸出層的每一組權重 w 與偏置 b——都會被整併為一個高維向量 θ，因此選擇一個模型假設 g，本質上就是從 θ 空間中選取一組向量。最佳化的目標，便是 minimize 這組 θ 對應的 E_in。把 Loss Function 視為 θ 空間中的高低起伏地形圖，最低谷就是最好的模型；地形圖的形狀完全由 Loss Function 的設計決定，會直接影響後續 Gradient Descent 在此地形上能否順利抵達谷底，也決定最終模型的泛化表現。Loss Function 既是評分工具，也是整個深度學習最佳化流程的「地形設計者」。
