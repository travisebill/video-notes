## 一、主題與背景

台大資訊工程學系 2026 年春季「人工智慧導論（FAI）」正式開課，這是 FAI 1.1 第一講「監督式機器學習之線性模型」。陳縕儂教授用約10 分鐘的時間，從課程整學期規劃（schedule）切入，介紹 Machine Learning 與上半學期學過的傳統 AI（Tree Search、Knowledge Representation）的銜接關係，最後透過一個簡單但經典的 Lib-SVM Demo 讓學生具體感受「分類器（classifier）」在做什麼。

本講的核心實作示範是 libSVM 的 2D 平面互動 Demo：學生可以點不同顏色的點，讓 SVM 自動找一條區隔線。Demo 刻意展示兩種對比——把模型複雜度調高，會得到一條「不自然」但把所有訓練點都分對的曲線；把模型改回 Linear（複雜度歸零），反而得到一條「簡單但更接近真實分佈」的直線。這個 Demo 預示了整個 ML 章節的核心張力：bias-variance tradeoff、overfitting 與 generalization——這些會在 FAI 1.4「機器要開始學習囉」深入展開。

**核心議題一句話**：Machine Learning 不是要發明新的演算法，而是要用「資料」與「模型」這兩個簡單元素，搭配搜尋演算法，從無限多的候選函數中挑出最接近真實分佈的那一個。

### Section 2｜Lib-SVM Demo 介紹（02:00 ~ 05:00）

**重點摘要：** libSVM 是歷史悠久、類似 PyTorch 知名度的工具；用它做 2D 點分類的視覺化教學。

**內容：**
- Lib-SVM 是「Library SVM」之意，知名程度類似今天的 PyTorch
- Demo 流程：在黑底畫面上點不同顏色的點（黃色 / 藍色 / 紅色），libSVM 會即時畫出分類線
- 2D 平面呈現：把點視為 (x, y) 二維空間的點，目標是找一條曲線（或直線）把不同顏色分開
- 學生可以自己玩玩看：點夠多就會看到決策邊界如何變化
- 工具連結會放在課程網頁上（libsvm 官方 demo 頁）

> "這個呢是林軒田老師他們開發的一個工具叫做 libSVM…在 SVM 的世界就是 libSVM 是非常非常非常有名，就是最有點像是現在大家經過的比如說 PyTorch 一樣這種知名度。"

### Section 3｜複雜度過高的分類器 = 死背考古題（05:00 ~ 08:00）

**重點摘要：** 過於複雜的模型會在訓練資料上完全分對，但對新資料反而表現差——overfitting 的直覺。

**內容：**
- Demo 第一輪：把模型複雜度調高，得到的分類線「非常不自然」——把所有訓練點都分對，但形狀奇異
- 為什麼會這樣？模型在「死背考古題」：每個訓練點都對，但學不到真正的概念
- 學生類比：考古題寫很多遍之後，相同題目會對，但變形題就錯——因為沒真的理解概念
- 真實資料常有 noise：例如一張貓的圖被標成狗（標籤錯誤），如果模型 100% 相信標籤，就會學到奇怪的分類邊界
- Demo 第二輪：把模型複雜度調回 0（Linear），分類線變成一條直線，雖然有幾個點分錯，但更接近「通用」分類
- 結論：**簡單的模型可能比複雜模型在真實資料上表現更好**——這是 ML 的核心直覺

> "模型它設定得很複雜的時候，它有可能就為了要能夠讓我的訓練資料看到這些資料上面分了對…就像是你看到很多考古題你可能還沒有完全了解它為什麼這個答案是這個，然後你就硬自把它死背起來。"

### Section 4｜Linear Model 為什麼簡單有效（08:00 ~ 09:49）

**重點摘要：** Linear Model 在 2D 平面就是一條直線；簡單、可解釋、適合當 baseline。

**內容：**
- Linear Model 的幾何意義：2D 平面上一條直線，3D 平面上一個平面，n 維空間一個 hyperplane
- 把 libSVM 設定調回 Linear，邊界就變回直線
- 學生可在 Demo 上點更多黃色點，會看到直線位置移動
- Linear Model 是後續所有 ML 方法的 baseline：先試 linear，不行再升級到 non-linear
- Demo 也支援 3 個類別（黃 / 藍 / 紅）——多類別分類
- 影片最後陳縕儂教授呼應「同學要記得 4/29 有 Final Project Meeting」的提醒

> "Linear Model 它就是一個很簡單的分類的方式，那 Linear 甚至在這邊都不是這樣寫的，如果是 2D 的平面上面，我們的線性的分類器它就會是一條直線。"

## 三、關鍵概念定義表

| 概念 | 定義 | 出處 / 應用 |
|------|------|-------------|
| Linear Model | $\mathbf{w}^T \mathbf{x} + b$ 形式的分類器；2D 為直線 | 本講核心 |
| Classifier | 分類器：給定 input $\mathbf{x}$ 輸出類別 $\hat{y}$ | Lib-SVM Demo |
| Hypotheses Set | 候選函數集合 $H$，ML 要從中選最好的 $g$ | FAI 1.3 詳細介紹 |
| Overfitting | 模型在訓練集表現好、測試集表現差（過度擬合） | 本講 Demo 預示 |
| Underfitting | 模型太簡單，連訓練集都學不好 | FAI 1.4 詳細介紹 |
| libSVM | Chih-Chung Chang 與林軒田老師開發的 SVM 函式庫 | 本講 demo 工具 |
| Tree Search | 傳統 AI 的搜尋方法（A*, DFS, BFS, Minimax） | 上半學期 |
| Knowledge-Driven AI | 以人類專家規則為核心的 AI 典範 | 上半學期 |
| Data-Driven AI | 以資料 + 模型為核心的 AI 典範 | 下半學期 |
| Baseline | 對照組模型，作為後續改進的參考起點 | Linear Model 通常當 baseline |

## 五、核心主旨

> 監督式機器學習的核心問題是：給定無限多可能的候選函數集合 $H$，如何從中找出一個 $g$ 接近真實的 target $f$？Linear Model 是這個問題的最簡單起點——它在 2D 平面就是一條直線，在 n 維空間就是一個 hyperplane。複雜度高的模型可能在訓練資料上完全分對，但對新資料反而表現差（overfitting）；簡單的 Linear Model 雖然訓練上有錯，卻更接近「通用」的真實分佈。這也是為什麼業界的 ML 實踐永遠從最簡單的 baseline 開始——先試 Linear，再考慮升級。
