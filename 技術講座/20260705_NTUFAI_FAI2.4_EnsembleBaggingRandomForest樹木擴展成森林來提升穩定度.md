## 一、主題與背景

FAI 2.4「樹木擴展成森林來提升穩定度」是陳縕儂教授 Machine Learning 章節介紹 Ensemble 第一講，約 19 分鐘。本講用一句老話開場：「三個臭皮匠勝過一個諸葛亮」——這就是 Ensemble 的核心精神。多個弱模型組合起來，可以達到比單一強模型更好的效果，這在工業界已經是 ML 的標準做法。

Decision Tree（FAI 2.3）雖然有可解釋性強、可處理 categorical/missing feature 等優點，但最大的問題是 **容易 Overfitting**——切切切切切就把 noise 學進去。為了解決這個問題，陳縕儂教授在 FAI 2.4 引入 **Bagging**——一種專門用於「易 Overfitting 模型」的 Ensemble 方法。

Bagging 的核心思想是 **Variance Reduction**。回憶 FAI 2.2 的 Bias-Variance Trade-off：模型複雜度 ↑ → bias ↓ 但 variance ↑。當我們已經包含到理想 $f$ 但 variance 太大時（就是 Overfitting 發生時），我們的目標是 **降低 variance**。Bagging 的做法很直觀：對同一個複雜模型，訓練 T 次（每次用不同的 random subset of data），得到 T 個 $g_t$；最後用 T 個 $g_t$ 的「平均」（regression）或「投票」（classification）作為最終預測。

為什麼 averaging 可以降低 variance？數學上，如果 T 個 $g_t$ 的 variance 都是 $\sigma^2$，它們的「平均」的 variance = $\sigma^2 / T$。換言之，跑的次數越多，平均起來越穩定，variance 越小。Bagging 把這個直覺應用到 ML：把 training data 分成 T 個 random subset（每個 sample N' 個），各訓練一個 model（同一個 algorithm），最後 average 起來。這個過程可以平行化（每個 tree 獨立生長），所以非常 efficient。

Bagging 對 Decision Tree 的應用就是 **Random Forest**：tree + tree + ... + tree = forest。Random Forest 是 Bagging 的特例——base learner 用 Decision Tree，而且每棵 tree 用 random sample 的 subset 訓練。Random Forest 保留了 Decision Tree 的所有優點（可解釋性、categorical/missing feature），但透過 ensemble 顯著降低 variance，緩解 Overfitting 問題。

Random Forest 的 **Voting / Averaging** 機制：分類問題用多數決（10 棵 tree 中 8 棵說 San Francisco 就說 San Francisco）；回歸問題用平均（10 棵 tree 各 output 一個值，平均起來）。這個機制讓單棵 tree 的 Overfit 噪聲被「稀釋」——noise 很難 consistent 影響所有 T 棵樹，所以 ensemble 後 noise 的影響被壓低，而真正的 pattern 容易被保留下來。

Random Forest 還有幾個重要特性：(1) 容易平行化（每棵 tree 獨立生長）；(2) 對 feature 維度高的問題效果好（也可以從 feature 做 random sampling 而不只是 data）；(3) 是 FAI 2.x 「Nonlinear Models」系列的延伸；(4) 與 bias-variance 平衡的關係——透過大量樹的 ensemble，variance 下降但 bias 維持，可以逼近 testing error 的最低點。

**核心議題一句話**：Random Forest 是 Bagging + Decision Tree 的結合——用「三個臭皮匠勝過一個諸葛亮」的精神，把多棵容易 Overfit 的 Decision Tree ensemble 起來，用 voting/average 降低 variance 緩解 Overfitting；保留了 Decision Tree 的所有優點，且容易平行化、對高維度資料效果好，是工業界最常用的 ML 模型之一。

## 二、章節脈絡

### Section 1｜三個臭皮匠勝過一個諸葛亮：Ensemble 概念（00:00 ~ 02:00）

**重點摘要：** Ensemble = 多個弱模型組合 → 比單一強模型好；三個臭皮匠勝過一個諸葛亮；之前 final project 表現最好的組就是用很多簡單 model ensemble。

**內容：**
- 老梗：「三個臭皮匠勝過一個諸葛亮」
- Ensemble = 把多個模型組合起來
- 之前某門課的 final project：表現最好的組就是用很多簡單 model + ensemble
- 簡單但好用的方法，現實應用非常廣
- 本講聚焦 Bagging 跟 Random Forest

> "這是一個非常非常老梗的句子，就是三個臭皮這樣勝過一個諸葛亮。其實這還是一個非常非常好用，跟現在有非常多應用其實都還是可以借由 ensemble 再繼續 improve。"

### Section 2｜Bagging 的目標：Variance Reduction（02:00 ~ 04:30）

**重點摘要：** Overfitting 的特性：包含到 $f$ 但 variance 大；Bagging 的目標是 reduce variance；把多次實驗平均起來 → 接近 $f$。

**內容：**
- Overfitting 的情境：包含到理想 $f$ 但 variance 很大
- 對應打靶圖：瞄準了（bias 小）但每次射出去都散開（variance 大）
- 既然這樣，我們的目標是降低 variance
- 怎麼降？把多次實驗平均起來
- 例如打靶 5 次平均起來 → 可能很接近靶心 $f$
- Bagging = Bootstrap Aggregating：把大家都放到同一個袋子裡
- 找 T 個變異度大的 $g_t$，平均起來 → variance 降低

> "我們的目標就是下降 variance 對不對，怎麼下降 variance 很簡單嘛...如果我們把這 5 次平均起來，說不定其實就很接近這個 function f 的樣子。"

### Section 3｜Bagging 的做法：Random Sampling + Ensemble（04:30 ~ 07:30）

**重點摘要：** 從 N 個 data sample T 次（每次 N' 個）；各訓練一個 model；T 個 model 一起平均起來；complex model 透過 random sample 變得更 robust。

**內容：**
- 假設 data 有 1000 個 data point
- random sample T 次，每次取 N' = 800 個（可能不同 data points）
- 每個 subset 各訓練一個 model，得到 T 個 $g_t$
- T 個 $g_t$ 一起 average / vote
- 為什麼 random sample 可以讓 model 變 robust？
- 小的 noise 很難一致地影響所有 T 個 subset
- 透過 random sample，noise 在 averaging 時被抵消
- Robustness 提升，variance 下降
- 適用情境：模型本身容易 Overfitting（complex model）

> "我們的目標就是把它平均起來，然後就會得到一個比較好的 function，透過 reduce variance 之後呢，希望這一個 complex model 他會比較 robust。"

### Section 4｜Random Forest：Bagging + Decision Tree（07:30 ~ 11:00）

**重點摘要：** Forest = 多棵 tree；Random Forest = random sampling + decision tree；T 棵 tree 各學一個 $g_t$，最後 average 起來。

**內容：**
- Decision Tree 容易 Overfitting（切切切切切）
- Bagging 是解 Overfitting 的方法之一
- Bagging + Decision Tree = Random Forest
- Forest = 森林 = 多棵 tree
- Random：random sampling 不同的 subset
- 演算法：對 data 做 T 次 random sample → 每個 subset 跑 decision tree → 得到 T 個 $g_t$ → 最終 model = T 個 $g_t$ 的平均
- Classification 用 voting（多數決）
- Regression 用 averaging
- 簡單但有效

> "剛剛講到 decision tree 他有一個缺點是什麼，很容易 overfit 嘛對不對...所以我們把 decision tree 用 bagging 的方法來 ensemble，就有一個新的 algorithm 叫做 random forest。"

### Section 5｜Random Forest 的優點（11:00 ~ 14:00）

**重點摘要：** 容易平行化（每棵 tree 獨立生長）；保留 Decision Tree 所有優點；降低 variance 緩解 Overfitting；可以從 data 或 feature 做 sampling。

**內容：**
- 優點 1：容易平行化
  - 每棵 tree 可以獨立生長
  - 每個 subset 各自跑 decision tree
  - 學習時間 ≈ 跑一棵 tree 的時間
- 優點 2：保留 Decision Tree 優點
  - 可解釋性、categorical feature、missing feature、訓練測試快
  - 這些好處都在 random forest 保留
- 優點 3：降低 Overfitting
  - Fully-grown tree 每個 leaf 只剩一個 sample → 容易 Overfit
  - Random Forest 用 voting / average → 顯著降低 variance
  - noise 很難 consistent 影響所有 T 棵樹 → noise 被稀釋
- 優點 4：可以從 data 或 feature 做 sampling
  - data sampling：每次取不同 data subset
  - feature sampling：每次取不同 feature subset
  - feature sampling 在高維度資料特別有效

> "首先它是一個非常容易 parallel 的一個模型...因為現在我們每一棵樹都可以獨立生長嘛。"

### Section 6｜Random Forest 的 Decision Boundary 示意（14:00 ~ 16:30）

**重點摘要：** 1 棵 tree 的 boundary 很尖銳；6 棵 tree aggregate 後變平緩；T 越大 smoothness 越高；這就是 Variance Reduction 的視覺化。

**內容：**
- 1 棵 decision tree 的 decision boundary：很尖銳（橫切、直切）
- 6 棵 decision tree aggregate 後：boundary 變平緩
- T 越大 smoothness 越高（更多 tree 一起平均）
- 每棵 tree 切的位置不同（因為 subset 不同）
- Aggregate 後整體 boundary 是多棵 tree 的 consensus
- 視覺化：1 棵 vs 6 棵 vs 100 棵 → 越來越 smooth
- 這就是 variance reduction 的視覺化證明

> "如果你比較左邊單一一個 classifier，跟右邊其實 aggregate 多個 tree，它的不一樣的地方是什麼，你看它的 boundary 其實就從原本單一一個 decision tree 它是這種很尖銳的，漸漸就會變得平緩。"

### Section 7｜San Francisco vs New York 房價分類實例（16:30 ~ 18:47）

**重點摘要：** 單棵 decision tree：training 100% testing 89.7% → Overfitting 10%；Random Forest 4 棵 tree aggregate 後降低 variance；複雜度太高 → overfit，太低 → underfit，要平衡。

**內容：**
- 實例：San Francisco vs New York 房價分類
- Fully-grown decision tree：
  - training 100%（每個 leaf 只剩一個 sample）
  - testing 89.7%（差約 10%）
  - Overfitting 明顯
- Random Forest 4 棵 tree：
  - 每棵 tree 獨立訓練
  - 4 棵 tree aggregate 後降低 variance
- Bias-Variance 取捨再次出現：
  - x 軸：模型複雜度
  - bias 隨複雜度 ↑ 而 ↓
  - variance 隨複雜度 ↑ 而 ↑
  - 找 sweet spot
- 實務建議：不要太過分切（fully-grown），可以設 terminate criteria 提早停
- Model Selection 仍是非常困難的問題

> "所以這邊就取決於你要什麼時候讓這個數停下來，什麼時候 terminate，好像就不要全部都分到同一個 label，或是它這邊的比例多少你可能就可以讓它停止。"

## 五、核心主旨

> Random Forest 是 Bagging 與 Decision Tree 的結合——用「三個臭皮匠勝過一個諸葛亮」的精神，把多棵容易 Overfit 的 Decision Tree 透過 random sampling 訓練，再用 voting（分類）或 averaging（回歸）組合起來，顯著降低 variance 緩解 Overfitting。Random Forest 保留 Decision Tree 的所有優點（可解釋性、categorical/missing feature、快速訓練測試），同時透過 ensemble 大幅提升 robustness；數學上 T 個獨立模型的平均 variance = 原 variance / T。Random Forest 容易平行化、對高維度資料效果好、可用 data 或 feature sampling，是工業界最常用的 ML 模型之一。Bias-Variance 取捨在 random forest 中依然存在：複雜度 ↑ → bias ↓ 但 variance ↑；透過大量 tree 的 ensemble 維持 low bias 同時降低 variance，逼近 testing error 的 sweet spot。
