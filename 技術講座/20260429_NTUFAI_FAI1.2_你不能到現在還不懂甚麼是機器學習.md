## 一、主題與背景

FAI 1.2「你不能到現在還不懂甚麼是機器學習」是陳縕儂教授在台大資訊系「人工智慧導論」的 Machine Learning 第二講，約 26 分鐘。影片從「學習」這個日常詞彙出發，把人類學習（從經驗 / observation 累積 Skill）的過程對應到機器學習（從 Data 學到 Skill）的數學化版本。陳縕儂教授用約30 分鐘完整定義 Machine Learning 的三個核心要素：Observation → Skill 的轉換、透過 performance measure 驗證、必須有 pattern 可學。

本講的另一條主軸是「AI vs ML」的差異。陳縕儂教授從維基百科定義出發，把 AI 的三大能力（感知 Perception、合成 Generation、推論 Reasoning）對應到 ML 的應用場景——語音辨識（感知）、圖片生成（合成）、天氣預測（推論）。同時也明確劃清 AI 不一定要用 ML：傳統 AI 的 Game Tree Search（如圍棋的 AlphaGo）部分結合 ML、部分用傳統規則；專家系統（Expert System）也可以不用 ML。

最精彩的部分是用「Google Maps 餐廳評價分類」當例子，演示「為什麼簡單的 rule-based 方法會爆炸」。一開始寫「if 包含『love』就是正面」這種規則，看起來很直覺；但遇到 "Oh it's not bad"（負負得正）、"It's not good"（負）等情況，規則會越加越複雜，最後還是無法涵蓋所有例外。這精準說明了為什麼需要 ML：人類自己也很難講出完整的判斷規則，所以讓機器從大量 (input, output) 資料中自己學。

**核心議題一句話**：Machine Learning 的本質是「讓機器具備從資料中學習 Skill 的能力」——這個 Skill 必須可以透過 performance measure 量化驗證，而資料中必須存在可學習的 pattern。AI 與 ML 的關係是：ML 是實現 AI 的其中一條路徑，但不是唯一路徑。

## 二、章節脈絡

### Section 1｜什麼是學習：從 Observation 到 Skill（00:00 ~ 05:00）

**重點摘要：** 學習 = 透過經驗累積獲得 Skill；機器的學習則是把 Observation 換成 Data。

**內容：**
- 人類學習：透過寫程式、跑實驗、觀察輸出，從觀察中累積 Skill（programming skill、實驗設計 skill）
- 機器學習差異：機器沒有眼睛耳朵直接接收感知，所以是從 Data（資料）學
- 把人類的 Observation 完整收集起來變成 Data，讓機器在這些資料上做學習——這是 ML 的本質
- 圖示：人類 Learning = 大量 Observation → Skill；機器 Learning = 大量 Data → Skill（結構對應）

> "學習其實就是在各式各樣的經驗累積的情況下，然後透過這些經驗以及你觀察的這些內容去學習到這些 Skill。我們訓練機器其實也是很類似，我們的目標也是希望讓它具備這個 Skill。"

### Section 2｜如何驗證 Skill：Performance Measure（05:00 ~ 09:00）

**重點摘要：** 驗證 Skill 的方法是看 performance 有沒有 improve；ML 也是同樣邏輯。

**內容：**
- 人類驗證 Skill：考試——你學完滑雪課後會不會滑雪？學完數學後考卷分數有沒有提升？
- ML 同理：給機器一個「考卷」（test data），看 performance 有沒有改善
- 例：原本只能考 50 分，訓練後可以考 70 分 → 學到了
- 注意：考試可能無法反映真實能力（考試失場），所以 evaluation 設計本身就是一個重要的研究問題
- 之後就不會再用抽象的「Skill」這個詞，改用具體的「在某個任務上的 performance」

> "Mershulani 也是一樣的，我的目標其實是希望這個模型、這個機器具備某一個能力具備某一個技能，可是我要怎麼確認它具備這個技能？我們的目標就是希望它能一個技能上面的一個考卷，比如說驗證它是否具有某個數學的能力，它的 Performance 可以有提升。"

### Section 3｜什麼情境適合用 ML：股市預測、餐廳評價分類（09:00 ~ 16:00）

**重點摘要：** 兩個生活化例子——股市預測（K 線圖有 pattern）vs 餐廳評價分類（人類也說不出完整規則）。

**內容：**
- 股市預測：股市達人看 K 線圖決定買賣，雖然說不出完整論述但必定有 pattern 可學→適合 ML
- 餐廳評價分類：Google Maps 上好評 / 負評的二分類
- 一開始寫 rule-based："if 包含 love 就是正面"、"if 包含 hate 就是負面"——看起來很直覺
- 遇到例外："Oh it's not bad"（not bad 是正面，rule 寫 bad 就會錯）；"It's not good, it's bad one"（雙重負）
- rule 加越多越複雜：計算 bad 出現次數、看 not 後面的詞…最後還是無法涵蓋
- 結論：人類自己也說不出判斷好評 / 負評的完整規則→規則無法用 if-else 寫→必須用 ML
- **關鍵前提**：要有大量 (Input, Output) 配對資料，才能讓 ML 自己學

> "就像剛剛看到的我沒有辦法直接根據這個 Review 裡面的這些文字，然後就直接判斷它是正面還是負面。所以你發現你好像沒有辦法列舉所有可能的狀況，所以我們一定要用 Machine Learning 的方法。"

### Section 4｜ML 的數學視角：函數近似（16:00 ~ 19:00）

**重點摘要：** ML 就是從大量 (x, f(x)) 配對中「猜」f 的長相——就像數學課學函數推導。

**內容：**
- ML 的數學本質：透過大量 (x, f(x)) 配對，推測 f 的長相
- 類比數學課：給你 f(1)=1, f(2)=4, f(3)=9, f(4)=16 → 推論 f(x) = x²
- 越多 (x, f(x)) 配對 → 越能排除錯誤候選 → 越能逼近真實 f
- ML 模型就像一個「黑盒子」（Black Box），丟 input 進去會 output 結果
- 應用例子：語音辨識（input=聲音, output=文字）、手寫辨識（input=影像）、車牌辨識、天氣預測、地震預測、股市預測
- 共通點：input 與 output 之間有 pattern 可學

> "各式各樣的 Machine Learning 的應用，比如說語音辨識，我們的 x 輸入就是一段聲音，我們的 f(x) 就是希望它經過運算之後可以變成對應的這個文字。"

### Section 5｜AI vs ML：方法 vs 目標（19:00 ~ 22:00）

**重點摘要：** AI 是目標（模擬人類智慧行為），ML 是方法（從資料學）；ML 是 AI 的其中一條路徑。

**內容：**
- AI 的定義（維基百科）：讓機器模擬人類的智慧行為
- 三大類智慧行為：感知（聽、讀、看）= Perception、合成（寫、畫、唱）= Generation、推論（舉一反三）= Reasoning
- ML 是 AI 的其中一種實現方法：透過資料讓機器具備 Skill
- **不是所有 AI 都需要 ML**：傳統 AI 的 Game Tree Search、Expert System 都可以不用 ML
- 例：圍棋 AlphaGo 第一代結合 Game Tree + ML；AlphaGo Zero 純 ML
- 跨領域應用需要 domain knowledge：氣象學家 + AI 工程師合作；不是「無腦丟資料」就會 work

> "AI 我們通常其實就可以把它想成是我們就是要模擬出一些智慧的行為，那我們是透過運算的方式來做到的；那機器學習呢是從資料裡面，然後藉由這些資料去計算出這樣子的結果…所以 AI 並不是說所有的 AI 都一定要透過我們 Machine Learning 來做。"

### Section 6｜ML 與領域專家合作：颱風預測案例（22:00 ~ 26:19）

**重點摘要：** 純 ML 不夠——需要領域專家告訴你「南北半球雲圖旋轉方向不同」、「要參考多張圖」等隱性知識。

**內容：**
- 颱風預測例子：過去都是氣象專家看雲圖判斷強度、移動路徑
- 用 ML 做：把雲圖丟進 CNN 等影像模型，讓它學專家會看的 pattern
- 但純 ML 不夠好——氣象專家知道：
  - 南北半球的雲圖旋轉方向不同→需要預先翻轉方向
  - 只看雲圖不夠→還要參考風速圖、氣壓圖等其他資訊
- 結論：跨領域 AI 應用 = 領域專家 + AI 工程師合作；不是「無腦丟資料」就會 work
- 「Human Learning」是起點：專家先告訴你這個領域通常怎麼做、哪些重要
- 陳縕儂教授引用 AlphaGo 開發時大量依賴圍棋高手的指導作為類比

> "氣象學家可能就會告訴我說，這個雲圖南半球跟北半球它的旋轉方向是不一樣的…這個 Knowledge 如果你沒有很熟悉這個領域的話，你可能就不知道…做這些操作之後效果就會很顯著地變得更好。"

## 三、關鍵概念定義表

| 概念 | 定義 | 出處 / 應用 |
|------|------|-------------|
| Machine Learning | 讓機器從資料中學習 Skill，使其在某個任務的 performance 可以改善 | Section 1, 2 |
| Observation | 人類透過感官（眼、耳）接收到的經驗 | Section 1 |
| Skill | 學會的能力（例如：會寫程式、會下棋、會辨識圖片） | Section 1 |
| Performance Measure | 量化評估 Skill 的方式（考試分數、accuracy 等） | Section 2 |
| Hypothesis Set | 候選函數集合 $H$，ML 要從中選最好的 $g$ | FAI 1.3 詳細介紹 |
| Pattern | 資料中可學習的結構性規律 | Section 3 |
| Rule-based AI | 用 if-else 規則寫死的 AI（傳統 AI 範式） | Section 5 |
| Expert System | 以領域專家規則為核心的 AI 系統 | Section 5 |
| Perception AI | 感知型 AI（語音、影像辨識） | Section 5 |
| Generation AI | 生成型 AI（文字、圖片、聲音生成） | Section 5 |
| Reasoning AI | 推論型 AI（舉一反三、邏輯推理） | Section 5 |
| AlphaGo | DeepMind 圍棋 AI；2016 打敗世界冠軍 | Section 5 |
| Human Knowledge | 領域專家的隱性 / 顯性知識；ML 跨領域應用的必要元素 | Section 6 |

## 五、核心主旨

> Machine Learning 的本質是「讓機器具備從資料中學習 Skill 的能力」——這個 Skill 必須可以透過 performance measure 量化驗證，而資料中必須存在可學習的 pattern。ML 不是要發明新的 if-else 規則，而是當人類自己都說不出判斷規則時，讓機器從大量 (input, output) 配對中自己學。AI 與 ML 的關係是：ML 是實現 AI 的一條路徑，不是唯一路徑。純 ML 不夠——跨領域應用必須有領域專家的 Human Knowledge 介入。
