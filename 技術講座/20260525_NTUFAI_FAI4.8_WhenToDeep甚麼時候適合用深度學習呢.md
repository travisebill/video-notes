## 一、主題與背景

FAI 4.8「甚麼時候適合用深度學習呢 — When to Deep?」是深度學習章節的尾聲，林軒田教授用簡短的篇幅回顧整個章節的脈絡，並整理「什麼時候才適合使用深度學習」這個實務判斷問題。本講為超短片，僅作章節總結與方向指引。

節目開頭就點出，深度學習並非萬靈丹；它的成功需要滿足若干先決條件。第一個條件是「有沒有足夠多的資料」。深度學習模型的參數量動輒數百萬、數十億，沒有海量資料支撐，模型很快就會過擬合——記住訓練資料的雜訊，卻學不到可泛化的規律。如果資料量有限，往往更簡單的模型（線性回歸、決策樹）反而更實用。

第二個條件是「能不能輕易設計出有效的特徵」。如果某個問題領域的人類專家已經對重要特徵有清晰的理解（例如某些醫學指標、某些金融比率），那麼把這些特徵明確抽出來，搭配傳統機器學習模型，往往比讓深度學習從原始資料中自己摸索特徵更有效率。但如果是影像辨識、語音辨識、自然語言這類「人類自己也說不清楚關鍵特徵是什麼」的問題，深度學習從原始資料自動學特徵的能力就成為壓倒性優勢。

第三個條件是「有沒有充足的運算資源」。深度學習的訓練需要大量 GPU / TPU 時間與記憶體；如果運算資源不足，不僅訓練時間會拉到不切實際地長，還可能在反覆實驗中被成本卡住，反而難以產出有用的成果。

節目最後用一段總結回顧整個 FAI 4 章節的學習路徑：從深度學習的挑戰複習出發，進入反向傳播的詳細計算，理解表徵學習與預訓練的策略意涵，認識梯度消失問題與各種對策，學習 CNN 處理影像的設計哲學與 RNN 處理序列的核心機制，最後透過正則化技巧讓模型能夠真正泛化。這套由淺入深、由計算到結構再到策略的安排，正是深度學習工程的標準學習地圖。

**核心議題一句話**：深度學習是一個強大但昂貴的工具，使用前必須先回答「資料夠不夠、特徵能不能手動設計、運算資源撐不撐得住」這三個問題；能正面回答這三題的場景，才是深度學習真正發揮威力的舞台。

## 二、章節脈絡

### Section 1｜深度學習的三大使用前提（00:00 ~ 01:30）

**重點摘要：** 深度學習不是萬靈丹；需要足夠資料、無法手動設計特徵、充足運算資源三個前提；不符合條件時應用更簡單模型。

**內容：**
- 深度學習並非萬靈丹
- 第一前提：要有足夠多的資料（避免過擬合）
- 第二前提：不能輕易手動設計特徵
- 第三前提：要有充足的運算資源（GPU / TPU）
- 三個條件都符合 → 深度學習真正發揮威力的舞台
- 否則應用更簡單的模型（線性回歸、決策樹）

> "深度學習最重要的核心概念就是它是需要有很 Stable 的這些 Data，然後呢也需要有充足的這些運算資源，然後才可以讓它的這個能力發展出來。"

### Section 2｜FAI 4 章節回顧（01:30 ~ 02:15）

**重點摘要：** 從挑戰複習 → 反向傳播 → 表徵學習 → 梯度消失 → CNN / RNN → 正則化；這套學習地圖是深度學習工程的標準路徑。

**內容：**
- FAI 4 章節的學習路徑：
  - 4.1：深度學習挑戰複習 + 反向傳播動機
  - 4.2：反向傳播詳細計算（Forward + Backward + DP）
  - 4.3：表徵學習與預訓練（Autoencoder、BERT、GPT）
  - 4.4：梯度消失問題（ReLU、初始化、殘差連接）
  - 4.5：CNN（影像處理）
  - 4.6：RNN（序列處理）
  - 4.7：正則化技巧（Weight Decay、Early Stopping、Dropout）
  - 4.8：使用時機判斷
- 由淺入深、由計算到結構再到策略
- 這套安排是深度學習工程的標準學習地圖

## 三、關鍵概念定義

| 概念 | 定義 | 重要性 |
|------|------|--------|
| 過擬合（Overfitting） | 模型在訓練資料表現極好但測試資料表現差 | 深度學習失敗的主因之一 |
| 運算資源（Computation Resource） | GPU / TPU 時間與記憶體 | 深度學習訓練的硬基礎 |
| 特徵工程（Feature Engineering） | 人工設計有意義的特徵 | 傳統機器學習的核心 |
| 自動特徵學習 | 讓模型從原始資料自動學特徵 | 深度學習的核心能力 |
| 基礎模型（Foundation Model） | 在大規模資料上預訓練、可遷移到多任務的大模型 | 現代深度學習的前沿方向 |

## 五、核心主旨

> 深度學習是一個強大但昂貴的工具，並非所有場景都適合使用。在決定是否採用深度學習之前，必須先回答三個關鍵問題：第一，資料量是否足夠？深度學習模型的參數量動輒數百萬、數十億，沒有海量資料支撐很快就會過擬合；如果資料有限，線性回歸、決策樹等更簡單的模型往往更實用。第二，特徵能否手動設計？如果人類專家已經對重要特徵有清晰理解，把特徵明確抽出來搭配傳統機器學習模型通常更有效率；但若是影像、語音、自然語言這類「人類自己說不清楚關鍵特徵」的問題，深度學習從原始資料自動學特徵的能力就是壓倒性優勢。第三，運算資源是否充足？深度學習的訓練需要大量 GPU / TPU 時間與記憶體，運算資源不足時不僅時間會拉到不切實際地長，還會在反覆實驗中被成本卡住。能正面回答這三題的場景，才是深度學習真正發揮威力的舞台；不符合條件的場景，強行使用深度學習往往只會帶來不必要的複雜度與成本——這也是深度學習工程師在專案初期就必須培養的判斷力。
