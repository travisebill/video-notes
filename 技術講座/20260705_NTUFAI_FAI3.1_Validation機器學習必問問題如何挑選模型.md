## 一、主題與背景

FAI 3.1「機器學習必問問題：如何挑選模型？」是林軒田教授從 Non-Linear Models（FAI 2.x）推進到 Deep Learning（FAI 3.x）之間的銜接講。本講切入的問題是：在無數可能的模型候選中，從 Linear Model、Decision Tree、Random Forest、到 Deep Learning，怎麼決定要選哪一個？

林軒田教授先回到上一週 FAI 2.4 提到的策略：模型選擇「由簡入繁」的預設路徑——Linear → Decision Tree → Random Forest → Deep Learning。Linear 與 Decision Tree / Random Forest 之所以是好的起點，並非因為它們最強，而是因為它們具備良好的可解釋性，訓練結束後可以逐層檢視每一個分支的判斷依據是否符合領域知識；當簡單模型已經足夠時，升級到深度學習往往只會帶來計算成本上升與過擬合風險，而非實質性能提升。

接著，影片引入核心概念「驗證」（Validation）。真實世界無法直接觀察目標函數 f 的全貌，只能透過假設集合 H 訓練出候選函數 g，並試圖讓 g 接近 f。影片用「射擊靶心」的比喻拆解 g 與 f 的差距由兩個成分構成：Betas（瞄準器偏移）與 Variance（射手手抖）。這個 Bias-Variance 拆解呼應了 FAI 1.x 系列中先前討論的 Generalization 難題，並延伸出「模型複雜度」對 Bias-Variance 取捨的影響：模型太簡單會 underfitting（Bias 為主），模型太複雜會 overfitting（Variance 為主）。

實作層面，影片引入驗證集（Validation Set）作為模擬測試的機制，並介紹 V-Fold Cross Validation——當資料量有限無法切出獨立驗證集時，把訓練資料切成 V 等份並輪流當驗證集，最終選擇在 V 次驗證中表現穩定的模型，避免單一切法帶來的雜訊偏差。

**核心議題一句話**：模型選擇沒有唯一最佳路徑，但「由簡入繁」加上「驗證機制」提供了工程實務上最可靠的策略；V-Fold Cross Validation 是資料受限情境下的穩健解方。

## 二、章節脈絡

### Section 1｜模型選擇的預設路徑：由簡入繁（00:00 ~ 03:00）

**重點摘要：** 上週複習 Non-Linear Model；模型選擇預設從線性開始，升級到 Decision Tree、Random Forest，再到深度學習；理由是可解釋性與計算效率。

**內容：**
- 銜接上週 Non-Linear Model；今天進入 Validation 主題
- 模型選擇預設路徑：Linear → Decision Tree / Random Forest → Deep Learning
- Decision Tree 與 Random Forest 之所以是好的起點：具備良好的可解釋性
- 訓練結束後可逐層檢視每一個分支的判斷依據是否符合領域知識
- 模型複雜度越高越容易 overfitting，需要更多資料
- 如果簡單模型已經足夠時，升級到深度學習通常會帶來計算成本與過擬合風險

> "那如果可以簡單model就做得好的話，我們其實就沒有必要用更複雜的model...Decision tree跟random forest它其實是比較有interpretability的事情的。"

### Section 2｜驗證的核心：Betas-Variance 分解（03:00 ~ 10:00）

**重點摘要：** 真正函數 f 不可見，只能從假設集合 H 中選 g 接近 f；g 跟 f 的差距由 Betas 與 Variance 兩個成分構成；射擊靶心比喻用來拆解這兩個成分。

**內容：**
- 真正理想的函數 f 永遠看不到，只能找 g 接近 f
- f 可能是 Linear、可能是 Non-Linear，事前不知道
- 模型選擇變成核心問題，沒有唯一的最佳路徑
- g 與 f 的差距由兩個成分構成：Betas 與 Variance
- 射擊靶心比喻：Betas 相當於槍本身的偏移，Variance 相當於射手手抖
- Betas 為主（瞄準偏移）：模型瞄準的位置本來就偏掉了
- Variance 為主（射手手抖）：瞄得準但手會抖，導致結果散開
- 理想情況：Betas 與 Variance 都很低

> "g跟f之間其實會有一個差異，那這個差異通常是有兩個東西所組成的，就是這邊說的Betas跟Variance。"

### Section 3｜Underfitting 與 Overfitting 的學習曲線（10:00 ~ 17:00）

**重點摘要：** 模型複雜度太低 → Underfitting（Betas 主）；模型複雜度太高 → Overfitting（Variance 主）；理想的選擇是中間合適的複雜度。

**內容：**
- 真實函數假設是三次多項式
- 選 Linear Model（Betas 主）：表示力不夠，瞄準位置根本包不到 f
- 選 6 次多項式（Variance 主）：可以包到 f，但自由度過高，Variance 過大
- 理想的複雜度：剛好包到 f 但又不會超過太多
- 學習曲線圖：模型越複雜，E_in（訓練誤差）越低，但 E_out（測試誤差）不一定最低
- 模型複雜度的 sweet spot 出現在中間
- Underfitting 區：模型太簡單，Betas 大，E_in 與 E_out 都高
- Overfitting 區：模型太複雜，Variance 大，E_in 低但 E_out 高

> "理想的這個f是在這邊...那你現在選了一個比較簡單的模型比如說你選了一個一次的多項式，那你這個hypothesis是不是絕對沒有可能包含到F。"

### Section 4｜驗證集機制（17:00 ~ 26:00）

**重點摘要：** 真實測試資料不能被打開；把訓練資料切成 D_train 與 D_val，用 D_val 模擬測試誤差挑選模型，最後用全部 D_train 重新訓練得到最終 g。

**內容：**
- 有一個關鍵原則：真實測試資料在選模型過程中完全不能被打開
- 否則就等同於偷看答案，選模型時看到的「測試誤差」會嚴重低估
- 把訓練資料切成 D_train 與 D_val
- D_train 訓練模型，D_val 模擬測試誤差
- 用 D_val 上表現最好的模型當作最終模型 M_star
- 把 M_star 套回全部 D_train 重新訓練，得到最終 g
- 為什麼要用全部 D_train 重新訓練？因為資料越多，模型理論上表現越好
- 選擇驗證集時要注意分布：盡量選擇分布接近測試集的部分
- 時間敏感資料：選較靠近測試期的部分當 D_val

> "testing data其實是不能看到的...所以我們有準備一個可以看到的public的testing叫做validation set，那我們就可以把training set分成兩部分。"

### Section 5｜V-Fold Cross Validation（26:00 ~ 33:15）

**重點摘要：** V-Fold Cross Validation：切成 V 等份輪流驗證；最終選 V 次驗證中表現穩定的模型；資料量有限時的最佳策略。

**內容：**
- V-Fold Cross Validation 是 Validation 的延伸，常用於資料量不足時
- 把訓練資料切成 V 等份（例如 V=10，每份 1/10）
- 每次用其中 1 份當作 D_val，其他 V-1 份當作 D_train
- 總共跑 V 次，每次都選一個最佳模型
- 最終選擇在 V 次驗證中表現最穩定的模型 M_star
- 好處：避免單一切法可能造成的分布偏斜
- 即使切得不夠好，V 次平均也會讓雜訊互相抵消
- 接著用 M_star 套回全部 D_train 重新訓練，得到最終 g
- 結尾 review 整個 ML 流程：Collect Data → Decide Hypothesis Set → 訓練 → 評估 → Validation

> "因為你的validation它的performance稍微比較差所以你就選到了一個model...所以我們全部都輪流做一遍這樣子，就最保險。"

## 三、關鍵概念定義表

| 概念 | 定義 | 出處 / 應用 |
|------|------|-------------|
| Hypothesis Set H | 所有可能函數的集合，ML 從中選最好的 g | Section 2 |
| Bias | g 跟 f 中心點的差距（瞄準器偏移） | Section 2 |
| Variance | 訓練資料改變時 g 的變動程度（射手手抖） | Section 2 |
| Underfitting | 模型太簡單，E_in 跟 E_out 都高 | Section 3 |
| Overfitting | 模型太複雜，E_in 低但 E_out 高 | Section 3 |
| 模型複雜度 | Hypothesis Set 的大小或 VC Dimension | Section 3 |
| Validation Set | 把訓練資料切成 D_train / D_val，模擬測試 | Section 4 |
| D_train | 用來訓練模型的部分 | Section 4 |
| D_val | 用來模擬測試誤差挑選模型的部分 | Section 4 |
| D_test | 真實測試資料，選模型時不可見 | Section 4 |
| M_star | 用 Validation 機制挑出的最佳模型 | Section 4 |
| V-Fold Cross Validation | 切成 V 等份輪流當 D_val 的驗證方法 | Section 5 |
| 偷看答案 | 直接拿 testing data 來選模型的違規做法 | Section 4 |
| 學習曲線 | 訓練誤差 vs 測試誤差隨模型複雜度的變化 | Section 3 |
| Interpretability | 模型可解釋性，可以檢視判斷依據 | Section 1 |

## 五、核心主旨

> 真實函數 f 永遠無法直接觀察，只能透過假設集合 H 訓練候選函數 g 來逼近。g 與 f 之間的差距由 Bias 與 Variance 兩個成分構成，Bias 反映瞄準位置是否涵蓋 f，Variance 反映射手手抖的程度；模型複雜度太低會 underfitting（Bias 主），複雜度太高會 overfitting（Variance 主），理想的複雜度是「剛好涵蓋 f 但又不會超過太多」。實作上將訓練資料切為 D_train 與 D_val，以 D_val 模擬測試誤差挑選模型 M_star，最後用全部 D_train 重新訓練得到最終 g；資料受限時改用 V-Fold Cross Validation 輪流驗證，取多次驗證中表現最穩定的模型。「由簡入繁」加上「驗證機制」，是機器學習工程實務上最可靠的模型選擇策略。

## 六、金句摘錄

1. "如果可以簡單model就做得好的話，我們其實就沒有必要用更複雜的model。"
2. "decision tree跟random forest它其實是比較有interpretability的事情的，所以當你把它訓練完之後，你也可以去檢視一下說它訓練出來的這些分類的原因是不是符合你的預期。"
3. "Betas跟Variance在重申一遍，Betas就是我們希望它小的話，就是我希望g能夠包到f...可是呢當你g太複雜的時候，你的Variance就會比較高。"
4. "testing data其實是不能看到的...不能的不能的，你完全不能把testing data打開，只能把training data裡面分成是training跟validation。"
5. "因為你的validation它的performance稍微比較差所以你就選到了一個model，並不是和這個test model算是比較保險的一個做法。"
6. "我們會藉由這些validation的設置像剛剛的這些做法來幫助我們去選擇比較適合的這個hypothesis。"
7. "機器學習其實沒有model是萬能的，你需要比較找進去的其實是一個合適的hypothesis。"
