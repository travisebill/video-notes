## 一、主題與背景

FAI 1.6「超實用的邏輯迴歸你真的懂嗎」是林軒田教授 Machine Learning 章節的第六講，約 23 分鐘，也是 Linear Models 單元的收尾。本講正式介紹 Logistic Regression——把 Linear Regression 的連續輸出升級為「binary 分類的機率」。

為什麼需要 Logistic Regression？現實中很多任務要的不是「yes / no」而是「yes 的機率」：病人會不會得癌症（不只是得 / 不得，而是得病機率）；申請者會不會接受 offer（不只是接受 / 不接受，而是接受機率）；病人是否會死亡（不只是死亡 / 活著，而是死亡機率）。這些任務「只能觀察到 binary label」但「想預測機率值」——這就是 Logistic Regression 解決的問題，也稱為「Soft Binary Classification」。

Logistic Regression 的核心是 Logistic Function $\theta(s) = \frac{1}{1 + e^{-s}}$：把 $\mathbf{w}^T \mathbf{x}$ 的任意實數 $s$ 對應到 0~1 之間的機率值。Error Measure 用 Cross-Entropy：$-y \log \hat{y} - (1-y) \log (1-\hat{y})$。目標函數是 Continuous Differentiable Convex，可以用 Gradient Descent 一步步滑到最低點。

本講的最後一段是整個 ML 章節的 Recap：比較 Linear Classification、Linear Regression、Logistic Regression 三種 Linear Model 的差異（Output、Error Measure、Solution）。最後給出實務建議——遇到任何 ML 問題先從 Linear 開始：簡單、快、Generalization 強，Linear 不夠再升級到 Deep。

**核心議題一句話**：Logistic Regression 解決「只能觀察到 binary 但想預測機率」的真實任務——用 Logistic Function 把 Linear Score 對應到機率、用 Cross-Entropy 量化誤差、用 Gradient Descent 找最佳 $\mathbf{w}$——這是工業界最常用的 ML 模型之一，也是 ML 章節從「理論推導」到「實務應用」的關鍵橋樑。

## 二、章節脈絡

### Section 1｜從 Regression 到 Logistic Regression：軟分類的需求（00:00 ~ 05:00）

**重點摘要：** 真實任務要的不是 binary 而是機率值；醫療、offer 接受、風險評估都需要機率預測。

**內容：**
- FAI 1.5 Linear Regression 預測連續值；FAI 1.1-1.4 Linear Classification 預測 binary
- 但很多真實任務要的是「機率」：病人得癌症機率、申請者接受 offer 機率、病人死亡機率
- 醫學研究：疾病有 / 沒有是 binary，但想知道「得病機率」、「死亡機率」、「治療副作用機率」
- 這些任務「只能觀察到 binary 但想預測機率」——稱為 Soft Binary Classification
- 真實挑戰：無法收集「這個人有 0.9 機率得病」的標籤，只能收集「得 / 不得」的 binary 標籤
- Logistic Regression 就是為了解決這個問題：Linear Score → Logistic Function → 機率值

> "我們想要知道說如果這個 offer 發出去之後，它有多少機率會接受 offer 會來公司上班，所以我們想要的資訊我們的這個 output 是一個機率值。"

### Section 2｜Logistic Function：把實數對應到機率（05:00 ~ 08:00）

**重點摘要：** $\theta(s) = \frac{1}{1+e^{-s}}$ 把任意實數 $s$ 平滑對應到 0~1 機率；中點 $s=0$ 對應 0.5。

**內容：**
- Linear Model 輸出 $\mathbf{w}^T \mathbf{x} = s$（任意實數）
- 想把 $s$ 對應到 0~1 的機率值——需要一個 Sigmoid / Logistic Function
- $\theta(s) = \frac{1}{1 + e^{-s}}$
- 特性：
  - $s = 0$ 時 $\theta = 0.5$（中點）
  - $s$ 越大 $\theta$ 越接近 1
  - $s$ 越小 $\theta$ 越接近 0
  - 平滑、可微、輸出範圍 (0, 1)
- 為什麼選 Logistic 而不是其他 Sigmoid？歷史與數學便利（cross-entropy 漂亮）
- 應用：信用評分（違約機率）、醫療診斷（疾病機率）、行銷（轉換機率）

> "於是我們就用一個 Function 把這一個原本的 S 沒有任何 Range 的東西，Mapping 到一個 0 到 1 之間的這個 Range，所以這邊的這一個 Function 就叫做 Logistic 的 Function。"

### Section 3｜Error Measure：Cross-Entropy（08:00 ~ 11:00）

**重點摘要：** Cross-Entropy $-y \log \hat{y} - (1-y) \log (1-\hat{y})$ 量化機率預測的誤差；convex 可微。

**內容：**
- Cross-Entropy（或稱 log-loss）：$-\sum y_n \log \hat{y}_n + (1-y_n) \log (1-\hat{y}_n)$
- 直觀：當 $y=1$ 時，希望 $\hat{y}$ 接近 1（loss 小）；當 $y=0$ 時，希望 $\hat{y}$ 接近 0
- 為什麼不用 Squared Error？因為 Squared Error 配 Logistic 函數不是 convex，gradient descent 會卡在 local minimum
- Cross-Entropy 的優勢：
  - Convex（單一 global minimum）
  - Continuous Differentiable（gradient descent 可用）
  - 機率語意清晰（K-L divergence 形式）
- 目標：minimize $E_{in}$ 找 $\mathbf{w}^*$
- Cross-Entropy 也是資訊理論的核心概念：衡量兩個機率分布的差異

> "這個東西其實我們會叫它 Cross Entropy 或者是它其實是 K-L divergence 那個詳細的推到這邊就不會介紹。"

### Section 4｜Gradient Descent：滑下坡找最佳 $\mathbf{w}$（11:00 ~ 16:00）

**重點摘要：** Cross-Entropy 是 convex 且可微；Gradient Descent 一步步更新 $\mathbf{w}$ 滑到谷底。

**內容：**
- Logistic Regression 沒有 Closed-form Solution（不像 Linear Regression）
- 但 $E_{in}(\mathbf{w})$ 是 Continuous + Differentiable + Convex → 可以用 Gradient Descent
- Gradient Descent 直觀：把 $\mathbf{w}$ 想像成一顆球在凸面上，沿著斜率（負梯度方向）滑下
- 數學：$\mathbf{w}_{t+1} = \mathbf{w}_t - \eta \nabla E_{in}(\mathbf{w}_t)$
- Learning rate $\eta$ 控制步長：
  - 太大：可能跨過最低點
  - 太小：收斂太慢
  - 實務：動態調整（開始大步，接近最低點小步）
- Gradient Descent 是 ML 中最重要的優化方法之一（後續 Deep Learning 也用 SGD / Adam 等變體）
- 比較 PLA（FAI 1.4）：PLA 是「看錯就改」的硬規則；Gradient Descent 是「沿著斜率滑」的平滑優化

> "這個 $\eta$ 其實是它的腳步…如果我更新的時候我一步要移動很大步，比如說我要從這邊移動一大步跑到這邊來，還是說我一次移動移動小小步…所以要怎麼用一個聰明的方式有效率的走到目標，其實也是很多 Algorithm 需要 Design 的。"

### Section 5｜三種 Linear Model Recap（16:00 ~ 19:00）

**重點摘要：** 三種 Linear Model 比較——Output、Error Measure、Solution 各有不同。

**內容：**

| 項目 | Linear Classification | Linear Regression | Logistic Regression |
|------|---------------------|-------------------|---------------------|
| Output | sign($\mathbf{w}^T \mathbf{x}$) | $\mathbf{w}^T \mathbf{x}$ | $\theta(\mathbf{w}^T \mathbf{x})$ |
| Error Measure | 0/1 Error | Squared Error | Cross-Entropy |
| Solution | PLA（疊代） | Closed-form | Gradient Descent |
| 應用 | binary 分類 | 連續值預測 | 機率預測 |

- 共同點：Hypotheses Set 都是所有 hyperplane $\mathbf{w}^T \mathbf{x}$；都是 Linear 的 Score Function
- 差異：Output、Error、Solution 三層各有不同
- Linear Classification 用 Sign function；Linear Regression 直接輸出；Logistic Regression 用 Logistic Function
- 業界實務：Logistic Regression 是 Classification 的首選 baseline——比 SVM、Neural Net 簡單、穩定、可解釋

> "今天講的 Linear Model 各自是應用在不同的情境，但它們相同的點呢，它都是一個 Linear 的 Scoring Function，所以它的核心就是 Linear 的。"

### Section 6｜ML Recap 與實務建議（19:00 ~ 23:14）

**重點摘要：** 遇到任何 ML 問題先從 Linear 開始；Linear 不夠再升級 Deep。

**內容：**
- 整個 ML 章節 Recap：
  - ML = 從資料學 Skill，使其 performance 可以 improve
  - ML Recipe 五步驟（判斷、Data、Hypotheses、Error、Learning）
  - 三種 Linear Model（Classification、Regression、Logistic）
- 實務建議：遇到 ML 問題先從最簡單的 Linear 開始
  - Linear 簡單、快、Generalization 強
  - Linear 不夠再升級到 Deep Learning
  - 很多業界任務 Linear Model 就夠用
- Linear Model 不需要大量資料訓練，適合中小型資料集
- 為什麼要先試 Linear？因為 Linear 不行通常代表「特徵不足」而非「模型太簡單」——加特徵往往比換模型更有效

> "大家在遇到任何 Machine Learning 相關的問題，其實你第一個可以先從 Linear 的 Model 來想，因為 Linear Model 是一個非常簡單然後它也很快就可以算出來，如果 Linear Model 可以解的話你就用 Linear Model 來擠。"
