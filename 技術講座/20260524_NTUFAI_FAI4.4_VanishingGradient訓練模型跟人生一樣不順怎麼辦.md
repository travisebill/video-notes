## 一、主題與背景

FAI 4.4「訓練模型跟人生一樣不順怎麼辦 — Vanishing Gradient」由陳縕儂教授切入深度學習訓練過程中一個看似無形、卻足以讓整個訓練流程停擺的關鍵難題——梯度消失（Vanishing Gradient）問題。

節目開頭先把問題的成因攤開來看。反向傳播的核心是計算損失函式對每一層權重的偏微分，並透過鏈鎖法則把每一層的局部導數相乘。問題出在「相乘」這兩個字——以 Sigmoid 激活函式為例，它的導數最大值只有 0.25，且大部分區間都遠小於 0.25；Tanh 雖然好一些，最大導數也只有 1，但在大部分區間仍小於 1。一旦網路層數很深，這些小於 1 的數字反覆相乘，會以指數速度衰減到底；到了靠近輸入層的那幾層，累乘出來的梯度值已經小到電腦浮點數精度無法表達，直接被截斷為零。當梯度為零，參數就不再更新，訓練也就此停擺。

節目點出這個問題的嚴重性：模型並不是真正「學完」了，而是因為梯度消失而「誤以為」自己學完了。這是一種極具欺騙性的失敗模式——從表面數字看，訓練誤差可能還在緩慢下降，但實際上靠近輸入層的權重早已凍住，只有靠近輸出層的權重還在動；整體模型的學習能力被嚴重限制在前幾層，無法充分運用深層結構的容量。

為了解決梯度消失，研究者提出了多種策略。第一種是改用更合適的激活函式——ReLU 系列的導數在正區間恆為 1，徹底避免了小於 1 反覆相乘的衰減問題。第二種是改良權重初始化方式，例如 He Initialization（適用於 ReLU）或 Xavier Initialization（適用於 Tanh），讓每一層輸入與輸出的方差維持穩定，避免信號在傳遞過程中逐步放大或縮小。第三種是引入殘差連接（Residual Connection），讓梯度可以透過捷徑直接從後層傳到前層，避免必須經過所有中間層的累乘衰減。

更深一層的啟示是：梯度消失揭示了「深度」並非免費的午餐。每一層理論上都帶來更大的表示容量，但前提是梯度訊號能夠順利穿透所有層。現代深度學習架構（ResNet、Transformer 等）的成功，很大程度上正是因為它們在結構設計上就內建了對抗梯度消失的機制。

**核心議題一句話**：梯度消失是深度學習從理論走向實務時遇到的第一個結構性障礙；理解它的成因與各種對策，是設計任何深層網路時必備的背景知識。

## 二、章節脈絡

### Section 1｜梯度消失的成因（00:00 ~ 04:30）

**重點摘要：** 反向傳播用鏈鎖法則把每一層的局部導數相乘；Sigmoid、Tanh 等激活函式的導數小於 1；層數一多 → 累乘衰減 → 梯度消失。

**內容：**
- 反向傳播需要從輸出端往輸入端計算 δ
- 每一層 δ 的計算都涉及激活函式的導數 σ'(z)
- Sigmoid 導數最大值 0.25，多數區間遠小於 0.25
- Tanh 導數最大值 1，但多數區間仍小於 1
- 層數深時，這些小於 1 的數字反覆相乘
- 結果：靠近輸入層的 δ 衰減到極小
- 累乘效應是指數級的，網路越深問題越嚴重

> "Ettivation function 是不是剛剛之前有看到很多的 Ettivation function 都會把你的值壓在零到一之間，所以從非常後面開始往前，當你的層數非常多的時候，你是不是會成到這種 Ettivation function 的篇尾分非常多次，那它如果是一個小於一的值的話，那很可能你乘了非常多次之後它就變得超小。"

### Section 2｜梯度消失的後果（04:30 ~ 07:30）

**重點摘要：** 梯度接近零 → 參數不更新 → 訓練「假性收斂」；靠近輸入層的權重完全凍住；模型的學習能力受限。

**內容：**
- 梯度消失 → ∂E/∂w ≈ 0
- 參數不再更新 → 模型「誤以為」自己學完了
- 訓練誤差可能還在緩慢下降，但實際上輸入層的權重早已凍住
- 只有靠近輸出層的權重還在動
- 整體模型的學習能力被嚴重限制
- 這是一種極具欺騙性的失敗模式

> "你很有可能在中間你的這些規點的值就是因為太小，所以你的電腦精度沒辦法復合，所以就消失了這樣。所以它你這邊很小的話你這邊沉出來，你這邊的這個最後要得到這個篇尾分的值你是不是就會受到影響。"

### Section 3｜解決方案一：改用更好的激活函式（07:30 ~ 09:00）

**重點摘要：** ReLU 系列的導數在正區間恆為 1；徹底避免小於 1 反覆相乘的衰減問題；是現代深度學習的標準選擇。

**內容：**
- ReLU：f(z) = max(0, z)，正區間導數 = 1
- 反向傳播時不衰減 → 梯度可順利穿透所有層
- Leaky ReLU、PReLU、ELU 等變體
- 現代深度學習幾乎都預設使用 ReLU 系列

### Section 4｜解決方案二：更好的權重初始化（09:00 ~ 10:30）

**重點摘要：** He Initialization（適用 ReLU）、Xavier Initialization（適用 Tanh）；讓每一層輸入輸出方差穩定；避免信號放大或縮小。

**內容：**
- Xavier Init：權重方差 = 2 / (輸入維度 + 輸出維度)，適用 Tanh
- He Init：權重方差 = 2 / 輸入維度，適用 ReLU
- 目標：讓每一層的輸入方差 ≈ 輸出方差
- 信號在傳遞過程中既不放大也不縮小
- 從初始化階段就避免梯度消失的根源

### Section 5｜解決方案三：殘差連接（10:30 ~ 11:56）

**重點摘要：** ResNet 引入殘差連接；梯度可透過捷徑直接從後層傳到前層；不需要經過所有中間層的累乘。

**內容：**
- 殘差連接：output = F(x) + x
- 梯度反向傳播時，可以透過捷徑直接傳遞
- 即使中間層的局部導數很小，梯度仍能順利穿透
- ResNet 讓訓練數百層的網路成為可能
- 現代 Transformer 架構也廣泛使用殘差連接

## 三、關鍵概念定義

| 概念 | 定義 | 重要性 |
|------|------|--------|
| 梯度消失（Vanishing Gradient） | 反向傳播時，靠近輸入層的梯度累乘衰減到接近零 | 深度學習訓練的隱性失敗模式 |
| 鏈鎖法則（Chain Rule） | 複合函式的微分可拆解為各層微分的乘積 | 梯度累乘的數學基礎 |
| Sigmoid 導數 | σ'(z) = σ(z)(1 - σ(z))，最大值 0.25 | 早期深度學習梯度消失的主因 |
| ReLU | f(z) = max(0, z)，正區間導數 = 1 | 解決梯度消失的標準激活函式 |
| Xavier Initialization | 權重方差 = 2 / (n_in + n_out) | 適用 Tanh 的權重初始化策略 |
| He Initialization | 權重方差 = 2 / n_in | 適用 ReLU 的權重初始化策略 |
| 殘差連接（Residual Connection） | output = F(x) + x，提供梯度捷徑 | ResNet 解決深層網路訓練問題的核心設計 |

## 五、核心主旨

> 梯度消失（Vanishing Gradient）是深度學習從理論走向實務時遇到的第一個結構性障礙——反向傳播透過鏈鎖法則把每一層的局部導數相乘，當網路層數很深時，這些小於 1 的數字反覆相乘會以指數速度衰減；到了靠近輸入層的那幾層，累乘出來的梯度已經小到電腦浮點數精度無法表達，被截斷為零，使得參數不再更新，訓練進入「假性收斂」。這個問題的本質是「深度並非免費的午餐」——每一層理論上都帶來更大的表示容量，但前提是梯度訊號能夠順利穿透所有層。現代深度學習透過三種策略解決這個問題：使用 ReLU 等不會衰減的激活函式、採用 He 或 Xavier 等良好的權重初始化、以及引入殘差連接讓梯度可以透過捷徑直接穿透。理解這三種對策背後的原理，是設計任何深層網路時必備的背景知識；現代 ResNet、Transformer 等架構的成功，很大程度上正是因為它們在結構設計上就內建了對抗梯度消失的機制。
