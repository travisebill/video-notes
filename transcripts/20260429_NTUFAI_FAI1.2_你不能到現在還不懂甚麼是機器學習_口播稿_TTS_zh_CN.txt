台大资讯工程学系 2026 年春季「人工智慧导论」FAI 1.2「你不能到现在还不懂甚么是机器学习」，由林轩田教授主讲，约 26 分钟，是 Machine Learning 章节的第二讲。

从学习的本质角度，林轩田教授从日常的「学习」这个词汇切入，把人类学习——透过观察累积 Skill——对应到机器学习——透过资料学到 Skill。人类透过写程式、跑实验、观察输出，从观察中累积 Skill；机器没有眼耳直接接收感知，所以是从 Data 学。把人类的 Observation 完整收集起来变成 Data，让机器在这些资料上做学习，这是 ML 的本质定义。图示上，人类 Learning 等于大量 Observation 变成 Skill，机器 Learning 则是把 Observation 换成 Data。

从验证 Skill 的方法角度，林轩田教授指出 ML 与人类学习共享同一个评估逻辑：透过 Performance Measure 量化 Skill 是否提升。人类验证 Skill 用考试——学完滑雪课后会不会滑雪？学完数学后考卷分数有没有提升？ML 同理：给机器一个考卷（test data），看 performance 有没有改善。但他也提醒，考试可能无法反映真实能力——这是为什么 evaluation 设计本身就是重要的研究问题。

从 ML 适用情境角度，影片用两个生活化例子对比。股市预测：股市达人看 K 线图决定买卖，虽然说不出完整论述但必定有 Pattern 可学，这是适合 ML 的情境。餐厅评价分类：一开始写 rule-based 方法——if 包含 love 就是正面、if 包含 hate 就是负面——看起来直觉但遇到 "Oh it's not bad"（负负得正）、"It's not good, it's bad one"（双重负）就爆炸。规则加越多越复杂，最后还是无法涵盖所有例外——这精准说明为什么需要 ML：人类自己说不出判断规则时，就让机器从大量 (input, output) 配对中自己学。

从 ML 的数学本质角度，林轩田教授把 ML 类比成数学课学函数推导：给你 f(1)=1, f(2)=4, f(3)=9, f(4)=16，越多 (x, f(x)) 配对越能排除错误候选、越能逼近真实 f。ML 模型就像 Black Box，丢 input 进去会 output 结果，应用从语音辨识、手写辨识、车牌辨识到天气预测、地震预测、股市预测无所不包。

从 AI 与 ML 的关系角度，林轩田教授从维基百科定义出发，把 AI 的三大能力——感知 Perception（语音、影像辨识）、合成 Generation（图片、文本生成）、推论 Reasoning（举一反三）——对应到 ML 的应用场景。但他明确划清界线：ML 是实现 AI 的一条路径，不是唯一路径。传统 AI 的 Game Tree Search（如围棋 AlphaGo）、Expert System 都可以不用 ML。围棋 AlphaGo 第一代结合 Game Tree 加 ML；AlphaGo Zero 纯 ML——这条演进路径本身就是「传统 AI 与 ML 典范转移」的最佳教材。

从跨领域 AI 应用角度，林轩田教授用台风预测案例总结：纯 ML 不够，需要领域专家的 Human Knowledge 介入。气象学家知道南北半球云图旋转方向不同、需要参考多张图——这些隐性知识不是「无脑丢资料」就会学到。AlphaGo 开发时也大量依赖围棋高手的指导——这也是为什么业界的 AI 应用永远是「领域专家 + AI 工程师」的合作模式。

一句话总结：FAI 1.2 给出 ML 最核心的定义——Skill 必须可量化、资料必须有 Pattern、ML 不是唯一 AI 路径、跨领域必须有 Human Knowledge 介入——这四个原则也是后续 FAI 1.3-1.6 整个监督式学习章节的理论基础。
