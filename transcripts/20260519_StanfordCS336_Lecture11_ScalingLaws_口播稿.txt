Stanford CS336 Lecture 11 由 Tatsu Hashimoto 主講,是 Lecture 9「Scaling Laws」基礎篇的進階延伸。Lecture 9 建立了 loss 是模型大小、訓練 tokens、訓練算力的冪律函數這個基礎;Lecture 11 把視角拉到 2024 到 2026 年 open-source 模型實際訓練 scaling 是怎麼做的,聚焦 MiniCPM、DeepSeek、Hunyuan、LLaMA 3 這四個代表。Tatsu 開宗明義點出,scaling laws 在 initial presentation 看似科學——畫一條線、做步驟就能預測大模型表現——但實際操作中非常 messy,沒有所謂的銀彈。

第一條主軸是 MiniCPM 的 muP 與 WSD 策略。muP 是 Maximal Update Parametrization 的縮寫,設計目標是讓 optimal learning rate 在 scale up 與 scale down 時保持穩定。MiniCPM 用 5x 差距的 scaling ladder,從小模型推到目標 5x 大小,不需要 brute-force 訓練數百個模型。muP 做的事包含 scale embedding output、residual connections 用 layer 數的平方根、matrix initialization 用 fan-in 與 fan-out ratio、引入 per-parameter learning rates、以及 scale LM heads。實驗顯示 optimal learning rate 在不同 model size 下都接近 10 負 2 次方,minimal loss curve 乾淨漂亮。MiniCPM 也確認 optimal batch size 隨 tokens processed 呈冪律變化,跨不同 model sizes 都成立——與 Kaplan 2020 的 critical batch size scaling 一致。

MiniCPM 第二個關鍵是 WSD schedule,全名 Warmup-Stable-Decay。三段式:warmup 升溫、穩定 plateau、最後 annealing decay。WSD 比 cosine schedule 更乾淨,final annealing 對最終表現非常關鍵。WSD 也讓 Chinchilla 分析更容易:跑一個 long run,反覆 decay checkpoint 就能 sweep 沿 data dimension。MiniCPM 重現 Chinchilla method 1 與 method 3 雖 exponent 與原始 Chinchilla 不同,但 scaling curves 看起來 reasonable。Tatsu 把 muP 與 WSD 列為研究者必備知識。

第二條主軸是 DeepSeek 的對比策略。DeepSeek 不用 muP,而是透過大量 grid search 跑 scaling laws,把 optimal learning rate 與 optimal batch size 都 fit 成 nonembedding FLOPs 的冪律。執行方式是跨多個 scales 跑 extensive grid search,掃描 LR 與 batch size 的二維空間,對每個 scale 找最低 loss 的組合。然後把 optimal batch size 對 nonembedding FLOPs 畫成線,得到 power law fit;同樣對 optimal learning rate 也 fit 一條線。DeepSeek 結果顯示,較高的 FLOPs 對應較高的 batch size 與較低的 learning rate,實際跑的 full runs 都符合 fit 預測。Tatsu 對 DeepSeek LLM 評價極高——即使在 R1 之前的 paper,就已經是 open world 中做得最好的 scaling analysis 之一。

第三條主軸是 MoE 與 sparsity 對 scaling laws 的影響。切換到 MoE 後,scaling laws 要重新做——重點不是 total parameters 而是 active parameters。給定 FLOPs,sparsity 越高 validation loss 越低,但 quantitative 關係讓人可以理性選 sparsity。Hunyuan 固定 sparsity level,最終用 96 data points per active parameter ratio。LLaMA 3 也做 IsoFLOP analysis,得到與 Chinchilla 相似但稍有不同的 token-to-model ratio;Tatsu 認為 LLaMA 3 的 scaling laws「不是特別 useful」但 illustrative。

最後是 Optimizers 的 scale sensitivity 與 muP 的限制。Optimizer 對 scale 非常敏感——initialization、learning rate、batch size 都需要當作 scale 的函數來思考。muP 不是萬靈丹——Lion optimizer 精神上類似 sign gradient 的 Muon,會 break muP 的 scaling assumptions;large decoupled weight decay 是 muP 最明顯的 failure mode。Tatsu 對 scaling laws 的最終結論很務實:scaling laws 在現實中不是科學而是 art,沒有 silver bullet。muP 是 interesting open area of research 有 promise;fitting scaling laws 也有 promise。Lecture 11 結束時 Tatsu 留下一句「maybe next year, there will be a module that's like, we solved it, but not quite yet」——scaling 的故事還沒寫完。
