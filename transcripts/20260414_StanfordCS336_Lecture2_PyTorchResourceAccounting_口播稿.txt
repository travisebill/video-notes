CS336 第二講由 Percy Liang 主講,主題是 PyTorch 基礎與 resource accounting。開場先報告好消息:Moraine 計畫預訓練的大模型 loss 與預測值差距僅 0.05——這證明 scaling law 的外推有效,他們甚至公開了外推到 GPT-5 等級的預測 loss 數字。

本講要建立的核心 mindset 是:每寫一行 code,先想它的性能特徵。Percy 用了兩個經典問題開場。第一個是「70B 參數模型在 15T tokens 上用 H100 訓練要多久?」答案透過 6ND 公式算 FLOPs、查 spec sheet 拿 H100 throughput、再乘 MFU 0.5 估實際效率,得到 143 天。第二個是「8 張 H100 用 AdamW 能 train 最大的 model?」每個參數要 12 bytes(2 + 2 + 4 + 4,BF16 參數加梯度加兩個 FP32 optimizer state),80GB HBM 算出上限約 53B 參數。這些都是 back-of-envelope 計算,重點是掌握 rough shape,不是精確數字。

進入技術細節。Tensors 是 LM 的萬物基礎——參數、gradient、optimizer state、data、activation 全部都是 tensor。DeepSeek 3.2 為例,每個矩陣有 shape 與 precision,模型就是一堆 tensor 的集合。Float32 是預設,但現代訓練已走向低精度:BF16 是 sweet spot,因為 1 + 8 + 7 bits 結構讓 dynamic range 同 FP32 但精度同 FP16。FP8 有兩個變體——E4M3 偏 dynamic range、E5M2 偏 resolution——NVIDIA Transformer Engine 內建支援。最激進的是 FP4,NeMo-3 Super 2025 年已實際用 NVFP4 訓練,透過 block 共享 scale 來擴展表示範圍。

Einops 是本講最重要的工具。Percy 用 einsum notation 取代傳統 matmul 寫法,因為 named dimensions 讓 broadcasting 與 chain rule 不再出錯。例如 forward 是 `Y[b,k] = einsum("b d, d k -> b k", X, W)`,backward 算 X 的梯度時只要記得 scalar case 倒推——`X.grad[b,d] = einsum("b k, d k -> b d", Y.grad, W)`——永遠對。學會 einops 後,transformer 的 QKV attention 與 MLP 都可以乾淨表達。

Matmul FLOPs 的核心公式是 2BDK——B 是 batch、D 是 input dim、K 是 output dim,每個 output element 做 D 次乘加。換個角度:matmul FLOPs = 2 × tokens × params。延伸到整個 LLM,forward pass 2ND、backward pass 約 4ND,總計 6ND——這就是業界常用的 6ND 公式由來。

MFU(Model FLOPs Utilization)是實際 FLOPs/sec 除以承諾 FLOPs/sec。Spec sheet 寫 H100 有 989 TFLOPs BF16,實際可能只有一半——0.5 MFU 算優秀,純 matmul 可能達 0.8,0.1 表示嚴重問題。為什麼拿不到 100%?因為大部分運算落在 memory-bound 區,不是 compute-bound。

Roofline analysis 把這件事講清楚。Arithmetic intensity = FLOPs 除以 bytes——也就是每搬一個 byte 的資料要做多少次計算。低 intensity 的運算(像 dot product、element-wise)被記憶體頻寬限制,realized FLOPs 跟 intensity 線性成長;高 intensity(像大矩陣 matmul)被計算能力限制,realized FLOPs 飽和在 peak。B200 spec 是 2.25 PFLOPS BF16 配 8 TB/s memory bandwidth,這兩個數字決定 roofline 的轉折點。

Backward pass 需要算兩個 gradient:對 input 的梯度、對參數的梯度,兩者都是 2BDK FLOPs,所以 backward 約等於 2 倍 forward。一個 layer 總計 6BDK FLOPs。延伸到 LLM 就是 6ND。Optimizer state 方面,AdamW 每個參數 12 bytes 是固定成本——H100 80GB 不計 activation 最多只能裝 6.6B 參數,所以大模型必須靠 sharded optimizer、gradient checkpointing 等技巧突破。

Deep network 的另一個記憶體殺手是 activation——隨 layer 數線性增長。Forward pass 的中間值必須保存供 backward 使用,可以用 gradient checkpointing 換記憶體換 compute,只重算部分 activation。

一句話總結:這堂課教的是 LLM 工程師的核心 mindset——把每一個 op 拆成 FLOPs 與 bytes 兩個維度思考,選擇落在 roofline 飽和區的演算法,用 einops 寫出無歧義的 forward/backward,這才是從能跑到能 scale 的底層功夫。
