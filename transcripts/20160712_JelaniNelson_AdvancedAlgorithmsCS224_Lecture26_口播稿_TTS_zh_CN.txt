这是 Harvard CS224 高阶演算法第二十六讲，主题是串流演算法与线性草图：从 F0 distinct elements 到 F2 L2 norm estimation 的 AMS 草图。

Section 1｜串流模型回顾
教授开场复习串流模型：输入是长度 m 的更新序列 x_i 变成 x_i 加 v，初始为零向量。要计算 x 的某个函数 f 而无法储存整个向量。典型问题是支援大小 F0：向量中有多少非零座标，等同于路由器一天收到多少不同 IP 位址。最直接的演算法用 n 位元记录每座标是否出现过，或用 m 对数 n 位元储存所有看过的元素。要做得更好需要随机化与近似：deterministic 演算法对 F0 需要 omega(min(n,m)) 位元，无法超越线性。

Section 2｜Random Hash 估计 F0
下界来自编码论证：若存在少于 n 位元的串流演算法，则可压缩任意 n 位元字串成更短，违反鸽笼原理。F0 的随机近似演算法是 Bar-Yossef 与 Kumar 的：对串流中每个元素 i 用随机 hash 函数 h 映射到零到一区间，记录所有 hash 值中的最小值 z。期望上 z 约等于 1 除以 F0+1，所以 F0 的估计就是 1 除以 z 减 1。期望估计的变异数还是有常数倍，用中位数技巧取 R 个独立 hash 函数平均，可把变异数压到 1+epsilon 倍，这就是 AMS 草图的核心想法。

Section 3｜L2 Norm 与 Linear Sketching
教授转到 L2 norm 估计：给定 x 的 L2 norm 平方，估计到 1+epsilon 倍误差，空间远小于 n。L2 norm 不能像 F0 一样只记录最小值，因为删除时最小值失去意义。教授介绍 Linear Sketching 框架：选一个 m 乘 n 的矩阵 Pi，更新时把 y 加 v 乘以 Pi 的第 i 行，无论插入或删除都成立。Lyu 与 Woodruff 证明任何支援删除的串流演算法都可转成 linear sketch，空间最多差对数 n 倍。对 L2 norm，Pi 的每个条目是独立的正负一除以根号 n。

Section 4｜AMS 演算法分析
AMS 演算法存的是 y 等于 Pi 乘以 x，L2 norm 估计为 y 的 L2 norm 平方。分析：y_r 是 1 除以根号 n 乘以 sigma_ri 乘 x_i 之和，所以期望 y_r 平方等于 1 除以 m 乘以 x 的 L2 norm 平方，期望 y 的 L2 norm 平方等于 x 的 L2 norm 平方。变异数：y_r 平方的变异数来自四阶矩 sigma 平方，计算后变异数是 O(1) 乘以 x 的 L2 norm 4 次方除以 m。取 m 等于 1 除以 epsilon 平方即可让变异数被期望值压到 epsilon 平方倍，由 Chebyshev 不等式得到 1+epsilon 估计，失败机率 1/3。

Section 5｜失败机率降低与维度缩减
失败机率 1/3 不够好，做两件事来降低：把 m 增为对数 1 除以 delta 个独立副本取中位数，或对单一草图做多次 random sign 的平均。教授简介快速回归与点查询：给定矩阵 A，要在串流中估计 A 乘以 x 的某个座标。使用 Johnson-Lindenstrauss 维度缩减，把 x 投影到 d 等于对数 n 除以 epsilon 平方 维空间，于是可在 d 维空间做估计，空间复杂度大幅下降。

Section 6｜结论与延伸
这堂课展示了串流演算法的核心思想：当资料量太大无法储存时，用随机 hash 或线性草图压缩到小空间，再从草图估计所需的统计量。F0 用最小 hash 值估计支援大小；L2 norm 用 AMS 草图估计能量；维度缩减把高维向量降到对数维度。教授预告 CS 229r 会更深入探讨点查询、稀疏复原、以及矩阵与图的串流模型。
