Stanford CS336 Lecture 17 把 Post-Training 拉到 Multimodal Alignment。Percy Liang 开宗明义:transformer 仍是最通用的跨模态 backbone,关键是把所有 modality 变成 transformer 吃得下的 tokens;文字已是 subword tokens,图像要 patch-based tokenization。沿 2021 到 2025 时间轴展开。

CLIP 在 2021 由 OpenAI 提出,是现代 VLM foundation:用 400M image-text 对做 contrastive learning,对齐 dot product 再 softmax,拉近配对推开非配对;图像编码器用 ViT-L/14 加 attention pooling,zero-shot 在 ImageNet 打败 ResNet trained on 1.2M labels——但 CLIP 有两个痛点:30K+ 大 batch 是必要条件、softmax over 整 batch 不可平行化。Google 的 SigLIP 把 softmax 改成 sigmoid binary classification,对角线 +1、其他 −1,loss 完全跟 batch size 解耦,16K batch 就赢 CLIP;WebLI dataset 达 1B 到 10B image-text pairs,含 OCR,训练比 CLIP 快数倍。LLaVA 2023 把 CLIP 加 linear projector 加 Vicuna LLM 拼起来,两阶段训练:Stage 1 只 train projector,Stage 2 train projector 加 LLM,158K 用 GPT-4 从 MS COCO 生成的对话 examples,证明 open model 可做视觉推理。

LLaVA OneVision 2024 升级三大件:用 SigLIP 取代 CLIP、Qwen-2 取代 Vicuna、projector 改两层 MLP,推出 AnyRes 动态解析度处理 OCR——把大 image 切成多块 336×336 各自编码再 concate vectors,语言 transformer 本来就能处理变长输入。Qwen-VL 系列三代演进:Qwen-VL 用 OpenCLIP 加单层 cross-attention adapter,固定 256 tokens;Qwen-2-VL 推到 dynamic resolution 加 Multimodal RoPE 把 RoPE 推到 3D height 乘 width 乘 time;Qwen-3-VL 用 SigLIP-2 改进 M-RoPE 交错 three-axis dimension 避免低/高频聚集,explicit video timestamp tokens 可被 reference,用 √L per-token loss normalization 避免 long video 压过 short text,DeepStack adapter 把 vision encoder 每一层 output 直接加入 LLM residual stream 做 deep fusion。

Chameleon 2024 走另一条路:VQ-VAE 把 image 变 1024 个 discrete tokens 从 8000 codebook,然后全部 LLM 一个模型处理 text 加 image,但 training 不稳定——text token 低 entropy、image token 高 entropy,混在一起 norm 会爆掉,只能 QK norm 加 z-loss regularization 缓解;discretization 也让 OCR 细节掉光。Percy 最后反思理解模态跟生成模态本质不对称:CLIP 等 high-level semantic encoder 对 classification 够用,但 OCR 或 image generation 需要 fine-grained detail,必须 diffusion 才能做;没有 single universal encoder 能 cover 所有需求,这是 multimodal 设计的根本限制。Vision encoder 参数通常小于 1B,patch 是 local operation,大部分知识仍在 LLM。
