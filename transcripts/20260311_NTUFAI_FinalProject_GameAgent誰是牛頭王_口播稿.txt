本集介紹台大資訊系「人工智慧導論（FAI 2026）」的 Final Project 說明會，由四位教學 TA 主講，主題為設計一個玩德國卡牌遊戲「誰是牛頭王」的 Game Agent。

遊戲規則相對單純：四位玩家、每場各發十張牌、共十輪，每輪各出一張牌依數字大小排序進到四個牌堆之一。卡牌的尾數決定了 Penalty 數量：尾數五是兩顆牛頭、尾數零是三顆、結尾是零零多達五顆、最危險的結尾五五則高達七顆。十輪結束時，牛頭總數最少的玩家勝出。然而真正變化莫測的是收牌機制：當某個牌堆已經累積五張牌時，下一個出牌者會成為第六張玩家，被迫收走整堆六張牌作為 Penalty。另一方面，若手牌太小、無法接續任何牌堆結尾，演算法會自動挑選一個最不虧的牌堆先丟掉，再把這張小牌放到堆頂。

Game Engine 是這份作業的核心，由三段式架構組成：Initialization 負責發牌與確認玩家名單、PlayGame 跑完整場十輪遊戲、PlayRound 則是每輪的核心迴圈，負責呼叫每位 Agent 的決策函式並執行四項檢查：決策時間不得超過一秒、輸出必須是合法整數、記憶體使用不得超過一 GB、且禁止任何額外的 Multi-process 或 Multi-thread。違規處置分為兩層；偽違規如嘗試竄改 Game History 或 Random Seed 不會扣分，因為系統每次都傳遞複製版、並使用自製 RNG，重設完全無效；但真違規如記憶體爆炸導致 Process 被 kill、額外 spawn 子行程、或故意 catch TimeoutException 來規避時間限制，會分別被扣最高 50% 到 70% 的分數。

錦標賽賽制方面，系統將全體學生 Agent 隨機四人分成 Partition，每個 Partition 跑多場對局，全場至少跑五百個 Partition 以消除隨機波動。每場對局的四個名次會換成分數，平手時平均分配，整場總分取所有場次的平均 Rank，再線性內插到零到一百分。評分政策分為兩階段：四月二十九日的 Mid Submission 占十五分，跑四十個 Face Line；六月十四日的 Final Submission 占五十五分，跑五十五個 Face Line。線性內插的分數門檻設計相當合理：Mid 階段贏過第五條 Face Line 保底六十分、贏過第二十條可達九十分；Final 階段贏過第二十條保底四十分、贏過第五十條可達九十分。

額外加分機制也值得一提：當某位 Agent 在同一個 Board 上面對所有對手的決策分布明顯不同、卻仍能勝出時，最多可額外加五分。這項 Novelty 加分採取主觀與量化雙重標準，TA 主觀看 Report 加上系統跑 script 比對所有 Agent 的決策機率分布，避免全班策略過於同質化。

開發環境條件嚴格：Python 三點一三點一一、CPU only、無 GPU、無網路、評分環境保證與測試機同級但可能更強。寫 Agent 的介面極為精簡，只需要實作一個 Player Class，含兩個函式：初始化函式接收玩家編號、決策函式接收當前手牌與完整遊戲歷史並回傳要出的牌。學生特別需要注意兩件事：手牌以 Reference 傳遞不可直接修改、不要自行儲存內部狀態，每次都從傳入的歷史重算所有需要的數值，這是為了防止 Process 被 kill 後狀態不一致造成後續回合崩潰。

提交時每位學生需繳交兩個最佳 Agent、所有相關 code、以及一份不超過三頁的 Report，內含嘗試過的方法、為何選擇這兩個 Agent、未來改進方向等。整份 Project 的精髓在於以工業級嚴苛條件為框架，把課堂所學的 Exploitation 與 Exploration、Search、Reinforcement Learning 概念壓縮到一秒決策與一 GB 記憶體上限的真實對戰之中。
