台大資訊工程學系 2026 年春季「人工智慧導論」FAI 1.3「請按照步驟來用機器學習」，由林軒田教授主講，約 25 分鐘。本講的核心任務是把 ML 整理成一套可操作的標準化 Recipe。

從工程實踐角度，林軒田教授用「申請工作」這個生活化範例貫穿全講——履歷（features）轉成公司好壞（label）找出 pattern 用 ML 學習。給定申請者的履歷，判斷是否適合公司；履歷包含年紀、性別、實習次數、project、主題、GPA 等 6 維特徵，每個申請者是 6 維空間中的一個點。Input 是履歷 feature vector，Output 是適合或不適合的 binary 標籤，目標是找到一個 function 把輸入對應到輸出，假設這樣的 pattern 存在。

從 Recipe 步驟 0 角度，林軒田教授強調這個步驟最關鍵：判斷任務適不適合用 ML。三個前提必須都滿足：第一，必須有 Pattern 可學——如果輸入與輸出完全 random，就不該用 ML。第二，Pattern 無法或很難用 Rule 寫——餐廳評價分類就是經典例子，規則加再多都涵蓋不了所有例外。第三，要有 Data——沒有資料 ML 無用武之地。努力與中樂透沒有 pattern，不該用 ML；實習次數大於等於 2 就 accept，可以直接寫 Rule 也不需要 ML。

從 Recipe 步驟 1 角度，Data 是 (x, y) 的配對，資料的 quality 與 quantity 決定 ML 的上限。林軒田教授引用 2010 KDD Cup 為例：當年題目是給定學生特徵加考題特徵，預測學生能不能答對題目，當年台大團隊拿世界冠軍。訓練完的 model 會在 unseen test data 上評估 generalization。

從 Recipe 步驟 2 角度，Hypotheses Set 的選擇就是 Model 的選擇。Hypothesis Set H 是所有可能的候選函數集合——例如「只要 GPA 大於 4.0 就 accept」、「只要有 NLP 相關 project 就 accept」、「H1 AND H2」等都是不同 Hypothesis。現實中 H 非常大，幾乎無限多。Linear Model 的 H 是所有 hyperplane 的集合，這也是選擇 Linear Model 等於把候選函數限制在直線 / hyperplane 集合中。Linear Model 簡單但有侷限性——XOR 不能用單一 Perception 分。多層 Perception（Multi-Layer Perceptron）可以突破：每一層都是 Linear，多層疊起來可以表示 XOR——這也是 Deep Learning 的理論基礎。

從 Recipe 步驟 3 角度，Error Measure 量化 g 與 f 的差距。Pointwise Error 對每個點算 error 加起來平均，In-Sample Error 是在 training data 上的 error，Out-of-Sample Error 是在 unseen data 上的 error。我們能 minimize 的是 E_in，因為看不到 E_out。分類任務常用 0/1 Error，回歸任務常用 Squared Error，Logistic 用 Cross-Entropy。林軒田教授用「前 2 年考古題讀書、第 3 年考古題測試」這個類比，讓學生直觀理解 E_in 跟 E_out 的差異。

從 Recipe 步驟 4-5 角度，Training 是給定 D、H、Error Measure，用 algorithm 從 H 中找 g minimize E_in；Testing 是給 unseen D' 用 g 預測算 E_out。最終目標不是 E_in 最小化而是 E_out 最小化——也就是 Generalization。Linear Model 在高維空間的限制是某些 distribution 切不開，例如 XOR。這就是為什麼需要 Deep Learning——很多層簡單的 Linear 疊起來可以學到非常複雜的 function。

從實務角度，這套 Recipe 是後續 FAI 1.4-1.6 所有具體模型的標準操作框架。FAI 1.4 會進入 Learning Algorithm（PLA、Gradient Descent）；FAI 1.5-1.6 會把 Recipe 套用到 Linear Regression 與 Logistic Regression。這也是為什麼業界 ML 專案永遠從 Recipe 步驟 0 開始——先想清楚 ML 適不適合，比寫 model.fit() 更重要。

一句話總結：FAI 1.3 把 ML 從「神奇的黑盒子」變成「可操作的工程流程」——五大步驟（判斷、Data、Hypotheses、Error、Learning）每一步都有明確的設計決策，最終目標是 E_out 最小化而非 E_in 最小化——這也是整個 Machine Learning 章節的理論骨架。
