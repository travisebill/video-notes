Stanford CS336 Lecture 11 由 Tatsu Hashimoto 主讲,是 Lecture 9「Scaling Laws」基础篇的进阶延伸。Lecture 9 建立了 loss 是模型大小、训练 tokens、训练算力的幂律函数这个基础;Lecture 11 把视角拉到 2024 到 2026 年 open-source 模型实际训练 scaling 是怎么做的,聚焦 MiniCPM、DeepSeek、Hunyuan、LLaMA 3 这四个代表。Tatsu 开宗明义点出,scaling laws 在 initial presentation 看似科学——画一条线、做步骤就能预测大模型表现——但实际操作中非常 messy,没有所谓的银弹。

第一条主轴是 MiniCPM 的 muP 与 WSD 策略。muP 是 Maximal Update Parametrization 的缩写,设计目标是让 optimal learning rate 在 scale up 与 scale down 时保持稳定。MiniCPM 用 5x 差距的 scaling ladder,从小模型推到目标 5x 大小,不需要 brute-force 训练数百个模型。muP 做的事包含 scale embedding output、residual connections 用 layer 数的平方根、matrix initialization 用 fan-in 与 fan-out ratio、引入 per-parameter learning rates、以及 scale LM heads。实验显示 optimal learning rate 在不同 model size 下都接近 10 负 2 次方,minimal loss curve 干净漂亮。MiniCPM 也确认 optimal batch size 随 tokens processed 呈幂律变化,跨不同 model sizes 都成立——与 Kaplan 2020 的 critical batch size scaling 一致。

MiniCPM 第二个关键是 WSD schedule,全名 Warmup-Stable-Decay。三段式:warmup 升温、稳定 plateau、最后 annealing decay。WSD 比 cosine schedule 更干净,final annealing 对最终表现非常关键。WSD 也让 Chinchilla 分析更容易:跑一个 long run,反复 decay checkpoint 就能 sweep 沿 data dimension。MiniCPM 重现 Chinchilla method 1 与 method 3 虽 exponent 与原始 Chinchilla 不同,但 scaling curves 看起来 reasonable。Tatsu 把 muP 与 WSD 列为研究者必备知识。

第二条主轴是 DeepSeek 的对比策略。DeepSeek 不用 muP,而是透过大量 grid search 跑 scaling laws,把 optimal learning rate 与 optimal batch size 都 fit 成 nonembedding FLOPs 的幂律。执行方式是跨多个 scales 跑 extensive grid search,扫描 LR 与 batch size 的二维空间,对每个 scale 找最低 loss 的组合。然后把 optimal batch size 对 nonembedding FLOPs 画成线,得到 power law fit;同样对 optimal learning rate 也 fit 一条线。DeepSeek 结果显示,较高的 FLOPs 对应较高的 batch size 与较低的 learning rate,实际跑的 full runs 都符合 fit 预测。Tatsu 对 DeepSeek LLM 评价极高——即使在 R1 之前的 paper,就已经是 open world 中做得最好的 scaling analysis 之一。

第三条主轴是 MoE 与 sparsity 对 scaling laws 的影响。切换到 MoE 后,scaling laws 要重新做——重点不是 total parameters 而是 active parameters。给定 FLOPs,sparsity 越高 validation loss 越低,但 quantitative 关系让人可以理性选 sparsity。Hunyuan 固定 sparsity level,最终用 96 data points per active parameter ratio。LLaMA 3 也做 IsoFLOP analysis,得到与 Chinchilla 相似但稍有不同的 token-to-model ratio;Tatsu 认为 LLaMA 3 的 scaling laws「不是特别 useful」但 illustrative。

最后是 Optimizers 的 scale sensitivity 与 muP 的限制。Optimizer 对 scale 非常敏感——initialization、learning rate、batch size 都需要当作 scale 的函数来思考。muP 不是万灵丹——Lion optimizer 精神上类似 sign gradient 的 Muon,会 break muP 的 scaling assumptions;large decoupled weight decay 是 muP 最明显的 failure mode。Tatsu 对 scaling laws 的最终结论很务实:scaling laws 在现实中不是科学而是 art,没有 silver bullet。muP 是 interesting open area of research 有 promise;fitting scaling laws 也有 promise。Lecture 11 结束时 Tatsu 留下一句「maybe next year, there will be a module that's like, we solved it, but not quite yet」——scaling 的故事还没写完。
