台大資訊工程學系 2026 年春季「人工智慧導論」FAI 1.2「你不能到現在還不懂甚麼是機器學習」，由林軒田教授主講，約 26 分鐘，是 Machine Learning 章節的第二講。

從學習的本質角度，林軒田教授從日常的「學習」這個詞彙切入，把人類學習——透過觀察累積 Skill——對應到機器學習——透過資料學到 Skill。人類透過寫程式、跑實驗、觀察輸出，從觀察中累積 Skill；機器沒有眼耳直接接收感知，所以是從 Data 學。把人類的 Observation 完整收集起來變成 Data，讓機器在這些資料上做學習，這是 ML 的本質定義。圖示上，人類 Learning 等於大量 Observation 變成 Skill，機器 Learning 則是把 Observation 換成 Data。

從驗證 Skill 的方法角度，林軒田教授指出 ML 與人類學習共享同一個評估邏輯：透過 Performance Measure 量化 Skill 是否提升。人類驗證 Skill 用考試——學完滑雪課後會不會滑雪？學完數學後考卷分數有沒有提升？ML 同理：給機器一個考卷（test data），看 performance 有沒有改善。但他也提醒，考試可能無法反映真實能力——這是為什麼 evaluation 設計本身就是重要的研究問題。

從 ML 適用情境角度，影片用兩個生活化例子對比。股市預測：股市達人看 K 線圖決定買賣，雖然說不出完整論述但必定有 Pattern 可學，這是適合 ML 的情境。餐廳評價分類：一開始寫 rule-based 方法——if 包含 love 就是正面、if 包含 hate 就是負面——看起來直覺但遇到 "Oh it's not bad"（負負得正）、"It's not good, it's bad one"（雙重負）就爆炸。規則加越多越複雜，最後還是無法涵蓋所有例外——這精準說明為什麼需要 ML：人類自己說不出判斷規則時，就讓機器從大量 (input, output) 配對中自己學。

從 ML 的數學本質角度，林軒田教授把 ML 類比成數學課學函數推導：給你 f(1)=1, f(2)=4, f(3)=9, f(4)=16，越多 (x, f(x)) 配對越能排除錯誤候選、越能逼近真實 f。ML 模型就像 Black Box，丟 input 進去會 output 結果，應用從語音辨識、手寫辨識、車牌辨識到天氣預測、地震預測、股市預測無所不包。

從 AI 與 ML 的關係角度，林軒田教授從維基百科定義出發，把 AI 的三大能力——感知 Perception（語音、影像辨識）、合成 Generation（圖片、文本生成）、推論 Reasoning（舉一反三）——對應到 ML 的應用場景。但他明確劃清界線：ML 是實現 AI 的一條路徑，不是唯一路徑。傳統 AI 的 Game Tree Search（如圍棋 AlphaGo）、Expert System 都可以不用 ML。圍棋 AlphaGo 第一代結合 Game Tree 加 ML；AlphaGo Zero 純 ML——這條演進路徑本身就是「傳統 AI 與 ML 典範轉移」的最佳教材。

從跨領域 AI 應用角度，林軒田教授用颱風預測案例總結：純 ML 不夠，需要領域專家的 Human Knowledge 介入。氣象學家知道南北半球雲圖旋轉方向不同、需要參考多張圖——這些隱性知識不是「無腦丟資料」就會學到。AlphaGo 開發時也大量依賴圍棋高手的指導——這也是為什麼業界的 AI 應用永遠是「領域專家 + AI 工程師」的合作模式。

一句話總結：FAI 1.2 給出 ML 最核心的定義——Skill 必須可量化、資料必須有 Pattern、ML 不是唯一 AI 路徑、跨領域必須有 Human Knowledge 介入——這四個原則也是後續 FAI 1.3-1.6 整個監督式學習章節的理論基礎。
