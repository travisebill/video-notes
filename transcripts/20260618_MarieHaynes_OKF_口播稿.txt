Marie Haynes 在 6 月 16 號發布的影片中，拆解了 Google 剛公告的 Open Knowledge Format（簡稱 OKF），一個用 Markdown 檔案承載企業或個人專業知識的開放標準。這部影片的核心張力在於：OKF 的技術門檻極低（只是 Markdown 檔案），但它代表的典範轉移卻極為深遠——從「SEO 讓人找到你」轉向「讓 agent 能讀懂你的企業知識」。

從資訊架構角度，OKF 的本質是形式化 Andrej Karpathy 早在 LLM Wiki pattern 中提出的觀念：與其讓語言模型在查詢時從大量文件中抓資料，不如讓它維護一份持續更新的 wiki。Google 這次做的事，是把這個觀念做成可交換的開放標準。Marie 的影片裡有一段相當關鍵的引述——Karpathy 主張「語言模型不只是把來源索引起來供未來查詢；它會讀取、萃取關鍵資訊、整合進既有 wiki，更新實體頁面、修改主題摘要、標記新資料與舊論點矛盾之處」。

換句話說，OKF 解決的不是 RAG 的問題，而是「人類如何組織自己的知識給機器」的問題。Marie 從 SEO 顧問的視角特別強調：過去她對結構化資料（schema）一直很掙扎，因為把每個實體都 mapping 出來感覺違反人類自然組織知識的方式。OKF 的「以概念為單位」（每個 Markdown 檔案 = 一個知識單位，而非一個網頁）正好回應這個痛點。從知識管理學的角度，這呼應了 long-form 知識本來就以「概念節點」存在的特性，而非以「網頁」為單位。

從經濟學角度，Marie 在影片中明確點出 OKF 將催生兩條新營收路徑。第一條是服務面：把 OKF 建置服務賣給企業，理解企業、幫忙把概念組織成 knowledge bundle。這跟早期 SEO 顧問幫忙建 sitemap 是同一層次，但難度更高，因為要「理解企業」而非只是「讓搜尋引擎看懂」。第二條是商品面：銷售 OKF bundle 本身。Marie 給的具體圖像是「律師、會計師、SEO 顧問可以打包自己的專業知識出售」，別人的 OKF 可整合進自己的系統，不必再用 MCP 或 Web MCP 連到對方網站。隨著專業領域更新，自動吸收新版本。這與「知識即服務」（Knowledge-as-a-Service）的商業模式完全契合——從知識管理學的觀點，這正是 Peter Drucker 早年預言的「知識工作者」經濟的具體實現：知識不再依附於個人時間，而是被打包成可交易、可整合的商品。

不過 Marie 的解方並非沒有盲點。她低估了一個關鍵問題：概念邊界的判定高度仰賴人類對企業的深度理解。她舉的 playbook 範例（trigger 型知識）雖然實用，但「哪些 trigger 屬於同一個 playbook」、「trigger 與 trigger 之間的優先序」這些問題，並非丟給 LLM 就能自動解決——這需要領域專家花時間釐清。此外，她對「每個概念都是一個 markdown 檔」的描述雖然簡潔，但實際企業知識常有層級關係（概念 A 是概念 B 的子集），用平鋪的 markdown 檔案結構是否能承載複雜的知識圖譜，是個值得持續觀察的實作問題。

從比較觀點看，OKF 與既有的 RAG、fine-tuning、prompt engineering 三條 AI 知識整合路徑形成有趣的對照。RAG 是查詢時即時抓資料；fine-tuning 是把知識燒進模型權重；prompt engineering 是用提示引導模型表現。OKF 走第四條路：在推理時動態讀取「外部持續維護的結構化知識庫」，且這個知識庫由人類與 LLM 共同維護。這條路徑的最大優勢是「知識可即時更新」——法律變了、新產品上市了，你更新你的 OKF bundle，所有整合這份 bundle 的 agent 立即看到新版本。

影片接近結尾時，Marie 引用了 Gemini 給她的一句話：「semantic unbaking」（語意反烘焙）。這是一個非常精準的隱喻：過去人類為了讓機器讀懂，必須把自己的知識「烘焙」成機器可讀的格式（schema、API、SQL）；OKF 反過來，讓人類用自己的語言組織知識，機器自己來讀懂。Marie 最後的願景是：未來人類可以過著「學習並分享」的生活，努力培養專業的人會被獎勵，因為他們的知識 bundle 真的會被需要。

這場 Google 推出 OKF 的真正遺產，不在技術細節（Markdown 真的不難），而在它把 Karpathy 的 LLM Wiki 觀念做成可交換標準、並且重新定義「知識工作者」在 AI 時代的價值交換方式。Marie Haynes 這支影片最鮮明的訊息是：SEO 不再只是被找到，而是讓你的專業對 agent 可用；懂 OKF 的人，未來幾年會是高需求人才。
