这段节目是 FAI 第 3 章的尾声，处理深度学习中的最佳化问题——给定 Loss Function 与模型结构后，要如何实际把参数 θ 推向最小值。林轩田教授把焦点放在三个关键的工程变量：起步位置、步伐大小、与每一步看到的样本数。

第一个问题是「滚球会卡在局部低谷」。影片把 Gradient Descent 比喻为沿著 Loss 地形图滚下的小球，从某个起点开始，计算当下位置的切线斜率，若斜率为负就往 θ 增加的方向滚一步；若斜率为正就往 θ 减少的方向滚。最终小球会停在斜率为零的位置，也就是局部低谷。然而深度学习的 Loss 地形远比线性模型来得崎岖——层数一多，局部低谷的数量呈指数级增加；从不同起点出发，最终停下的位置也会截然不同。这就是为什么深度学习对「初始化点」极为敏感。实务上的解法是在多个随机起点上各跑一次，最终选 Loss 最低的那一个 θ 作为候选解。

第二个问题是「步伐大小」。学习率（Learning Rate）若设得太大，参数可能直接跨过低谷，反而让 Loss 上升；若太小，虽然方向正确，却会在低谷附近卡住，动弹不得。理想的策略是「远离低谷时大步走，接近低谷时小步走」——也就是自适应（adaptive）的学习率调整。现代最佳化器如 Adam、RMSProp 内建了这类机制，自动根据梯度历史调整步伐；若使用固定学习率，则需要手动监控训练曲线、逐步下调。

第三个问题是「每一步要看到多少资料」。标准的 Full-Batch Gradient Descent 每次看完所有训练样本才更新一次，方向最精准但速度最慢，尤其当资料量达到上万笔以上时，每一次更新都要运算整体平均，效率极差；SGD（Stochastic Gradient Descent）则每看完一笔就更新一次，更新次数多，但单笔的梯度估计杂讯大；实务上的最佳折衷是 Mini-Batch SGD——一次取 B 笔（例如 32、64、128）平均后再更新，兼顾速度与稳定性，并且能充分利用矩阵与矩阵相乘的硬体加速效益。

影片最后提醒，深度学习的最佳化并不是「找到一个确定最佳的 θ」，而是「尽量少卡在糟糕的局部低谷」。Batch Size 通常建议固定，避免每次重新调整 cache 机制；每个 epoch 内对训练资料做随机洗牌，确保模型学不到资料顺序的伪相关；Learning Rate 则建议参考已发表论文的设定，并结合 gradient clipping 等技巧处理梯度爆炸问题。

整段节目的核心讯息是：深度学习的最佳化既包含梯度下降的数学原理，也叠加了 Learning Rate、Batch Size、随机化与 adaptive optimizer 等多项工程技巧；这些「小撇步」看似琐碎，却是把理论模型推向实际可用的关键所在。
