Stanford CS336 Lecture 16 是 Post-Training 系列第二讲,主轴是 RLVR(Reinforcement Learning from Verifiable Rewards)——Tatsu 从 RLHF 困境出发,指出 reward model 终究会 overoptimization,所以需要找「可程式验证」的 reward 来取代人类偏好标注,这就是 RLVR:reward 直接由答案正确或 unit test 通过与否决定,根本绕开 annotation bottleneck。本讲核心算法是 GRPO——它是 PPO 的精简版,只动两个地方:拿掉 value function,以及用 group z-score 取代 advantage estimation。具体做法是对同 prompt 抽 k 个 rollout,把每个 reward 减掉 group mean 再除 group std,作为 advantage。Online 情况下 clipping 直接消失,objective 化简成「advantage 减 KL penalty」,整个 GRPO 可写在半张 slide 上。DeepSeek Math 论文的实验显示 GRPO 稳定赢 RFT(rejection fine tuning)与 process supervision。

但 GRPO 并非 first-principles policy gradient——Sutton-Barto 说 baseline 可任意相减,只要不改 gradient direction,但 GRPO 用「除以 std」直接违反了 baseline contract。同时其 length normalization 又鼓励错误答案无限延伸。也就是说,长 CoT 变长很可能是 GRPO 的副作用,而不是模型真的变聪明。后续工作直接拿掉这两个 normalization 就能得到更好的行为。

Lecture 16 后半逐一解析四个开源 release 的技术报告。DeepSeek R1 Zero 是纯 RLVR 路线的证明:拿 MIT-trained base model 加 GRPO accuracy reward 加 format reward 直接训练,结果达到接近 OpenAI O1 等级,而且 recipe 简单到可在 assignment 重现;真正 ship 的 R1 是 SFT on long CoT(疑为蒸馏自家另一个更大模型)→ GRPO 加 language consistency reward(避免 CoT 中英混杂)→ RLHF on non-verifiable tasks。R1 paper 中 viral 的「aha moment」与「长 CoT」两个现象,后续工作证实前者 base model 已会、后者是 GRPO 副作用,并非突破本身。Kimi K1.5 同期释出,从 DPO-style 推导出 objective 竟与 GRPO 相似,证明这条设计方向是 robust 的;独特之处是用 length reward 主动压缩 CoT ——对正确答案鼓励短、对错误答案鼓励稍短于 average 但不要无限短,否则 recover 不到正确 reward。

Qwen 3 用 thinking tag 让模型一份权重同时支援 thinking 与 non-thinking 模式,可手动 early-exit,效能随 thinking budget 优雅降级,4K examples 就够做 RL。Coder-Next 是 agent 环境的 RLVR:mid-training 先把 coding/agentic 能力塞进去、SWE-bench 大量合成环境做 RL,但关键警告是 reward hacking——agent 会去看 Git future commits 偷答案,因此必须在 reward 函数中专门禁止 git log 与 remote origin query 等 hack。RLVR 不是银弹:answer equivalence 仍是 deep rabbit hole,length normalization 有副作用,agent 环境的 reward hacking 是长期风险。但整体而言,RLVR 已成为 2024 到 2026 年开源思考模型的标准引擎,从 R1 到 Coder-Next 都建立在同一套 GRPO × verifiable reward 的基础上。
