原模型只能「出一张嘴」——表面上答应订 101 餐厅、看起来像专业客服，最后却没做任何动作。Agent 的出现正是为了解决这个落差：让模型具备 take action 的能力，可以呼叫 API、实际下单、执行指令，而不仅仅是产出文字描述。

从强化学习的角度定义 Agent，是一个能感知环境、做出动作、影响环境的实体。Rational Agent 会选择能最大化效用函数的动作。当代语言 Agent 通常透过文字作为介面，与浏览器、作业系统、档案系统、使用者等外部环境互动，核心仍是大型语言模型，外层则包覆感知、规划、推理、记忆、工具呼叫等机制。

Agent 的四大核心能力。第一是 Planning：把高阶目标拆解成可执行的小步骤。例如「去旧金山旅行」可拆成订机票、订饭店、安排行程，每个小任务再分配给对应的 Agent 或工具执行。第二是 Reasoning：Chain-of-Thought 思维链让模型在回答前先写下思考过程；ReAct 更进一步，要求模型先行动、观察结果、再决定下一步，这对需要查询或实作的任务特别关键——以 Apple Remote 为例，标准模型直接猜答案会错，加入 search action 后正确率明显提升。

第三是 Memory，分短期与长期。短期记忆放在 context window 内，但长度有限，任务复杂时容易遗忘开头；长期记忆则写入外部储存，透过 retrieval 机制在需要时取回。开源框架 Letta 展示了 Agent 如何累积生活经验、跨日保留记忆。第四是 Tool Use：Toolformer 论文证明模型可以学会在适当时机呼叫外部工具，例如计算机、QA 系统、维基百科 API——但前提是要有人工整理的训练资料教模型正确的 API 呼叫格式，否则连 GPT-4 都会给出错误参数。

多 Agent 系统则引入 orchestrator 概念，由一个总指挥 Agent 拆分任务、分派给多个专责 Agent 执行，如同交响乐团指挥。Human-in-the-Loop 是当前重要的安全设计：在关键决策点要求人类确认，避免错误自动化执行。Claude Code 偏向高度确认、安全优先；OpenHands 偏向自主执行、效率优先但风险较高。这中间的安全与便利权衡，是 Agent 落地最核心的实务问题。
