這段節目由林軒田教授介紹如何使用徑向基底函數網路（RBF Network）來做密度估計（Density Estimation），這是密度估計系列中最抽象、也最具模型觀點的一種做法。

節目一先複習機率密度函數（PDF）的概念：對一個連續隨機變數，PDF 給出落在某個位置附近的機率密度。實務上常以直方圖估計，但直方圖的問題是切 bin 寬度會嚴重影響形狀，鄰近 bin 的差異可能被過度放大或抹平。教授點出，更好的做法是把密度視為若干「高斯形的山丘」加總而成，這正是 RBF Network 的物理圖像。

RBF Network 的結構與傳統神經網路相似，但中間層不是 sigmoid 或 tanh 函數，而是徑向基底函數（Radial Basis Function）。教授先介紹 RBF 的兩個關鍵字：Basis Function 是構成整體形狀的基本元件；Radial 描述的是「以某個中心 μ 為基準，距離越遠強度越弱」的形狀，最典型的就是高斯函數。把 K 個高斯基底函數各自乘上一個權重 β，再加上一個偏置項，線性組合起來就是 RBF Network 的輸出公式。當有監督資料時，β 與 μ 可以直接學；本節重點放在無監督情境下的做法。

最簡單的無監督做法叫做樸素密度估計（Naive Density Estimation）：假設每一筆觀測到的 X 都來自一個獨立的、以該 X 為中心的高斯分布，且每個高斯的權重相等（都是 1/N）。把所有高斯加起來，就得到整體密度的估計值。這種做法後來被稱為核密度估計（Kernel Density Estimation, KDE）。

然而，KDE 把每一筆 X 都當成一個獨立高斯，過於精細且容易過擬合。教授提出更務實的做法：先用 K-Means 把資料分成 K 群，再以每群的中心作為 RBF 的中心 μ，每群的權重 β 則依群內資料比例設定。這種 hybrid 做法既保留了 RBF 的幾何直覺，又避免了 KDE 過度擬合的問題。

節目最後比較 RBF 與一般神經網路的差異：一般神經網路中間層是 sigmoid 等內積型函數，RBF 則是距離型函數；兩者的輸出層都是線性組合。教授也點出 RBF Network 在異常偵測上的應用：當一個新樣本落在 RBF 估計出的低密度區域，就能合理懷疑它是異常點。

整段節目的核心訊息是：RBF Network 把密度估計問題轉化為「以幾個代表性的高斯中心為基底，線性組合出整體分布」的模型問題；KDE 是最直觀的特例，K-Means + RBF 是更具代表性的改良方案——這條思路展示了非監督式學習如何用淺層模型逼近複雜分布。
