本集影片是 Kaggle 与 Google 共同举办的「五天 Gen AI 密集课程」第五天（最终场）的直播内容。 主讲者 Paige Bailey 与客座讲师 Ilya 带来 MLOps for Generative AI 的完整介绍。 整场直播的核心张力在于：如何把 Gen AI 从 prototype 推向 production， 并在长时间运行中保持品质。

Day 4 聚焦于 domain-specific model 在 healthcare 与 security 的应用， Day 5 转向 MLOps——把这些模型实际部署到 production 的工程实践。 从 software engineering 角度， 这代表著从 proof-of-concept 走向 production-ready 系统的成熟化路径。

Ilya 带来 end-to-end GenAI app starter pack——开箱即用的 reference 架构。 涵盖 Terraform 部署、 Vertex AI Model Garden、 Vertex AI Evaluation、 BigQuery logging 与 monitoring。 从 SRE 角度， 这正是 production 系统需要的可观察性与可维护性设计。

Q&A 环节由多位业界专家共同参与， 回顾 MLOps 的演进：2010 年前后是传统 ML 的 MLOps； late 2010s 出现完整 ecosystem； 2020s 进入 GenAI 时代后， prompt template、 chaining pipeline、 continuous evaluation 成为新核心。 从 platform engineering 角度， 这代表著工具与流程的典范转移。

DeepMind 来宾 Olivia 深入讨论 multimodal evaluation——不只评估文字， 还要评估 image 对 prompt 的对齐。 从 evaluation methodology 角度， 这代表著从单一指标走向 task-specific 多维度评估。 关键 insight 是先定义 use case 再决定 metric。

其他来宾介绍 Vertex AI 在 GenAI MLOps 的工具组合。 Vertex AI Model Garden 提供模型选择； Pipelines 提供 CI/CD 与 recurrent evaluation； Feature Store 与 Model Registry 提供 governance。 从 cloud architecture 角度， 这是 managed service 对 self-hosted 的典范转移——降低 MLOps 基础建设的维护负担。

讨论的最后聚焦在 production 阶段的 observability。 需要详细 log：prompts、 generated answers、 user feedback、 latencies。 透过 BigQuery 集中储存， 并用 drift detection 监控 model quality 是否退化。 从 incident response 角度， 这是 LLM 应用从开发阶段进入维运阶段的关键设计。

随堂测验回顾五天主题。 正确答案包括：prompt template versioning、 Vertex AI pipelines for recurrent eval、 evaluation 为衡量 model output quality。

这场直播最鲜明的讯息是：MLOps 不是 ML 的附属品， 而是 Gen AI 从实验走向 production 的桥梁。
