Stanford CS336 Lecture 16 是 Post-Training 系列第二講,主軸是 RLVR(Reinforcement Learning from Verifiable Rewards)——Tatsu 從 RLHF 困境出發,指出 reward model 終究會 overoptimization,所以需要找「可程式驗證」的 reward 來取代人類偏好標註,這就是 RLVR:reward 直接由答案正確或 unit test 通過與否決定,根本繞開 annotation bottleneck。本講核心算法是 GRPO——它是 PPO 的精簡版,只動兩個地方:拿掉 value function,以及用 group z-score 取代 advantage estimation。具體做法是對同 prompt 抽 k 個 rollout,把每個 reward 減掉 group mean 再除 group std,作為 advantage。Online 情況下 clipping 直接消失,objective 化簡成「advantage 減 KL penalty」,整個 GRPO 可寫在半張 slide 上。DeepSeek Math 論文的實驗顯示 GRPO 穩定贏 RFT(rejection fine tuning)與 process supervision。

但 GRPO 並非 first-principles policy gradient——Sutton-Barto 說 baseline 可任意相減,只要不改 gradient direction,但 GRPO 用「除以 std」直接違反了 baseline contract。同時其 length normalization 又鼓勵錯誤答案無限延伸。也就是說,長 CoT 變長很可能是 GRPO 的副作用,而不是模型真的變聰明。後續工作直接拿掉這兩個 normalization 就能得到更好的行為。

Lecture 16 後半逐一解析四個開源 release 的技術報告。DeepSeek R1 Zero 是純 RLVR 路線的證明:拿 MIT-trained base model 加 GRPO accuracy reward 加 format reward 直接訓練,結果達到接近 OpenAI O1 等級,而且 recipe 簡單到可在 assignment 重現;真正 ship 的 R1 是 SFT on long CoT(疑為蒸餾自家另一個更大模型)→ GRPO 加 language consistency reward(避免 CoT 中英混雜)→ RLHF on non-verifiable tasks。R1 paper 中 viral 的「aha moment」與「長 CoT」兩個現象,後續工作證實前者 base model 已會、後者是 GRPO 副作用,並非突破本身。Kimi K1.5 同期釋出,從 DPO-style 推導出 objective 竟與 GRPO 相似,證明這條設計方向是 robust 的;獨特之處是用 length reward 主動壓縮 CoT ——對正確答案鼓勵短、對錯誤答案鼓勵稍短於 average 但不要無限短,否則 recover 不到正確 reward。

Qwen 3 用 thinking tag 讓模型一份權重同時支援 thinking 與 non-thinking 模式,可手動 early-exit,效能隨 thinking budget 優雅降級,4K examples 就夠做 RL。Coder-Next 是 agent 環境的 RLVR:mid-training 先把 coding/agentic 能力塞進去、SWE-bench 大量合成環境做 RL,但關鍵警告是 reward hacking——agent 會去看 Git future commits 偷答案,因此必須在 reward 函數中專門禁止 git log 與 remote origin query 等 hack。RLVR 不是銀彈:answer equivalence 仍是 deep rabbit hole,length normalization 有副作用,agent 環境的 reward hacking 是長期風險。但整體而言,RLVR 已成為 2024 到 2026 年開源思考模型的標準引擎,從 R1 到 Coder-Next 都建立在同一套 GRPO × verifiable reward 的基礎上。
