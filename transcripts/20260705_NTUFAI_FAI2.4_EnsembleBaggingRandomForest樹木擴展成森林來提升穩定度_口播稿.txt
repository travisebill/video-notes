台大資訊工程學系 2026 年春季「人工智慧導論」FAI 2.4「樹木擴展成森林來提升穩定度」，由林軒田教授主講，約 19 分鐘，是 Machine Learning 章節介紹 Ensemble 的第一講。

從 Ensemble 概念角度，林軒田教授用「三個臭皮匠勝過一個諸葛亮」這句老話開場。Ensemble = 把多個弱模型組合起來，達到比單一強模型更好的效果。某門課 final project 表現最好的組就是用很多簡單 model + ensemble，最後 prediction performance 衝很高。Ensemble 是簡單但非常好用的方法，現實應用非常廣——本講聚焦 Bagging 跟 Random Forest。

從 Bagging 目標：Variance Reduction 角度，回憶 FAI 2.2 的 Bias-Variance Trade-off：Overfitting 情境是「包含到理想 $f$ 但 variance 很大」——對應打靶圖是「瞄準了但每次射出去都散開」。目標是降低 variance。怎麼降？把多次實驗平均起來：例如打靶 5 次平均起來可能很接近靶心 $f$。Bagging = Bootstrap Aggregating：把大家都放到同一個袋子裡，找 T 個變異度大的 $g_t$，平均起來 → variance 降低。

從 Bagging 做法：Random Sampling + Ensemble 角度，假設 data 有 1000 個 data point，random sample T 次，每次取 N' = 800 個（可能不同 data points）；每個 subset 各訓練一個 model，得到 T 個 $g_t$；T 個 $g_t$ 一起 average / vote。為什麼 random sample 可以讓 model 變 robust？小的 noise 很難一致地影響所有 T 個 subset，透過 averaging noise 被抵消；robustness 提升、variance 下降。Bagging 適用情境：模型本身容易 Overfitting（complex model）。

從 Random Forest：Bagging + Decision Tree 角度，Decision Tree 容易 Overfitting（切切切切切），Bagging 是解 Overfitting 的方法之一。Bagging + Decision Tree = Random Forest：Forest = 多棵 tree，Random = random sampling 不同的 subset。演算法：對 data 做 T 次 random sample → 每個 subset 跑 decision tree → 得到 T 個 $g_t$ → 最終 model = T 個 $g_t$ 的平均。Classification 用 voting（多數決），Regression 用 averaging。

從 Random Forest 優點角度，容易平行化（每棵 tree 獨立生長，學習時間 ≈ 跑一棵 tree 的時間）；保留 Decision Tree 優點（可解釋性、categorical feature、missing feature、快速訓練測試）；降低 Overfitting（Fully-grown tree 每個 leaf 只剩一個 sample 容易 Overfit，Random Forest 用 voting / average 顯著降低 variance，noise 很難 consistent 影響所有 T 棵樹 → noise 被稀釋）；可以從 data 或 feature 做 sampling（feature sampling 在高維度資料特別有效）。

從 Random Forest Decision Boundary 示意角度，1 棵 decision tree 的 boundary 很尖銳（橫切、直切），6 棵 decision tree aggregate 後 boundary 變平緩。T 越大 smoothness 越高。每棵 tree 切的位置不同（subset 不同），aggregate 後整體 boundary 是多棵 tree 的 consensus。視覺化：1 棵 vs 6 棵 vs 100 棵 → 越來越 smooth，這就是 variance reduction 的視覺化證明。

從 San Francisco vs New York 房價分類實例角度，單棵 Fully-grown decision tree：training 100%（每個 leaf 只剩一個 sample）、testing 89.7%（差約 10%）——Overfitting 明顯。Random Forest 4 棵 tree aggregate 後降低 variance。Bias-Variance 取捨再次出現：x 軸模型複雜度，bias 隨複雜度 ↑ 而 ↓，variance 隨複雜度 ↑ 而 ↑。實務建議：不要太過分切（fully-grown），可以設 terminate criteria 提早停；Model Selection 仍是非常困難的問題。

一句話總結：FAI 2.4 把 FAI 2.3 的 Decision Tree 推進到 Random Forest——用 Bagging + Decision Tree 的組合，透過 random sampling + voting/averaging 顯著降低 variance 緩解 Overfitting；保留 Decision Tree 所有優點，且容易平行化、對高維度資料效果好，是工業界最常用的 ML 模型之一；Bias-Variance 取捨依然存在，需透過大量 tree 的 ensemble 逼近 testing error 的 sweet spot。
