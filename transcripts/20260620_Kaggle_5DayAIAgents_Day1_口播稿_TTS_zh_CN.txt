本集影片是 Kaggle 与 Google 共同举办的「五天 AI Agent 密集课程」第一天的直播内容。 主讲者 Smitha Ken 是 Google Cloud 资深开发者关系工程师， 与共同主持人 Anand Navalgaria 带领观众进入 AI Agent 的开发世界。 整场直播的核心张力在于： 当 AI 从「工具」进化为「代理人」， 人类在开发流程中的角色将如何重新定义？ 影片以一份围绕 AI Agent 的白皮书为起点。 白皮书指出， 在 AI Agent 的开发流程中， 「需求规格」与「验证」将成为新的瓶颈。 过去这些环节由人类负责； 当 agent 接管复杂任务后， 这些工作也必须交给 agent 来处理。 从软体工程角度， 这呼应了经典的 specification problem——系统越复杂， 规格越难以穷举。 课程聚焦于一个核心问题： 如何让 AI Agent 在生产环境中可靠地运行。 从原型到 production 之间存在大约两成的差距。 这个差距主要来自错误处理、 一致性与案例覆盖度。 Anand 强调， 「最后一哩」需要的不只是更好的模型， 更是系统化的评估与部署流程。 从 DevOps 角度， 这正是 continuous integration 与 continuous deployment 精神在 AI Agent 领域的延伸。 在访谈中， Anand 谈到 AI 对电脑科学教育的冲击。 他认为， AI 不会取代基础技术能力， 但会大幅改变学习路径——从「写程式」转向「指导 AI 写程式」。 Anand 分享了团队常用的 prompt-to-prototype 流程， 能在短时间内将想法转化为可运行的原型。 从软体开发方法论角度， 这代表著 rapid prototyping 典范的再一次进化。 Anand 也提到了 Alpha Evolve 在 Google 内部的应用——从 DNA 序列分析到分子模拟， 这套工具已被用于解决各领域的科学问题。 AI Agent 在科学研究中的角色， 正在从辅助工具进化为共同研究者。 从 AI for Science 角度， 这印证了深度学习在跨领域泛化的能力。 第一天的程式码实作环节， 学员将安装并设定 Anti-Gravity 开发环境， 并打造一个简单的 Corgi 应用程式。 从安装到部署云端， 整个流程只需数分钟。 Anand 在示范中提到， 「过去需要数天的设定工作， 如今可以在几分钟内完成」。 从平台工程角度， 这展示了现代开发环境的成熟度。 直播的最后安排了随堂测验， 涵盖五大重点： AI Agent 的五个组成部分、 规格品质的瓶颈、 评估检查的关键角色、 隔离沙盒的执行模型， 以及长期偏好的记忆储存。 明日主题将聚焦于「互操作性」——也就是不同 agent 系统之间如何协同合作。 这场直播最鲜明的讯息是： AI Agent 的成熟度， 正卡在「最后一哩」——而这条路， 必须由人类与 AI 共同铺设。
