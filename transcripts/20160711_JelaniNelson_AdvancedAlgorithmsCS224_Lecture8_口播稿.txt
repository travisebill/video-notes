Online algorithms 的核心問題是：面對未知的未來，每次 request 都必須立即做決定，怎麼做才能讓「後悔」最少？這個問題的解答框架叫做 competitive analysis——把 online algorithm 與事先知道整個 request sequence 的 offline OPT 比較，以倍率衡量 regret。Competitive analysis 概念由 Sleator 與 Tarjan 在 1985 年提出，跟同年提出的 splay tree 是兩篇不同的論文。具體來說，若對所有 request sequence σ 來說，online A 的 cost ≤ r 乘以 OPT 的 cost 加上一個常數，就說 A 是 r-competitive。

這堂課介紹三個 toy example 帶出 competitive analysis。第一個是 ski rental：每天租雪具一塊錢，買雪具 B 塊錢，但不知道會滑雪幾天。最簡單的 2-competitive 演算法是「前 B 天都租，第 B 天就買」。這個演算法在 m ≤ B 時跟 OPT 一樣只租 m 天；在 m > B 時 OPT 會選一天買完，online 演算法成本最壂是 2B。

第二個是 move-to-front list：每次 access 某個 item 就把該 item 搬到 list 的最前端，cost 是 access 路徑長度。MTF（move-to-front）演算法對任何 access sequence σ 來說都能達到 2-competitive 加 O(n)，代表 MTF 把 hot items 自然推到前面，逼近 offline OPT 任意 reorder 的能力。

第三個是 paging：cache 大小 K、總共有 N 個 pages 散落在磁碟，每次 access 要不在 cache 就得 evict 一個。LRU 與 FIFO 都是 K-competitive。下界證明用 adversarial argument：adversary 構造 sequence 讓 online 每次 evict 一個 page，下一個 request 就是剛剛被 evict 的 page，因此 online cost 達到 K × OPT cost。所以對 paging 來說，K 是 deterministic online 的 tight bound，LRU 與 FIFO 都達到這個 bound。

但這個 K 的 tight bound 對 deterministic 而言，randomized 是否能突破？這就是下堂課要回答的問題——Mark 演算法達 2 H_K competitive，其中 H_K 是第 K 個 harmonic number，代表 randomize 確實比 deterministic LRU 的 K competitive 更好，但要付出 randomness 的代價。

整堂課展示的核心思想在於：online 算法不能跟 OPT 單純比成本，而是要跟「在同樣 input 上 OPT 必須付出的成本」比。這個 framework 讓許多看似無解的決策問題——滑雪買租、cache 替換、list reorder——能被厳谧分析並設計出證明上限的演算法。Sleator-Tarjan 1985 的這套方法論之後廣泛應用於 algorithm 設計，如 k-server problem、metrical task system、online convex optimization 等不同抽象的 online 問題。
