Stanford CS336 Lecture 17 把 Post-Training 拉到 Multimodal Alignment。Percy Liang 開宗明義:transformer 仍是最通用的跨模態 backbone,關鍵是把所有 modality 變成 transformer 吃得下的 tokens;文字已是 subword tokens,圖像要 patch-based tokenization。沿 2021 到 2025 時間軸展開。

CLIP 在 2021 由 OpenAI 提出,是現代 VLM foundation:用 400M image-text 對做 contrastive learning,對齊 dot product 再 softmax,拉近配對推開非配對;圖像編碼器用 ViT-L/14 加 attention pooling,zero-shot 在 ImageNet 打敗 ResNet trained on 1.2M labels——但 CLIP 有兩個痛點:30K+ 大 batch 是必要條件、softmax over 整 batch 不可平行化。Google 的 SigLIP 把 softmax 改成 sigmoid binary classification,對角線 +1、其他 −1,loss 完全跟 batch size 解耦,16K batch 就贏 CLIP;WebLI dataset 達 1B 到 10B image-text pairs,含 OCR,訓練比 CLIP 快數倍。LLaVA 2023 把 CLIP 加 linear projector 加 Vicuna LLM 拼起來,兩階段訓練:Stage 1 只 train projector,Stage 2 train projector 加 LLM,158K 用 GPT-4 從 MS COCO 生成的對話 examples,證明 open model 可做視覺推理。

LLaVA OneVision 2024 升級三大件:用 SigLIP 取代 CLIP、Qwen-2 取代 Vicuna、projector 改兩層 MLP,推出 AnyRes 動態解析度處理 OCR——把大 image 切成多塊 336×336 各自編碼再 concate vectors,語言 transformer 本來就能處理變長輸入。Qwen-VL 系列三代演進:Qwen-VL 用 OpenCLIP 加單層 cross-attention adapter,固定 256 tokens;Qwen-2-VL 推到 dynamic resolution 加 Multimodal RoPE 把 RoPE 推到 3D height 乘 width 乘 time;Qwen-3-VL 用 SigLIP-2 改進 M-RoPE 交錯 three-axis dimension 避免低/高頻聚集,explicit video timestamp tokens 可被 reference,用 √L per-token loss normalization 避免 long video 壓過 short text,DeepStack adapter 把 vision encoder 每一層 output 直接加入 LLM residual stream 做 deep fusion。

Chameleon 2024 走另一條路:VQ-VAE 把 image 變 1024 個 discrete tokens 從 8000 codebook,然後全部 LLM 一個模型處理 text 加 image,但 training 不穩定——text token 低 entropy、image token 高 entropy,混在一起 norm 會爆掉,只能 QK norm 加 z-loss regularization 緩解;discretization 也讓 OCR 細節掉光。Percy 最後反思理解模態跟生成模態本質不對稱:CLIP 等 high-level semantic encoder 對 classification 夠用,但 OCR 或 image generation 需要 fine-grained detail,必須 diffusion 才能做;沒有 single universal encoder 能 cover 所有需求,這是 multimodal 設計的根本限制。Vision encoder 參數通常小於 1B,patch 是 local operation,大部分知識仍在 LLM。
