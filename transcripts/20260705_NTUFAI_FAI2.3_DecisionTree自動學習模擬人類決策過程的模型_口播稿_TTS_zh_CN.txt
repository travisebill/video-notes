台大资讯工程学系 2026 年春季「人工智慧导论」FAI 2. 3「自动学习模拟人类决策过程的模型」， 由林轩田教授主讲， 约 36 分钟， 是 Machine Learning 章节介绍 Decision Tree 的第一讲。

从 AI 历史与 Expert System 限制角度， 1960-70 年代 Expert System 曾是 AI 顶峰——领域专家把 if-and-else 规则写进程式。 但 Expert System 不够通用， 因为人类很难把所有情况都写进 SOP； 写不完的 case 还是需要真人处理， 于是 1980 年代进入第二次 AI 寒冬。 Decision Tree 从这个寒冬崛起——「从资料里学出 if-and-else 规则」， 既像专家系统（学出来可以写成 if-else）， 又从资料学（比人手写更 general、 更全面）， 于是逐渐取代传统 Expert System， 从第二次寒冬 Exponential 成长到现在。

从写作业 Decision Tree 范例角度， 林轩田教授用同学决定要不要写作业的决策树开场：deadline 大于两周 → 不写； 一到两周看题目数量（大于 10 题开始写， 否则不写）； 小于一周看朋友进度（有朋友写完就写， 否则不写）。 这棵树的每个 internal node 都是条件、 每个 leaf 都是决定。 如果收集到「学生写作业 / 不写作业」的历史资料， 就能从资料学出这棵树——人说不出自己怎么决策， 但 data 学得出来。

从 Path-based 数学式角度， 每个 leaf 是 base hypothesis $g_t(x)$（分类用 majority、 回归用 average）； path condition $q_t(x)$ = 0/1 indicator 决定 $x$ 是否满足这条 path。 整个 tree = $G(x) = \sum_t q_t(x) \cdot g_t(x)$， 是所有 path 的加总。 每个 leaf 都是「满足某一组条件」的 $x$ 集合。

从 Recursive 数学式角度， $G(x)$ 从 root 开始； $b(x)$ 是 branching criteria； 如果 $b(x, c)$ 满足就展开成 subtree $G_c(x)$。 整棵 tree = $b(x)$ + 多个 subtree 加总。 这是 divide-and-conquer 的 recursive 演算法——对应到程式：recursive function， base case 要写出来（否则 segmentation fault）。

从 CART 演算法的四个关键决定角度， 整个 Decision Tree algorithm 包含四个 choice：(1) 每次切几个 branch（CART 用 2, binary）； (2) base hypothesis $g_t(x)$（分类 majority、 回归 average）； (3) branching criteria（dimension + threshold）； (4) terminate criteria（fully-grown 或 performance threshold）。 不同的 choice 对应不同演算法（CART、 C4. 5、 ID3 等）——所以 Decision Tree 不是单一演算法而是一个 category。

从 CART 在 2D 空间示意角度， 2D 二元分类只能找横线或直线切（不能斜切）。 过程：找 dimension + threshold 切一刀 → 分开后对 subset recursive 再切 → 完全分开为止。 例如 San Francisco vs New York 房价分类：先切直的把高 elevation 的旧金山分开 → 再对剩下的切横的把高价位的纽约分开 → 总共四次 branching、 四个 condition。

从 Decision Tree 优点角度， 可解释性高（学出来看哪些 dimension 重要、 threshold 多少）、 对 categorical feature 友善（不需 encoding 直接处理类别型 feature）、 对 missing feature 友善（某 dimension 没值跳过就好）、 训练与测试都快（divide-and-conquer 训练、 if-else 测试）。

从 Decision Tree 缺点角度， 容易 Overfitting（切太细把 noise 学进去）、 对类别多的 categorical feature 有偏误（倾向用该 feature 切）、 不能 model feature 间 interaction（每个 dimension 独立切）。 San Francisco vs New York 房价分类实例：training accuracy 100% 但 testing 89. 7%——Overfit 约 10%， 这为 FAI 2. 4 Random Forest 埋下伏笔。

一句话总结：FAI 2. 3 介绍第一个具体的非线性模型 Decision Tree——模拟人类 if-and-else 决策逻辑， 从资料自动学出； 用 CART 演算法的 recursive formulation， 可解释性强、 可处理 categorical/missing feature， 但容易 Overfitting； FAI 2. 4 将用 Random Forest（ensemble 多棵 tree）解决 Overfitting 问题。
