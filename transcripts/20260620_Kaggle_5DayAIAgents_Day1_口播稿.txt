本集影片是 Kaggle 與 Google 共同舉辦的「五天 AI Agent 密集課程」第一天的直播內容。 主講者 Smitha Ken 是 Google Cloud 資深開發者關係工程師， 與共同主持人 Anand Navalgaria 帶領觀眾進入 AI Agent 的開發世界。 整場直播的核心張力在於： 當 AI 從「工具」進化為「代理人」，人類在開發流程中的角色將如何重新定義？ 影片以一份圍繞 AI Agent 的白皮書為起點。 白皮書指出，在 AI Agent 的開發流程中，「需求規格」與「驗證」將成為新的瓶頸。 過去這些環節由人類負責； 當 agent 接管複雜任務後，這些工作也必須交給 agent 來處理。 從軟體工程角度，這呼應了經典的 specification problem——系統越複雜，規格越難以窮舉。 課程聚焦於一個核心問題： 如何讓 AI Agent 在生產環境中可靠地運行。 從原型到 production 之間存在大約兩成的差距。 這個差距主要來自錯誤處理、一致性與案例覆蓋度。 Anand 強調，「最後一哩」需要的不只是更好的模型，更是系統化的評估與部署流程。 從 DevOps 角度，這正是 continuous integration 與 continuous deployment 精神在 AI Agent 領域的延伸。 在訪談中，Anand 談到 AI 對電腦科學教育的衝擊。 他認為，AI 不會取代基礎技術能力，但會大幅改變學習路徑——從「寫程式」轉向「指導 AI 寫程式」。 Anand 分享了團隊常用的 prompt-to-prototype 流程，能在短時間內將想法轉化為可運行的原型。 從軟體開發方法論角度，這代表著 rapid prototyping 典範的再一次進化。 Anand 也提到了 Alpha Evolve 在 Google 內部的應用——從 DNA 序列分析到分子模擬，這套工具已被用於解決各領域的科學問題。 AI Agent 在科學研究中的角色，正在從輔助工具進化為共同研究者。 從 AI for Science 角度，這印證了深度學習在跨領域泛化的能力。 第一天的程式碼實作環節，學員將安裝並設定 Anti-Gravity 開發環境，並打造一個簡單的 Corgi 應用程式。 從安裝到部署雲端，整個流程只需數分鐘。 Anand 在示範中提到，「過去需要數天的設定工作，如今可以在幾分鐘內完成」。 從平台工程角度，這展示了現代開發環境的成熟度。 直播的最後安排了隨堂測驗，涵蓋五大重點： AI Agent 的五個組成部分、規格品質的瓶頸、評估檢查的關鍵角色、隔離沙盒的執行模型，以及長期偏好的記憶儲存。 明日主題將聚焦於「互操作性」——也就是不同 agent 系統之間如何協同合作。 這場直播最鮮明的訊息是： AI Agent 的成熟度，正卡在「最後一哩」——而這條路，必須由人類與 AI 共同鋪設。
