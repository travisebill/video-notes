本集介绍台大资讯系「人工智慧导论（FAI 2026）」的 Final Project 说明会，由四位教学 TA 主讲，主题为设计一个玩德国卡牌游戏「谁是牛头王」的 Game Agent。

游戏规则相对单纯：四位玩家、每场各发十张牌、共十轮，每轮各出一张牌依数字大小排序进到四个牌堆之一。卡牌的尾数决定了 Penalty 数量：尾数五是两颗牛头、尾数零是三颗、结尾是零零多达五颗、最危险的结尾五五则高达七颗。十轮结束时，牛头总数最少的玩家胜出。然而真正变化莫测的是收牌机制：当某个牌堆已经累积五张牌时，下一个出牌者会成为第六张玩家，被迫收走整堆六张牌作为 Penalty。另一方面，若手牌太小、无法接续任何牌堆结尾，演算法会自动挑选一个最不亏的牌堆先丢掉，再把这张小牌放到堆顶。

Game Engine 是这份作业的核心，由三段式架构组成：Initialization 负责发牌与确认玩家名单、PlayGame 跑完整场十轮游戏、PlayRound 则是每轮的核心回圈，负责呼叫每位 Agent 的决策函式并执行四项检查：决策时间不得超过一秒、输出必须是合法整数、记忆体使用不得超过一 GB、且禁止任何额外的 Multi-process 或 Multi-thread。违规处置分为两层；伪违规如尝试窜改 Game History 或 Random Seed 不会扣分，因为系统每次都传递复制版、并使用自制 RNG，重设完全无效；但真违规如记忆体爆炸导致 Process 被 kill、额外 spawn 子行程、或故意 catch TimeoutException 来规避时间限制，会分别被扣最高 50% 到 70% 的分数。

锦标赛赛制方面，系统将全体学生 Agent 随机四人分成 Partition，每个 Partition 跑多场对局，全场至少跑五百个 Partition 以消除随机波动。每场对局的四个名次会换成分数，平手时平均分配，整场总分取所有场次的平均 Rank，再线性内插到零到一百分。评分政策分为两阶段：四月二十九日的 Mid Submission 占十五分，跑四十个 Face Line；六月十四日的 Final Submission 占五十五分，跑五十五个 Face Line。线性内插的分数门槛设计相当合理：Mid 阶段赢过第五条 Face Line 保底六十分、赢过第二十条可达九十分；Final 阶段赢过第二十条保底四十分、赢过第五十条可达九十分。

额外加分机制也值得一提：当某位 Agent 在同一个 Board 上面对所有对手的决策分布明显不同、却仍能胜出时，最多可额外加五分。这项 Novelty 加分采取主观与量化双重标准，TA 主观看 Report 加上系统跑 script 比对所有 Agent 的决策机率分布，避免全班策略过于同质化。

开发环境条件严格：Python 三点一三点一一、CPU only、无 GPU、无网路、评分环境保证与测试机同级但可能更强。写 Agent 的介面极为精简，只需要实作一个 Player Class，含两个函式：初始化函式接收玩家编号、决策函式接收当前手牌与完整游戏历史并回传要出的牌。学生特别需要注意两件事：手牌以 Reference 传递不可直接修改、不要自行储存内部状态，每次都从传入的历史重算所有需要的数值，这是为了防止 Process 被 kill 后状态不一致造成后续回合崩溃。

提交时每位学生需缴交两个最佳 Agent、所有相关 code、以及一份不超过三页的 Report，内含尝试过的方法、为何选择这两个 Agent、未来改进方向等。整份 Project 的精髓在于以工业级严苛条件为框架，把课堂所学的 Exploitation 与 Exploration、Search、Reinforcement Learning 概念压缩到一秒决策与一 GB 记忆体上限的真实对战之中。
