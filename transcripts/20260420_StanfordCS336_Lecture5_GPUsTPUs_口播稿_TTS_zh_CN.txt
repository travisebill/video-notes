CS336 第五讲由 Percy Liang 主讲,正式进入 systems module 的核心:GPU 程式设计与效能工程。开场 Percy 承诺,听完这堂课学生会理解 matmul throughput plot 为什么有那些奇怪波动,以及如何让自己的程式码跑在 roofline 饱和区。

CPU 与 GPU 的哲学差异是理解一切的起点。CPU 是 latency 导向——大控制单元配少量 ALU,适合复杂 branching 与低延迟。GPU 是 throughput 导向——数百个 lightweight cores 换 aggregate throughput,适合大量平行运算。每个 task 在 GPU 上跑得久,但总吞吐量远高于 CPU。

GPU 硬体基本单元是 Streaming Multiprocessor(SM),H100 有 132 个 SM。每个 SM 有自己的 L1 cache 与 shared memory,latency 只要 20-30 cycle;L2 cache 跨 SM 共用,慢一点;HBM global memory 是 GPU 主流外部记忆体,H200 有 144GB 但 latency 是 L1 的十倍。这层 memory hierarchy 是现代 LLM optimization 的核心——memory bandwidth,不是 compute,才是真正瓶颈。为什么不全用 SRAM?SRAM 贵几百倍且耗电,memory hierarchy 是 cost 与 speed 的 trade-off。Grok(Nvidia 收购)是例外,用巨大 SRAM 设计 inference,证明特定 workload 下 SRAM 确实有效。

Tensor Cores 是 GPU 的矩阵运算加速器。V100(2017)首次引入,接下来 structured sparsity、FP8 等低精度格式推动 FLOPs 持续成长。FP32 到 BF16 到 FP8 到 FP4 的精度演进,bits 减半意味著 memory 流量也减半,大幅缓解 memory bottleneck。

TPUs 是 Google 的 GPU 替代方案,核心是 systolic array——streaming data through fixed circuit,强制 batch 大才有高效率。GPU 通常 132 个 SM 与 528 个矩阵运算单元,TPU 只有 2 个 tensor core 与 8 个 MXU。TPU 与 GPU 虽然架构不同但在 memory hierarchy 与 matrix multiply unit 这两大核心上高度收敛——这是 convergent evolution 的结果。注意 TPU 与 GPU 都叫 tensor core,但意思不同——TPU 的 tensor core 是 processor,GPU 的 tensor core 是 matrix multiply unit。

GPU 程式设计用 SIMD,threads 非常 lightweight——scheduler 可随时 swap 停滞的 warp,换其他 warp 执行以保持 throughput 高。早期人们手写 shader 做 matrix multiply,现在透过 Tensor Cores 与高阶 kernel 语言如 Triton。

Performance engineering 有六大策略:vectorization 用 SIMD 一次算多元素,parallelism 用多 thread 多 SM 并行,memory hierarchy exploitation 优先用 shared memory 与 register,operator fusion 把多 op 合成一个 kernel,loop transformations 如 tiling 改善 cache locality,low precision 用 BF16/FP8/FP4 减 bits。

Operator fusion 是最关键的策略。原本 read 然后 compute A 然后 write 然后 read 然后 compute B 然后 write,两轮 HBM 流量;改成 read 然后 compute A 加 B 然后 write,一轮 HBM 流量,直接省一半。Tiling 是 fusion 的进阶版,把大矩阵切成 tile,每个 tile 在 shared memory 完成计算后再写回。

Flash Attention 是这套思维的极致展现。标准 attention 需要存完整的 n 乘 n attention matrix,记忆体 O(n²)。Flash Attention 透过 tiling 加 online softmax,不存完整 attention matrix,改用 block by block 计算,让 attention 的 memory complexity 从 O(n²) 降到 O(n)。这是长 context 训练的关键突破,也是 Assignment 2 的核心 kernel。

Matmul throughput plot 的奇怪波动来自三个因素:matrix size 对齐 Tensor Core 要求与否、L2 cache 命中与否、warp scheduling 平衡。Roofline model 解释了整体形状——从 memory-bound 对角线成长,到 compute-bound 水平饱和。所有优化策略都是为了把 op 推到 roofline 饱和区。

一句话总结:GPU 与 TPU 在 memory hierarchy 与 matrix multiply unit 高度收敛,所有 kernel 优化都围绕最小化 data movement 展开,因为现代硬体瓶颈是 memory bandwidth。Flash Attention 把这套思维推到极致,让 O(n²) memory 降到 O(n),是现代 LLM 长 context 训练的基石。
